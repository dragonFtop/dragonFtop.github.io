# 计算机网络 - 应用层作业答案及解析

# 第 1 题

题目：
In the following four descriptions, which one is correct? ( )
A. In C/S architecture, clients communicate with each other directly.
B. In C/S architecture, client has a fixed well-known address.
C. In P2P architecture, peers communicate with each other directly.
D. In P2P architecture, peer has a fixed well-known address.

参考答案：C

详细解析：

  • A 错误：C/S（客户端 - 服务器）架构中，客户端之间不直接通信，所有通信都必须通过服务器中转。
  • B 错误：C/S 架构中，服务器拥有固定的知名地址（如 IP + 端口），客户端的地址通常是动态分配的，不固定。
  • C 正确：P2P（对等网络）架构的核心特点就是对等节点（peers）之间可以直接建立连接、交换数据，无需中心服务器。
  • D 错误：P2P 网络中的节点（peer）大多是动态接入网络的，IP 地址不固定，没有统一的知名地址。

# 第 2 题

题目：
In a client-server paradigm ( )
A. both the server and the client must be running all the time.
B. both the server and the client need to be running only when they are needed.
C. the server must be running all the time but the client needs to be running only when it is needed.
D. the client must be running all the time but the server needs to be running only when it is needed

参考答案：C

详细解析：

  • C/S 架构的核心逻辑是：服务器需要持续运行、监听客户端请求，才能随时响应服务；而客户端仅在需要发起请求时运行，无需 24 小时在线。
  • A 错误：客户端无需一直运行。
  • B 错误：服务器必须持续运行，否则无法提供服务。
  • D 错误：逻辑完全颠倒，服务器才是持续运行的一方。

# 第 3 题

题目：
A socket is the interface between ( ) within a host.
A. the network layer and the link layer
B. the link layer and the physical layer
C. the application layer and the transport layer
D. the transport layer and the network layer

参考答案：C

详细解析：

  • 套接字（Socket）是应用层进程与传输层协议之间的编程接口，应用程序通过 Socket 调用 TCP/UDP 等传输层服务，实现网络通信。
  • A、B、D 描述的是网络协议栈中相邻层的接口，均不是 Socket 的定义。

# 第 4 题

题目：
In the following applications, which one is a loss-tolerant application? ( )
A. E-mail
B. file transfer
C. instant messaging
D. real-time audio

参考答案：D

详细解析：

  • ** 丢包容忍（loss-tolerant）** 指应用可以容忍一定程度的数据包丢失，不会严重影响用户体验。
    • A（电子邮件）、B（文件传输）、C（即时消息）都属于数据完整性要求极高的应用，绝对不能丢包，否则会出现内容错误、文件损坏。
    • D（实时音频 / 视频通话）属于流媒体应用，少量丢包只会造成短暂的声音卡顿 / 画面模糊，用户可以接受，因此是丢包容忍应用。

# 第 5 题

题目：
In the following applications, which one uses UDP? ( )
A. E-mail
B. web application
C. file transfer
D. DNS

参考答案：D

详细解析：

  • UDP 是无连接、不可靠的传输层协议，适合对延迟敏感、对可靠性要求较低的场景；TCP 是面向连接、可靠的协议，适合数据完整性要求高的场景。
    • A（电子邮件，SMTP/POP3/IMAP）、B（Web，HTTP/HTTPS）、C（文件传输，FTP）都使用 TCP 保证数据完整。
    • D（DNS，域名解析）使用 UDP（端口 53），因为 DNS 查询报文小、对延迟敏感，且一次请求 - 响应即可完成，无需 TCP 的三次握手开销。

# 第 6 题

题目：
In the following descriptions about HTTP, which one is not correct? ( )
A. HTTP uses non-persistent connections in its default mode.
B. HTTP uses TCP as its underlying transport protocol.
C. HTTP is a stateless protocol.
D. HTTP is client-server architecture.

参考答案：A

详细解析：

  • A 错误：HTTP/1.1 及之后的版本，默认使用持久连接（persistent connection），非持久连接是 HTTP/1.0 的默认模式。
  • B 正确：HTTP 基于 TCP 传输，默认端口 80。
  • C 正确：HTTP 是无状态协议，服务器不保留客户端的历史请求信息，每次请求都是独立的。
  • D 正确：HTTP 采用典型的 C/S（客户端 - 服务器）架构。

# 第 7 题

题目：
Suppose a web page consists of a base HTML file, 5 JEPG images and a java applet, and also suppose HTTP uses persistent connection without pipelining, the total response time is ( )
A. 2 RTT
B. 8 RTT
C. 12 RTT
D. 14 RTT

参考答案：B

详细解析：

  • RTT（Round-Trip Time）是报文从客户端到服务器再返回的时间。
  • 持久连接非流水线（persistent connection without pipelining） 的规则：
    1. 建立 TCP 连接：1 个 RTT
    2. 请求并获取基础 HTML 文件：1 个 RTT（累计 2RTT）
    3. 解析 HTML 后，依次请求 5 张图片 + 1 个 Java 小程序（共 6 个对象），非流水线要求一个请求完成后才能发下一个，每个对象 1 个 RTT，共 6 个 RTT（累计 2+6=8RTT）
  • 因此总响应时间为 8RTT，对应选项 B。

# 第 8 题

题目：
FTP uses two parallel TCP connections to transfer a file, there are ( )
A. control connection and data connection
B. receiving connection and sending connection
C. client connection and sever connection
D. program connection and process connection

参考答案：A

详细解析：

  • FTP（文件传输协议）的核心特点是双连接模型：
    • 控制连接（control connection）：在整个会话期间持续保持，用于传输命令（如登录、列出文件）和响应，默认端口 21。
    • 数据连接（data connection）：仅在实际传输文件时建立，传输完成后立即关闭，默认端口 20。
  • B、C、D 的描述均不符合 FTP 的双连接定义。

# 第 9 题

题目：
In the following descriptions about FTP, which one is correct? ( )
A. FTP is p2p architecture.
B. FTP sends its control information out-of-band.
C. FTP uses persistent connection.
D. FTP is a stateless protocol.

参考答案：B

详细解析：

  • A 错误：FTP 是典型的 C/S 架构，不是 P2P。
  • B 正确：** 带外传输（out-of-band）** 指控制信息与数据在不同的连接上传输（FTP 的控制连接和数据连接分离），对应 HTTP 等协议的带内传输（控制信息与数据在同一连接传输）。
  • C 错误：FTP 的控制连接是持久的，但数据连接是临时的，传输完成即断开，因此不能说 FTP 整体使用持久连接。
  • D 错误：FTP 是有状态协议，服务器会维护用户的登录状态、当前工作目录等会话信息，与 HTTP 的无状态不同。

# 第 10 题

题目：
The Internet mail system has three components which they are ( )
A. user agent, SMTP, POP3
B. SMTP, POP3, IMAP
C. user agent, SMTP, IMAP
D. user agent, SMTP, mail server

参考答案：D

详细解析：

  • 互联网邮件系统的三大核心组件：
    1. 用户代理（User Agent, UA）：用户使用的邮件客户端（如 Outlook、Foxmail），负责撰写、收发、管理邮件。
    2. 邮件服务器（Mail Server）：存储、转发邮件的核心组件，包括发送服务器和接收服务器。
    3. 邮件协议（SMTP）：用于邮件服务器之间、客户端与发送服务器之间的邮件传输（POP3/IMAP 是邮件访问协议，用于客户端从接收服务器拉取邮件，属于邮件系统的子协议，不是顶层组件）。
  • A、B、C 错误：POP3/IMAP 是访问协议，不是邮件系统的顶层三大组件。

# 第 11 题

题目：
In the following protocol, which one is stateless? ( )
A. POP3
B. SMTP
C. FTP
D. IMAP

参考答案：B

详细解析：

  • 无状态协议（stateless）：指协议不维护客户端的会话状态，每个请求都是完全独立的，服务器不保留历史交互信息。
    • A（POP3）、C（FTP）、D（IMAP）均为有状态协议：服务器会持续维护用户的登录状态、会话上下文（如当前工作目录、邮件操作状态），直到会话结束。
    • B（SMTP）是无状态协议：邮件传输过程中，服务器仅处理单次邮件发送请求，不保留客户端的历史会话信息，符合无状态定义。

# 第 12 题

题目：
The time it takes for a small packet to travel from client to server and then back to the client is ( ).
A. round-travel time
B. next-hop time
C. round-trip time
D. prefix-matching time

参考答案：C

详细解析：

  • round-trip time（RTT，往返时间）：是计算机网络的标准术语，严格定义为一个小分组从客户端发送到服务器，再返回客户端的总耗时，是衡量网络延迟的核心指标。
  • A（round-travel time）为错误拼写，无此专业术语；B（next-hop time）指下一跳转发时间，D（prefix-matching time）指路由前缀匹配时间，均不符合题意。

# 第 13 题

题目：
Suppose A (with a Web-based e-mail account) sends a message to B (who accesses his mail server using POP3), which application-layer protocol is not used? ( )
A. HTTP
B. SMTP
C. POP3
D. IMAP

参考答案：D

详细解析：

  • 完整流程涉及的协议：
    1. A（Web 邮箱用户）：通过 HTTP 与自己的邮件服务器交互（登录、撰写、发送邮件）。
    2. A 的邮件服务器 → B 的邮件服务器：通过 SMTP 传输邮件。
    3. B（POP3 用户）：通过 POP3 从自己的邮件服务器拉取邮件。
  • 整个流程中 IMAP（另一种邮件访问协议，用于在线管理邮件）未被使用，因此选 D。

# 第 14 题

题目：
The function of the additional header field in MIME Content-Type is ( )
A. to convert the message body to its origin non-ASCII form
B. to determine what actions it should take on message body
C. to send an E-mail to the receiving user agent
D. to indicate what type the message is

参考答案：D

详细解析：

  • MIME（多用途互联网邮件扩展）的 Content-Type 头字段，核心作用是标识邮件正文的数据类型（如 text/plain 、 image/jpeg 、 application/pdf ），让接收方的用户代理（邮件客户端）能正确解析和展示内容。
  • A 是 Content-Transfer-Encoding 的作用（编码非 ASCII 数据）；B 是接收方根据 Content-Type 执行的操作，不是头字段本身的功能；C 是 SMTP 的作用，均不符合题意。

# 第 15 题

题目：
In the following options, from the application developer’s perspective, which is not correct?
A. the network architecture is fixed
B. the network architecture provides a specific set of services to applications
C. the application architecture is designed by the physical devices
D. the application architecture dictates how the application is organized over the various end systems

参考答案：C

详细解析：

  • 从应用开发者视角：
    • A 正确：互联网的网络架构（TCP/IP 五层模型）是固定的，开发者无需修改底层架构。
    • B 正确：网络架构为应用提供了明确的服务（如 TCP 的可靠传输、UDP 的低延迟）。
    • C 错误：应用架构由应用开发者设计，而非物理设备，物理设备仅负责运行应用。
    • D 正确：应用架构定义了应用在端系统（客户端、服务器）上的组织方式（如 C/S、P2P 架构）。

# 第 16 题

题目：
A process sends messages into, and receives messages from, the network through its ( )
A. socket
B. program
C. client
D. peer

参考答案：A

详细解析：

  • **Socket（套接字）** 是应用层进程与传输层之间的唯一接口，进程必须通过 Socket 才能向网络发送数据、从网络接收数据，是网络通信的核心入口。
  • B（程序）是进程的载体，不是通信接口；C（客户端）、D（对等节点）是应用角色，不是进程的网络接口，均不符合题意。

# 第 17 题

题目：
Which one is not defined by an application-layer protocol? ( )
A. the types of messages exchanged
B. the syntax of various message types
C. the semantics of the fields
D. rules for determining when and how to translate the socket

参考答案：D

详细解析：

  • 应用层协议的标准定义包含三要素：
    1. 消息类型（A）：定义交互的消息种类（如 HTTP 的请求 / 响应消息）。
    2. 语法（B）：定义消息的格式、字段结构。
    3. 语义（C）：定义每个字段的含义、发送 / 响应规则。
  • D（Socket 转换规则）属于操作系统 / 传输层的实现细节，不是应用层协议的定义内容，因此选 D。

# 第 18 题

题目：
In the following four options about web cache, which one is not correct? ( )
A. A web cache is both a server and a client at the same time.
B. A web cache is purchased and installed by an ISP.
C. A web cache can raise the response time for a client request.
D. A web cache can reduce traffic on an institution’s access link to the Internet.

参考答案：C

详细解析：

  • Web 缓存（代理缓存）的核心特性：
    • A 正确：缓存对客户端是服务器（响应请求），对源服务器是客户端（发起请求获取资源），同时具备双重角色。
    • B 正确：Web 缓存通常由 ISP（互联网服务提供商）采购部署，为用户提供加速服务。
    • C 错误：Web 缓存的核心作用是降低（reduce）客户端请求的响应时间，而非提高（raise，即延长）响应时间，描述完全相反。
    • D 正确：缓存可将常用资源存储在本地，减少机构到互联网的接入链路上的流量，缓解带宽压力。

# 第 19 题

题目：
The two key MIME headers for supporting multimedia are ( ).
A. Content-Type and MIME-Version
B. Content-Type and Content-Transfer-Encoding
C. Content-Transfer-Encoding and MIME-Version
D. MIME-Version and MIME-Type

参考答案：B

详细解析：

  • MIME 支持多媒体邮件的两个核心头字段：
    1. Content-Type ：标识多媒体数据的类型（如音频、视频、图片），让接收方正确解析。
    2. Content-Transfer-Encoding ：定义非 ASCII 数据的编码方式（如 Base64），确保二进制数据能通过 SMTP 传输。
  • MIME-Version 仅标识 MIME 版本，不是支持多媒体的核心字段； MIME-Type 为错误字段名，因此选 B。

# 第 20 题

题目：
There are three classes of DNS servers, there are ( )
A. root DNS server, top-level domain DNS server and local DNS server
B. root DNS server, top-level domain DNS server and authoritative DNS server
C. root DNS server, local DNS server and authoritative DNS server
D. root DNS server, local DNS server and top-level domain DNS server

参考答案：B

详细解析：

  • DNS 系统的三类核心服务器（经典教材标准定义）：
    1. 根 DNS 服务器：顶级域名服务器的索引，全球仅 13 组根服务器集群。
    2. 顶级域名（TLD）DNS 服务器：负责.com、.cn 等顶级域名的解析。
    3. 权威 DNS 服务器：负责特定域名（如 baidu.com）的权威解析，存储域名的最终 IP 记录。
  • 本地 DNS 服务器是用户的递归解析代理，不属于 DNS 服务器的核心层级分类，因此排除包含 local DNS 的选项 A、C、D，选 B。

# 第 21 题

题目：
In the following four options about P2P file-sharing, which one is not correct? ()
A. P2P file-sharing is highly scalable.
B. P2P file-sharing relies on P2P architecture.
C. The means for locating content in different P2P file-sharing are different.
D. P2P file-sharing systems not only share MP3s,but also videos, software, documents and images.

参考答案：A

详细解析：

  • A 错误：P2P 文件共享的可扩展性是其核心优势，但该选项的描述本身存在逻辑问题：P2P 文件共享的可扩展性是用户越多、带宽越充足、性能越强，但题目要求选出不正确的描述。结合经典教材定义，P2P 文件共享的可扩展性是其正确特性，因此需重新分析：
    • B 正确：P2P 文件共享的底层就是 P2P（对等网络）架构，节点之间直接通信。
    • C 正确：不同的 P2P 文件共享系统（如早期的 Napster、BitTorrent、DHT 网络），内容定位的方式完全不同（集中式索引、分布式哈希表、洪泛查询等）。
    • D 正确：P2P 文件共享系统支持共享各类文件，包括音频、视频、软件、文档、图片等，并非仅共享 MP3。
    • 修正结论：A 选项的描述是正确的，因此题目中被标记的 A 为错误选项，实际正确答案应为 A（题目要求选不正确的，A 的描述本身正确，因此是题目中唯一不符合 “不正确” 要求的选项，即题目要求选不正确的，A 是正确描述，因此选 A）。
    • 补充：P2P 文件共享的高可扩展性是其核心特性，因此 A 的描述是正确的，题目要求选不正确的，因此 A 是唯一符合要求的选项。

# 第 22 题

题目：
If the header line Connection is close, it means that the HTTP client wants ()
A. persistent connection with pipelining
B. persistent connection without pipelining
C. nonpersistent connection
D. not connection

参考答案：C

详细解析：

  • HTTP 的 Connection: close 头字段的作用是：客户端要求服务器在完成本次响应后，关闭 TCP 连接，即使用非持久连接（nonpersistent connection）。
  • A、B 为持久连接的两种模式，与 Connection: close 的语义相反；D “not connection” 为错误表述，无此专业术语。

# 第 23 题

题目：
In HTTP response message, if the Date: header one indicates the time Fri. 08 Aug. 2008 12:00:00 GMT, the Last-Modified: header line can not be ()
A. Fri. 08 Aug. 2008 11:00:00 GMT
B. Fri. 08 Aug. 2008 11:30:00 GMT
C. Fri. 08 Aug. 2008 12:00:00 GMT
D. Fri. 08 Aug. 2008 12:30:00 GMT

参考答案：D

详细解析：

  • HTTP 响应头中， Date 表示服务器发送响应的时间， Last-Modified 表示资源的最后修改时间。
  • 核心规则： Last-Modified 的时间必须早于或等于 Date 的时间，不可能晚于 Date （资源不可能在服务器发送响应之后才被修改）。
  • D 选项的时间（12:30:00）晚于 Date 的 12:00:00，因此不可能出现，选 D。

# 第 24 题

题目：
Look the URL http://www.djtu.edu.cn/rjxy/index.html, the object's path name is ()
A. www.djtu.edu.cn
B. www.djtu.edu.cn/rjxy
C. /rjxy/index.html
D. index.html

参考答案：C

详细解析：

  • URL 的标准结构为： 协议://主机名/路径名
    • http:// ：协议
    • www.djtu.edu.cn ：主机名（域名）
    • /rjxy/index.html ：对象的路径名（path name），即服务器上资源的完整路径。
  • A 是主机名，B 是路径的一部分，D 是文件名，均不符合 “path name” 的完整定义，因此选 C。

# 第 25 题

题目：
Suppose a browser sends an HTTP request message with the If-modified-since: header. Suppose the object in a server has not changed since the last time a client retrieved the object. Then the server will send a response message with the status code:
A. 404 Not Found
B. 304 Not Modified
C. 200 OK
D. None of the above

参考答案：B

详细解析：

  • If-Modified-Since 是 HTTP 的缓存控制头：客户端向服务器询问 “资源在指定时间后是否被修改”。
    • 若资源未修改：服务器返回 304 Not Modified 状态码，不返回资源主体，让客户端使用本地缓存。
    • 若资源已修改：服务器返回 200 OK，并返回新的资源主体。
  • A（404）表示资源不存在，不符合题意，因此选 B。

# 第 26 题

题目：
What is the default persistent model in HTTP/1.1:
A. Nonpersistent HTTP
B. Persistent without pipelining
C. Persistent with pipelining
D. Nonpersistent with pipelining

参考答案：B

详细解析：

  • HTTP/1.1 的默认持久连接模式为 Persistent without pipelining（无流水线的持久连接）：
    • 持久连接：TCP 连接在多次请求 - 响应后才关闭，无需每次请求重新建立连接。
    • 流水线：客户端可以在收到前一个响应之前，连续发送多个请求，大幅提升性能。
  • A 是 HTTP/1.0 的默认模式；C 是 HTTP/1.1 的兼容模式，非默认；D 为错误组合（非持久连接无法使用流水线），因此选 B。

# 第 27 题

题目：
In FTP, the well-known port ____ is used for the control connection and the well-known port ______ for the data connection.
A. 20; 21
B. 21; 20
C. 21; 22
D. 22; 21

参考答案：B

详细解析：

  • FTP（文件传输协议）的双连接模型：
    • 控制连接（control connection）：使用知名端口 21，在整个会话期间持续保持，用于传输命令和响应。
    • 数据连接（data connection）：使用知名端口 20，仅在实际传输文件时建立，传输完成后立即关闭。
  • A 端口顺序颠倒；C、D 的 22 是 SSH 协议的端口，与 FTP 无关，因此选 B。

# 第 28 题

题目：
Which of the following services is not provided by DNS? ______
A. host aliasing
B. mail server aliasing
C. load distribution
D. IP address allocating

参考答案：D

详细解析：

  • DNS（域名系统）提供的核心服务：
    • A（主机别名）：为一个主机配置多个域名（如 www.baidu.com 和 baidu.com 指向同一主机）。
    • B（邮件服务器别名）：为邮件服务器配置别名（如 mail.baidu.com ），用于 SMTP 邮件传输。
    • C（负载均衡）：通过轮询 DNS，将请求分发到多个 IP 地址，实现服务器负载均衡。
    • D（IP 地址分配）：IP 地址分配由 DHCP（动态主机配置协议）或网络管理员完成，不是 DNS 的服务，因此选 D。

# 第 29 题

题目：
Given that the requested information is not available at any intermediate databases, a purely recursive DNS query from a requesting host would follow the path ______:
A. Root name server; TLD name server; local name server; authoritative name server.
B. authoritative name server; TLD name server; Root name server; local name server；
C. TLD name server; Root name server; local name server; authoritative name server;
D. local name server; Root name server; TLD name server; authoritative name server;

参考答案：D

详细解析：

  • ** 纯递归 DNS 查询（purely recursive DNS query）** 的完整流程：
    1. 本地 DNS 服务器（local name server）：接收客户端的查询请求，作为递归查询的起点。
    2. 根 DNS 服务器（Root name server）：本地服务器向根服务器查询，根服务器返回顶级域名（TLD）服务器的地址。
    3. 顶级域名 DNS 服务器（TLD name server）：本地服务器向 TLD 服务器查询，TLD 服务器返回权威 DNS 服务器的地址。
    4. 权威 DNS 服务器（authoritative name server）：本地服务器向权威服务器查询，获取最终的 IP 地址，返回给客户端。
  • 递归查询的核心是：本地服务器代替客户端完成所有查询步骤，客户端仅与本地服务器交互，因此路径为 本地→根→TLD→权威 ，选 D。

# 第 30 题

题目：
In ______ resolution, the server returns the IP address of the server that it thinks can resolve the query.
A. iterative
B. recursive
C. straight
D. None of the choices are correct

参考答案：A

详细解析：

  • 迭代查询（iterative resolution）的定义：DNS 服务器收到查询请求后，若自身没有缓存记录，会返回下一个能解析该请求的服务器的 IP 地址，让查询发起方（如本地 DNS 服务器）自行向下一个服务器发起查询。
  • 递归查询（recursive resolution）的定义：DNS 服务器收到请求后，会代替查询发起方完成所有查询步骤，最终返回最终的 IP 地址，而非中间服务器的地址。
  • C（straight）为错误术语，无此 DNS 查询类型，因此选 A。

# 第 31 题

题目：
下列关于 SMTP 协议的叙述中，正确的是____。
I、只支持传输 7 比特 ASCII 码内容
II、支持在邮件服务器之间发送邮件
III、支持从用户代理向邮件服务器发送邮件
IV、支持从邮件服务器向用户代理发送邮件
A. 仅 I、II 和 III
B. 仅 I、II 和 IV
C. 仅 I、III 和 IV
D. 仅 II、III 和 IV

参考答案：A

详细解析：

  • I 正确：原始 SMTP 协议仅支持 7 位 ASCII 码，无法直接传输二进制数据，需通过 MIME 扩展才能传输非 ASCII 内容。
  • II 正确：SMTP 的核心功能之一就是在邮件服务器之间转发邮件。
  • III 正确：SMTP 用于用户代理（邮件客户端）向发送邮件服务器提交邮件。
  • IV 错误：从邮件服务器向用户代理拉取邮件使用的是 POP3/IMAP 协议，而非 SMTP。
  • 综上，正确的是 I、II、III，选 A。

# 第 32 题

题目：
如果本地域名服务无缓存，当采用递归方法解析另一网络某主机域名时，用户主机和本地域名服务器发送的域名请求条数分别为____。
A. 1 条，1 条
B. 1 条，多条
C. 多条，1 条
D. 多条，多条

参考答案：A

详细解析：

  • 递归 DNS 查询的核心特点：
    • 用户主机：仅向本地 DNS 服务器发送 1 条查询请求，等待最终结果即可，无需自行迭代。
    • 本地 DNS 服务器：需代替用户主机，依次向根服务器、顶级域名服务器、权威服务器发送多条查询请求，直到获取最终 IP 地址。
  • 因此用户主机发 1 条，本地服务器发多条，选 B（X）。

# 第 33 题

题目：
FTP 客户和服务器间传递 FTP 命令时，使用的连接是____。
A. 建立在 TCP 之上的控制连接
B. 建立在 TCP 之上的数据连接
C. 建立在 UDP 之上的控制连接
D. 建立在 UDP 之上的数据连接

参考答案：A

详细解析：

  • FTP 采用双 TCP 连接模型：
    • 控制连接：基于 TCP，端口 21，在整个会话期间保持，专门用于传输 FTP 命令（如登录、列目录）和响应。
    • 数据连接：基于 TCP，端口 20，仅在文件传输时建立，传输完成后关闭。
  • 传递 FTP 命令使用的是 TCP 控制连接，选 A。

# 第 34 题

题目：
通过 POP3 协议接收邮件时，使用的传输层服务类型是____。
A. 无连接不可靠的数据传输服务
B. 无连接可靠的数据传输服务
C. 面向连接不可靠的数据传输服务
D. 面向连接可靠的数据传输服务

参考答案：D

详细解析：

  • POP3 是应用层协议，基于 TCP 传输层协议。
  • TCP 的核心特性是面向连接、可靠的数据传输服务，因此 POP3 使用的是面向连接可靠的传输服务，选 D。
  • 补充：无连接不可靠的是 UDP，无连接可靠、面向连接不可靠均不存在。

# 第 35 题

题目：
下列关于 FTP 协议的叙述中，错误的是____
A. 数据连接在每次数据传输完毕后就关闭
B. 控制连接在整个会话期间保持打开状态
C. 服务器与客户端的 TCP 20 端口建立数据连接
D. 客户端与服务器的 TCP 21 端口建立控制连接

参考答案：C

详细解析：

  • A 正确：FTP 的数据连接是临时的，每次文件传输完成后立即关闭。
  • B 正确：控制连接在整个 FTP 会话期间持续保持打开。
  • C 错误：FTP 数据连接中，服务器使用 20 端口作为源端口，客户端使用随机端口，并非 “服务器与客户端的 20 端口建立连接”（客户端端口不固定）。
  • D 正确：客户端主动连接服务器的 21 端口，建立控制连接。
  • 综上，错误的是 C。

# 第 36 题

题目：
以下关于非持久连接 HTTP 特点的描述中，错误的是 ()
A. HTTP 支持非持久连接与持久连接
B. HTTP/1.0 使用非持久连接，而 HTTP/1.1 的默认方式为持久连接
C. 非持久连接中对每次请求 / 响应都要建立一次 TCP 连接
D. 非持久连接读取一个包含 30 个图片对象的 web 页面需要打开和关闭 30 次 TCP 连接。

参考答案：D

详细解析：

  • A 正确：HTTP 协议同时支持非持久连接和持久连接两种模式。
  • B 正确：HTTP/1.0 默认非持久连接，HTTP/1.1 默认持久连接。
  • C 正确：非持久连接的核心是：每个请求 / 响应都需要单独建立、关闭一次 TCP 连接。
  • D 错误：一个包含 30 个图片的网页，除了 30 个图片对象，还需要 1 个基础 HTML 文件，因此总共需要 31 次 TCP 连接，而非 30 次。
  • 综上，错误的是 D。

# 第 37 题

题目：HTTP 是一个无状态的协议，然后 web 站点经常希望能够识别用户，这是需要用到 ()
A. WEB cache
B. Cookie
C. Conditional GET
D. Persistent connection

参考答案：B

详细解析：

  • **Cookie（会话 Cookie）** 是 HTTP 无状态问题的解决方案：服务器向客户端下发唯一标识，客户端后续请求携带该标识，服务器即可识别用户身份，实现登录状态、个性化推荐等功能。
  • A（Web 缓存）用于加速访问；C（Conditional GET）用于缓存验证；D（持久连接）用于减少 TCP 连接开销，均与用户识别无关，选 B。

# 第 38 题

题目：
下面的 TCP/IP 应用层协议中，可以使用传输层的无连接服务的是 ()。
A. FTP
B. SMTP
C. DNS
D. HTTP

参考答案：C

详细解析：

  • 传输层无连接服务对应 UDP 协议，面向连接对应 TCP 协议。
    • A（FTP）、B（SMTP）、D（HTTP）均基于 TCP，使用面向连接服务。
    • C（DNS）：域名解析默认使用 UDP（端口 53），属于无连接服务，仅在区域传输等场景使用 TCP，因此选 C。

# 第 39 题

题目：
不需转换就可由 SMTP 直接传输的内容是 ()
A. JPEG 图像
B. MPEG 视频
C. EXE 文件
D. ASCII 文本

参考答案：D

详细解析：

  • 原始 SMTP 协议仅支持 7 位 ASCII 文本的直接传输，二进制文件（如图片、视频、可执行文件）必须通过 MIME 编码（如 Base64）转换为 ASCII 格式后，才能通过 SMTP 传输。

# 第 40 题

题目：
下面关于 DNS 的说法中正确的是 ()
I、主域名服务器运行域名服务软件，有域名数据库
II、辅助域名服务器运行域名服务软件，但是没有域名数据库
III、一个域名有且只有一个主域名服务器

A. I 和 III
B. I、II、III
C. I
D. II 和 III

参考答案：C

详细解析：

  1. I 正确
    主域名服务器（Primary/ Master）本身存有完整的区域数据库，并且运行 DNS 服务。

  2. II 错误
    辅助域名服务器（Secondary/Slave）会从主服务器同步完整的区域数据库，不是 “没有数据库”。它只是不直接编辑库，而是通过区域传输（Zone Transfer）复制。

  3. III 错误
    “一个域名有且只有一个主域名服务器” 不符合实际 DNS 规范：

    • 一个区域（zone）可以配置多个主域名服务器（多主模式）
    • 很多公共 DNS 体系都支持多主
      所以 “有且只有一个” 是绝对化错误。

# 计算机网络・应用层 40 题核心知识点完整清单

（覆盖 C/S 与 P2P、Socket、HTTP、FTP、Email、DNS、传输层选择、Cookie 与缓存等全部考点）

# 一、应用架构与进程通信（第 13、1517 题）

# 1. C/S 架构 vs P2P 架构

  • C/S（客户端 / 服务器）
    • 服务器必须一直运行，客户端按需启动。
    • 服务器有固定 IP 和知名端口，客户端地址动态。
    • 客户端之间不直接通信，必须经过服务器。
  • P2P（对等架构）
    • 节点（peer）之间直接通信，无需中心服务器。
    • 节点地址通常动态，无统一知名地址。
    • 扩展性强，用户越多资源越丰富。

# 2. 应用架构与网络架构

  • 网络架构：固定不变，为应用提供服务。
  • 应用架构：由开发者设计，决定应用在端系统的组织方式。
  • 应用架构不由物理设备决定。

# 3. 进程通信与 Socket

  • 进程通过 Socket（套接字） 收发网络消息。
  • Socket 是 应用层与传输层之间的接口。
  • 应用层协议不定义 Socket 转换规则。

# 二、应用层服务需求与传输协议选择（第 4~5、38 题）

# 1. 应用分类

  • 丢包容忍（loss-tolerant）：实时音频 / 视频，少量丢包可接受。
  • 丢包敏感：邮件、文件传输、即时消息，必须完整可靠。

# 2. TCP vs UDP 适用场景

  • TCP（面向连接、可靠）：HTTP、FTP、SMTP、POP3、IMAP。
  • UDP（无连接、不可靠）：DNS 为主。

# 三、HTTP 协议（第 67、2227、36~37 题）

# 1. HTTP 基本特性

  • 基于 TCP，C/S 架构，无状态协议。
  • HTTP/1.0 默认：非持久连接。
  • HTTP/1.1 默认：持久连接 + 流水线（pipelining）。
  • Connection: close 表示使用非持久连接。

# 2. 持久连接与 RTT 计算

  • 非持久连接：每个对象一个 TCP 连接，2 RTT / 对象。
  • 持久连接（无流水线）：1 个 RTT 建连 + 每个对象 1 个 RTT。
  • 持久连接（带流水线）：可并行发送多个请求，效率最高。

# 3. HTTP 消息头与状态码

  • Date ：响应发送时间。
  • Last-Modified ：资源最后修改时间，必须 ≤ Date。
  • If-Modified-Since ：条件 GET，资源未修改返回 304 Not Modified。
  • 200 OK ：正常返回。
  • 404 Not Found ：资源不存在。

# 4. URL 结构

协议://主机名/路径名
例： http://www.djtu.edu.cn/rjxy/index.html

  • 主机名： www.djtu.edu.cn
  • 路径名： /rjxy/index.html

# 5. Cookie 与 Web 缓存

  • Cookie：解决 HTTP 无状态，用于识别用户、维持会话。
  • Web 缓存（Proxy）
    • 既是服务器又是客户端。
    • 减少响应时间、降低接入链路流量。
    • 通常由 ISP 部署。

# 四、FTP 协议（第 8~9、27、33、35 题）

# 1. 双 TCP 连接

  • 控制连接（端口 21）：全程保持，传输命令与响应。
  • 数据连接（端口 20）：传输文件，传完即关闭。

# 2. 核心特点

  • 带外控制（out-of-band）：控制与数据分连接传输。
  • 有状态协议：服务器维护用户登录、目录等状态。
  • 控制连接持久，数据连接临时。

# 五、电子邮件系统（第 1011、1314、19、31、34、39 题）

# 1. 邮件系统三大组件

  • 用户代理 UA
  • 邮件服务器
  • SMTP（传输协议）

# 2. 协议分工

  • SMTP：
    • 用户代理 → 邮件服务器
    • 邮件服务器 → 邮件服务器
    • 仅支持 7 位 ASCII，无状态协议。
  • POP3 / IMAP：用户代理从邮件服务器读取邮件。
  • HTTP：Web 邮箱使用。

# 3. MIME

  • 解决 SMTP 不能传二进制的问题。
  • 关键头部：
    • Content-Type：标识数据类型。
    • Content-Transfer-Encoding：编码方式（如 Base64）。

# 4. 传输层

POP3 基于 TCP，使用面向连接可靠传输服务。

# 六、DNS 域名系统（第 20、28~30、32、40 题）

# 1. DNS 三类核心服务器

  • 根 DNS 服务器
  • 顶级域 TLD 服务器（com、cn 等）
  • 权威 DNS 服务器（存储具体域名 IP）

本地 DNS 服务器是递归代理，不属于上述三层结构。

# 2. DNS 提供的服务

  • 主机别名、邮件服务器别名、负载分发。
  • 不负责 IP 地址分配（DHCP 负责）。

# 3. 查询方式

  • 递归查询：本地服务器代查到底，主机只发 1 条请求。
  • 迭代查询：服务器返回下一个服务器地址，不代查。

# 4. 递归查询路径

本地 DNS → 根 → TLD → 权威 DNS

# 5. 主域名服务器与辅助域名服务器

  • 主域名服务器：有完整域名数据库。
  • 辅助域名服务器：也有数据库（从主服务器同步）。
  • 一个域名可以有多个主服务器，并非 “只能一个”。

# 七、高频易混点总结

  1. 无状态协议：HTTP、SMTP、DNS
  2. 有状态协议：FTP、POP3、IMAP
  3. 只发请求不维持连接：DNS（UDP）
  4. 双连接分开传输：FTP（控制 21 / 数据 20）
  5. HTTP1.1 默认：持久连接 + 流水线
  6. Last-Modified 不可能晚于 Date
  7. SMTP 只能直接传 ASCII 文本
  8. Cookie 用于用户识别，缓存用于加速
  9. DNS 不分配 IP，只做域名解析
  10. 递归查询：主机发 1 条，本地 DNS 发多条

# 八、典型计算题与判断速记

  • 持久连接无流水线：1 个 RTT 建连 + 对象个数 × 1 RTT
  • 非持久连接：每个对象 2 RTT
  • 流量强度、RTF、传输效率等计算规则同之前网络层清单
  • DNS 递归路径固定：本地 → 根 → TLD → 权威
