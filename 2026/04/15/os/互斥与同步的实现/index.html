# 信号量

信号量的库文件为 semaphore.h，在实现并发中常用的定义和函数如下：


sem_t 定义一个信号量。
sem_init 初始化信号量，第二个参数为 0 表示进程内共享信号量，为 1 表示进程间共享信号量，一般不会让进程间共享资源而是进程内共享资源，第三个参数表示信号量的初始值，初始值为 1，表示允许一个线程进入临界区。
sem_wait 等待信号量，该语句后的临界区仅在信号量的值为 1 时才可执行，否则会阻塞。
sem_post 释放信号量，改变信号量的值为 1，并唤醒等待的线程。
sem_destroy 销毁信号量，一般在进程退出时调用。
一般这种问题我们需要先分析有多少类线程，多少个关系（互斥、同步），然后确定信号量个数，最后去写代码。
这里我个人的理解是：线程就是问题中的主体，就是要访问资源的对象，而信号量一般代表的就是资源可访问的剩余量。这里有时也可以把信号量理解为许可通知，用来唤醒对应线程。
注意：对于一些优先问题中，出现计数器的情况，需要使用信号量来维护计数器，防止计数器异常

# 互斥锁

互斥锁可以单独使用，也可以和条件变量一起使用。这里先说互斥锁单独使用。
互斥锁实现并发控制的函数如下：


函数说明：
pthread_mutex_t mutex ：定义一个互斥锁变量。
pthread_mutex_init() ：初始化互斥锁，参数 attr 为 NULL，表示使用默认属性。
pthread_mutex_lock() ：上锁，如果锁被其他线程占用，则等待。
pthread_mutex_unlock() ：解锁，其他线程可以获取锁，从而访问临界资源。
pthread_mutex_destroy() ：销毁互斥锁。
总之，单独使用互斥锁与使用信号量很相似，但是不同的是互斥锁只有两种状态持有锁和释放锁，而信号量可以初始为任意值，并且可以设置最大值，对应锁的等待队列长度。

互斥锁 + 条件变量是一种常用的同步机制，它可以实现多个线程之间共享数据，并确保数据访问的互斥性。以下是一个使用互斥锁和条件变量的例子：

注意：条件变量的改变必须是在锁保护下的，否则会出现数据竞争。

假设线程 A 在等 ready == 1 ：

可能出现这样的致命时序：

  1. 线程 A 刚判断完 ready == 0 ，准备进入 wait
  2. 操作系统突然切走 CPU，让线程 B 先跑
  3. 线程 B 立刻设置 ready = 1 并发出 signal
  4. 但此时线程 A 还没开始 wait，信号直接丢失
  5. 线程 A 被切回来，乖乖进入 wait 睡觉
  6. 结果：没人再唤醒它，线程 A 永远卡死
