# 进程与线程

进程的两个重要特点：

  1. 具有资源所有权：进程可以占据计算机资源，比如 CPU、内存、I/O 设备等。
  2. 调度 / 执行：进程执行时具有执行轨迹，不同进程的执行交替进行。进程具有执行态和被操作系统分配的优先级，可以被调度和分派。
    为了区别这两个特点，产生了线程的概念。
    进程 ：资源分配的基本单位，是运行中的程序
    线程 ：CPU 调度的基本单位，是进程内的执行路径
    可以用一个比喻来说，进程是一个房子，线程是这个房间里的人，房子内有很多财产资源，人可以在房子内完成各种不同的活动。

线程相较于进程的优点有：

  1. 在已创建的进程中创建新的线程比创建一个全新的进程快得多。
  2. 线程终止时间比进程短。
  3. 同一进程内线程的切换快于进程间的切换。
  4. 线程提高了不同执行程序之间的通信效率。进程之间的通信需借助内核，但线程由于与你行在同一个进程内，其存储空间和资源是共享的，所以就不需要内核介入，故线程之间的通信效率更高。

# 线程的功能

线程状态：与进程一样，线程主要状态有就绪态、运行态、阻塞态。挂起态对于线程来说没有意义，因为进程被挂起使，整个进程中的线程都被挂起，当进程被从硬盘中换出的时候，该进程的所有线程都一同被换出。
线程状态改变：

  1. 派生：进程被创建的同时会在该进程中派生出一个线程，后续进程中的线程也可以派生出新的线程。
  2. 阻塞：线程在等待事件时出现排队阻塞。
  3. 解除阻塞：线程被试间阻塞后被转移到就绪队列
  4. 结束：线程结束后释放其上下文和栈空间。
