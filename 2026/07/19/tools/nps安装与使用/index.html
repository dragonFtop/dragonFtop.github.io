# 简介

NPS（Ngrok Proxy Server）是一款轻量级、高性能的内网穿透代理工具，使用 Go 语言编写，无运行时依赖。它由服务端（nps）和客户端（npc）两部分组成，支持 TCP、UDP、HTTP、HTTPS、SOCKS5 等多种协议隧道。

# 核心特性

  • 全面的 Web 管理面板：所有客户端管理和隧道配置均可在浏览器中完成，告别命令行配置
  • 多协议支持：TCP / UDP 端口转发、HTTP / HTTPS 反向代理、SOCKS5 代理
  • P2P 穿透：两个内网客户端可直接通信，减轻服务端带宽压力
  • 多客户端配置：单个 npc 可同时管理多条隧道，支持配置文件批量管理
  • 域名解析：支持自定义域名绑定 HTTP 隧道，内置 DNS 代理
  • 流量加密与压缩：传输层采用加密通信，支持数据压缩，降低带宽消耗
  • 跨平台：Linux / Windows /macOS/ ARM 等全平台支持
  • 连接认证：客户端连接使用唯一 vkey 认证，支持 IP 白名单
  • 主机头映射：HTTP 隧道支持通过域名或请求 Host 头将流量路由到不同内网服务

# 关键参数说明

# bridge_port（客户端连接端口）

这是 nps 服务端和 npc 客户端之间的通道端口。部署时需确保：

  • VPS 安全组 / 防火墙放行此端口
  • 不与现有服务端口冲突
  • 建议使用非标准端口（如 18024 ）降低被扫描风险

# bridge_type（连接协议）

类型说明适用场景
tcp标准 TCP 连接通用场景，稳定可靠
kcp基于 UDP 的 KCP 协议高延迟 / 高丢包网络环境，牺牲部分带宽换取低延迟

# bridge_crypt 与 bridge_compress

  • bridge_crypt=true ：数据流加密，防止中间人窃听，建议始终开启
  • bridge_compress=true ：传输层压缩，在带宽受限时减小流量，但会增加 CPU 开销

# HTTP 代理域名绑定原理

当 nps 收到来自 http_proxy_port 的 HTTP 请求时：

  1. 检查请求的 Host 头
  2. 匹配隧道配置中绑定的域名
  3. 将请求转发到对应客户端的内网目标地址
  4. 将响应返回给请求方

这就是一个完整的反向代理流程，类似 Nginx 的 proxy_pass 。

# 5. 私密代理（Secret）

原理：与普通 TCP 隧道类似，但在连接时需要提供密钥字符串，否则拒绝连接。起到了简单的鉴权作用。

Web 面板创建时勾选「私密代理」并设置密钥字符串。

访问方式：

私密代理提供的是轻量级鉴权，不能替代专业的认证机制。不要将其作为唯一的安全措施。

# 6. P2P 穿透

原理：两个 NPC 客户端在 NPS 服务端的协助下建立直接连接，后续数据不经过服务端，减轻服务端带宽压力。

Web 面板操作：

  1. 创建两个客户端（客户端 A、客户端 B）
  2. 进入「P2P」菜单，创建 P2P 隧道
  3. 选择模式（TCP/UDP），分配两个客户端
  4. 客户端 A 访问本地映射端口即可连接客户端 B 的服务

P2P 穿透成功率取决于双方 NAT 类型。对称型 NAT 穿透困难，建议在可控网络环境下使用。

# 总结

NPS 是一款成熟的内网穿透工具，核心优势在于：

  1. 图形化管理：Web 面板完整覆盖所有隧道配置，运维门槛低
  2. 多协议支持：TCP/UDP/HTTP/HTTPS/SOCKS5 一应俱全
  3. 部署简单：单二进制文件，一条命令安装为系统服务
  4. 安全可靠：支持流量加密、IP 白名单、连接认证

适用场景包括远程办公、内网服务对外暴露、渗透测试代理链搭建等。与 frp 相比，NPS 管理体验更友好，但社区更新相对缓慢。两者功能大同小异，按个人操作习惯选择即可。
