# 前言

在大学期间，你可能已经通过 CTF、漏洞复现等接触过基础的网络安全技术，但对于真实的互联网资产是什么样的 —— 某个端口全球开放了多少、某个 CMS 的站点分布在哪里、某个漏洞的影响面有多大 ——FOFA 这类网络空间搜索引擎会给你一个直观的答案。本文将带你从零开始掌握 FOFA。

# 什么是 FOFA

FOFA 是国内白帽汇公司开发的网络空间资产测绘平台。它会持续扫描全球 IPv4/IPv6 地址空间的开放端口和服务，提取 Banner、HTTP 响应、SSL 证书等信息并建立索引，供安全研究人员检索。

一句话概括：Google 搜网页内容，FOFA 搜网络资产指纹。

特征说明

# API 调用注意事项

  1. 频率限制：普通会员 API 调用频率限制较低，短时间内大量调用会被暂时封禁（通常是 1 分钟恢复）
  2. 额度消耗：每条结果消耗 1 额度，不是每次 API 调用消耗 1 额度； size=100 表示每次最多返回 100 条数据，同时消耗 100 额度
  3. qbase64 编码：查询字符串必须先 Base64 编码再传参
  4. full 参数：设为 true 返回完整字段，否则仅返回 IP / 端口
  5. fields 参数：可指定返回字段以减少数据量： fields=ip,port,title,domain

# 如何编写 FOFA 产品识别规则

FOFA 的 app="" 查询之所以能识别产品，背后是社区贡献的识别规则。你也可以为尚未被识别的产品编写规则。

规则结构（YAML 格式）：

常用匹配类型：

类型说明示例
header匹配 HTTP 响应头中的字段Server / Set-Cookie / X-Powered-By
body匹配 HTML 响应体内容页面版权信息 / 特征 HTML 标签
title匹配 <title> 标签内容默认标题如 "phpMyAdmin"
icon_hash匹配 favicon murmurhash3_32上传 favicon 到 https://fofa.info/iconhash 获取
cert匹配 SSL 证书字段subject/issuer/dns_names
banner匹配非 HTTP 服务的 BannerSSH banner / FTP banner

编写好规则的优势：

提交流程：

  1. 准备识别规则的 YAML 文件
  2. 在 FOFA 上至少验证 10+ 个样本，确认规则准确（无误报、少漏报）
  3. 登录 FOFA -> FOFA Hub -> 上传规则
  4. FOFA 审核通过后，规则生效，全球用户都可以用 app="MyApp" 搜索

编写规则的关键原则：宁可漏报少报，不可误报。一旦规则误报率高导致错误识别，会影响所有用户的搜索结果准确性。

# 注意事项与法律边界

# 使用提示

  1. FOFA 仅展示互联网公开暴露的资产信息 —— 它扫描的是无需认证就能访问的开放端口和服务 Banner，不涉及任何入侵行为
  2. 搜索结果不代表漏洞 —— 一个资产在 FOFA 中出现只代表它互联网可达，不能证明它有漏洞。漏洞确认需要进一步测试
  3. 免费账户限制 —— 未付费用户每次搜索只能看到前 60 条数据，且不能导出。如果预算紧张，可以先用免费账户验证查询语句，确认查询准确后再付费导出
  4. 搜索频率 —— 短时间内大量搜索可能触发验证码或临时封禁 IP，写脚本时建议至少间隔 1 秒

# 法律与伦理边界

FOFA 是一个搜索引擎 —— 它索引的是互联网中公开可达的服务信息，本身是合法的。但在使用过程中有以下法律和伦理边界需要特别注意：

  1. 仅获取公开信息：FOFA 只提供公开可访问的服务信息，不要在 FOFA 搜索的基础上进行未授权的渗透测试

  2. 禁止利用漏洞：通过 FOFA 发现了某个服务的已知漏洞，未经授权对其进行利用是违法的

  3. 负责任披露：如果你在搜索中发现明显的未授权数据泄露（如暴露的数据库），建议通过合法途径联系资产所属单位，而不是：

    • 私自下载或传播数据
    • 在社交媒体或圈子里截图散播
    • 以任何形式牟利

  4. 合理使用：高频访问 FOFA 或大量调用 API 建议遵守其使用条款

  5. 取证来源标注：如果使用 FOFA 的数据生成报告或进行学术研究，建议注明数据来源

# 学生时代的安全研究

作为在校学生，在接触 FOFA 这类工具时，建议：

  • 找一个合法的靶场环境来练习，而不是搜索出真实目标的漏洞就去尝试
  • 参与 CTF 和学校授权的渗透测试，这是锻炼技术、积累经验的最正统途径
  • 关注学校信息化部门的纳新或合作机会 —— 部分大学的网络中心会组建学生安全团队，可以利用 FOFA 帮学校做资产测绘，这也是简历中的亮点
  • 用 FOFA 做漏洞影响面评估分析（如统计某个 CMS 的分布），写成技术文章或作业报告，这是完全合法且很有价值的研究方向

# FOFA 资产画像

资产画像（Asset Profile）是 FOFA 会员功能，可对指定目标（IP 或域名）生成一份完整的资产分析报告，相当于一键汇总出一个目标的完整暴露面。

# 使用方式

在 FOFA 搜索框中直接输入 IP 地址或域名，在搜索结果页上方点击「资产画像」标签。
