# title: Multimodal
categores: AI

# 什么是多模态？

# 什么是模态？

模态，理解为信息的不同表达形式：


单模态：只处理一种形式的信息。
多模态：同时处理多种信息或者说是把不同信息进行融合理解
简单的多模态例子：

  • 给模型一个图片和文本【猫的图片 +“这是一张动物图片，请你指出这是什么动物。”】
  • 给模型一段音频和文本【摇滚音乐 +“请你鉴别这首音乐的风格”】

# 模型如何理解模态？

# 模型如何理解单个模态？- 特征编码 - 向量化

  1. 对于文本模态：模型直接使用 tokenizer，将文本 token 化，转化为 word embedding。
  2. 对于图像模态：通过 ViT/CNN 方式，将图像转化为 patch embedding，
  3. 对于音频模态：通过 spectrogram 和 encoder 进行编码转换为 audio embedding

# 模型如何理解多模态？- 对齐

在上述理解单模态信息的基础上（向量化），使用 对齐（Alignment） 的方式，将不同模态的信息进行结合【使模型能将猫的图片和猫的汉字联系起来】。
常见使用两种方法：

  1. 共享向量空间：通过投影层，使用投影的方法将不同模态的编码向量投影到同一个维度，然后将 “匹配的图文靠近”，比如图片猫≈文本猫，这二者的向量距离减小，图片狗≠文本猫，二者的向量距离增大，通过这种方式将语义上相近的不同模态拉近聚拢，从而完成向量上的 “统一”。
  2. 相关性学习：在前者类似的基础上，更加关注图片集合和文本集合的整体性关系。让所有猫的图片和所有猫的文字，这两个集合的整体规律保持一致：整体分布、中心位置、散开程度（协方差）上保持一致。好处是这样可以让模型对于新的猫图片或文字有更好的理解和融合。

关键一步： 融合（Fusion Module） ，将不同模态的向量信息进行融合，以下为主流方式：

  • A. 交叉注意力机制（Cross-Attention）：让一个模态去查询另一个模态的信息。例如：文本问：图像里是什么动物？模型去看图像信息。文本 -> 图像。
  • B. 自注意力机制（Self-Attention）：将图片 token 和文本 token 拼接起来，让模型自动学习关系。
  • C. 适配器（Adapter）：在每个模态一个适配器，插入可学习模块，通过适配器和可学习模块控制不同模态之间的信息交互强度。

# 多模态安全问题

# 多模态安全目标

  • 模型误导：Adversarial/Jailbreak（对抗 / 越狱）
  • 数据被污染：Training/Federated poisoning（训练 / 联邦 投毒）
  • 隐私泄露：Corss-modal leakage（跨模态泄露）

# 多模态对抗攻击（Adversarial Attack）

在图片中加入对抗扰动，肉眼无法分辨，但模型会被欺骗。
图像模态攻击实例：

  • 在猫的图片中加入一点像素噪点
  • 肉眼观察：正常猫
  • 模型视角：武器 / 不安全内容

patch 攻击

  • 人类视角：正常的 “贴纸”
  • 模型视角：被 “贴纸” 误导，将类别判断错误

# 多模态提示词注入（Prompt Injection）

通过构造提示词将自己的命令注入给模型，让模型按照宪法做出行为。
实例：

  • 用户输入一个图片，但是图片中有一段隐藏的文字。
  • 多模态模型接下来通过 OCR 识别途中文字
  • 该段文字被当作真实指令
  • 被诱导执行，输出敏感数据

# 跨模态信息泄露

模型可能从一个弄模态泄露另一个模态的信息。
