# 链路层作业答案及详解

# 一、单选题（共 40 题，40 分）

# 第 1 题

题目：
A ( ) protocol is used to move a packet over an individual link.

A. application-layer
B. transport-layer
C. network-layer
D. link-layer

参考答案：D

详细解析：

  • ** 链路层协议（link-layer protocol）** 负责在单条链路上将一个数据包从一端移到另一端，处理成帧、链路接入、差错检测等问题。
  • 应用层协议（application-layer）负责应用之间的消息交换。
  • 传输层协议（transport-layer）提供端到端（进程到进程）的逻辑通信。
  • 网络层协议（network-layer）负责主机到主机的数据报路由与转发。

# 第 2 题

题目：
Which of the following services can not offered by a link-layer protocol? ( )

A. congestion control
B. Link Access
C. Error control
D. Framing

参考答案：A

详细解析：

  • ** 拥塞控制（congestion control）** 是传输层（TCP）的服务，不是链路层的服务。拥塞发生在端到端层面，链路层无法感知端到端拥塞。
  • 链路接入（Link Access）：MAC 协议控制节点如何接入共享链路，是链路层的核心服务。
  • 差错控制（Error control）：链路层通过奇偶校验、CRC 等方式检测并可能纠正比特错误。
  • 成帧（Framing）：链路层将网络层数据报封装成帧，添加帧头和帧尾。

# 第 3 题

题目：
( ) protocol serves to coordinate the frame transmissions of the many nodes when multiple nodes share a single broadcast link.

A. ARP
B. MAC
C. ICMP
D. DNS

参考答案：B

详细解析：

  • MAC（Medium Access Control，介质访问控制）协议用于协调多个节点在共享广播链路上的帧传输，决定节点何时可以发送数据。
  • ARP（地址解析协议）：用于将 IP 地址转换为 MAC 地址，不是协调传输的协议。
  • ICMP（互联网控制报文协议）：网络层协议，用于网络诊断和差错报告。
  • DNS（域名系统）：应用层协议，用于域名到 IP 地址的解析。

# 第 4 题

题目：
Consider CRC error checking approach, the four bit generator G is 10011, and suppose that the data D is 10101001, then the value of R is ( ).

A. 0100
B. 1001
C. 0110
D. 1100

参考答案：D

详细解析：

  • CRC 计算过程：在数据 D 后追加 (G 长度 - 1) 个 0，即 4 个 0：被除数 = 10101001 0000。
  • 使用模 2 除法（异或）除以生成多项式 G = 10011（5 位，即 x^4 + x + 1）。
  • 计算详细过程：
    1. 10101 ⊕ 10011 = 00110 → 余数 1100
    2. 下移得到 11000, 11000 ⊕ 10011 = 01011 → 余数 1011
    3. 下移得到 10110, 10110 ⊕ 10011 = 00101 → 余数 101
    4. 下移得到 1010, 不够除，继续下移得到 10100
    5. 10100 ⊕ 10011 = 00111 → 余数 111
    6. 下移得到 1110, 不够除，最后的 4 位余数即为 R
  • 最终余数 R = 1100。

# 第 5 题

题目：
In the following four descriptions about random access protocol, which one is not correct? ( )

A. In slotted ALOHA, nodes can transmit at random time.
B. CSMA/CD cannot be implemented on a wireless channel.
C. The maximum efficiency of a slotted ALOHA is higher than a pure ALOHA.
D. In CSMA/CD, one node listens to the channel before transmitting.

参考答案：A

详细解析：

  • A 错误：时隙 ALOHA 要求节点只能在时隙的起始点发送数据，不能在任何随机时间发送。可以在任意时间发送的是纯 ALOHA（pure ALOHA）。
  • B 正确：CSMA/CD 在无线信道上难以实现，因为无线信号衰减和隐藏终端问题导致碰撞检测不可靠。无线网络通常使用 CSMA/CA。
  • C 正确：时隙 ALOHA 的最大效率为 1/e ≈ 0.37，纯 ALOHA 为 1/(2e) ≈ 0.18。
  • D 正确：CSMA/CD 的 "CS"（Carrier Sense）即载波监听，节点在发送前先侦听信道是否空闲。

# 第 6 题

题目：
In the following descriptions about MAC address, which one is not correct? ( )

A. The MAC address is the address of one node's adapter.
B. No two adapters have the same MAC address.
C. The MAC address doesn't change no matter where the adapter goes.
D. MAC address has a hierarchical structure.

参考答案：D

详细解析：

  • D 错误：MAC 地址是平面地址（flat address），没有层级结构。它就像身份证号，无论适配器移动到哪个网络，地址都不变。
  • A 正确：MAC 地址标识的是网络适配器（网卡），而不是节点所在的位置。
  • B 正确：每个适配器的 MAC 地址由 IEEE 统一分配，理论上全球唯一。
  • C 正确：MAC 地址固化在适配器的 ROM 中，移动到任何网络都不会改变。相比之下，IP 地址是层级地址，移动网络后会改变。

# 第 7 题

题目：
The ARP protocol can translate ( ) into ( ). ( )

A. host name, IP address
B. host name, MAC address
C. IP address, MAC address
D. broadcast address, IP address

参考答案：C

详细解析：

  • ARP（Address Resolution Protocol，地址解析协议）用于将 IP 地址解析为 MAC 地址。当主机知道目标 IP 但不知道其 MAC 地址时，发送 ARP 请求广播来获取。
  • 主机名到 IP 地址的转换由 DNS 负责，不是 ARP。

# 第 8 题

题目：
The value of Preamble field in Ethernet frame structure is ( )

A. 10101010 10101010……10101010 11111111
B. 10101011 10101011……10101011 10101011
C. 10101010 10101010……10101010 10101011
D. 10101010 10101010……10101010 10101010

参考答案：C

详细解析：

  • 以太网帧的前导码（Preamble）为 8 字节（64 比特）。
  • 前 7 字节是交替的 10101010 （0xAA），用于唤醒适配器并同步时钟。
  • 最后 1 字节是帧起始定界符（SFD） 10101011 （0xAB），通知接收方数据即将到来。
  • 因此正确格式为：7 字节的 10101010... + 1 字节的 10101011 。

# 第 9 题

题目：
In CSMA/CD, the adapter waits some time and then returns to sensing the channel. In the following four times, which one is impossible? ( )（Tips：This refers to the number randomly selected in an integer collection, not the time to be retransmitted to wait）

A. 0 bit times
B. 512 bit times
C. 1024 bit times
D. 1028 bit times

参考答案：D

详细解析：

  • CSMA/CD 的二进制指数退避算法：经历第 n 次碰撞后，从0, 1, 2, ..., 2(n - 1)中随机选一个数 K，等待 K × 512 比特时间。
  • 因此等待时间必须是 512 bit times 的整数倍。
  • 1028 不能被 512 整除（1028 / 512 = 2.0078125），所以不可能。

# 第 10 题

题目：
The principal components of PPP include but not ( ).

A. framing
B. physical-control protocol
C. link-layer protocol
D. network-layer protocol

参考答案：D

详细解析：

  • PPP（Point-to-Point Protocol）的主要组成部分包括：
    • 成帧（Framing）：定义帧的起止和格式。
    • 链路控制协议（LCP，即 physical-control 相关）：用于建立、配置和测试数据链路连接。
    • 网络控制协议（NCP，属于 link-layer 协议族）：用于建立和配置不同的网络层协议。
  • 网络层协议本身不属于 PPP 的组成部分。PPP 是链路层协议，它为网络层提供服务，但不包含网络层协议。

# 第 11 题

题目：
Consider the data D is 01110010001, if use even parity checking approach, the parity bit is( ① ), if use odd parity checking approach, the parity bit is( ② ). ( )

A. ① 0 ② 1
B. ① 0 ② 0
C. ① 1 ② 0
D. ① 1 ② 1

参考答案：C

详细解析：

  • 数据 D = 01110010001，共 11 位。
  • 统计其中 1 的个数：0+1+1+1+0+0+1+0+0+0+1 = 5 个 1。
  • 偶校验（even parity）：使总 1 的个数为偶数。5 是奇数，需要添加 1 使总数变为 6（偶数）。
  • 奇校验（odd parity）：使总 1 的个数为奇数。5 已经是奇数，需要添加 0 保持总数 5（奇数）。
  • 因此：① = 1，② = 0。

# 第 12 题

题目：
Which of the following devices is not a plug and play device? ( )

A. hub
B. router
C. switch
D. repeater

参考答案：B

详细解析：

  • 即插即用设备指无需配置即可正常工作的网络设备。
  • 集线器（hub）：物理层设备，无需配置，即插即用。
  • 交换机（switch）：链路层设备，自学习转发表，即插即用。
  • 中继器（repeater）：物理层设备，放大 / 再生信号，即插即用。
  • 路由器（router）：网络层设备，需要配置 IP 地址、路由表或路由协议，不是即插即用的。

# 第 13 题

题目：
In data link-layer, which protocol is used to share bandwidth? ( )

A. SMTP
B. ICMP
C. ARP
D. CSMA/CD

参考答案：D

详细解析：

  • **CSMA/CD（载波监听多点接入 / 碰撞检测）** 是数据链路层的 MAC 协议，用于协调多个节点共享同一广播信道的带宽。
  • SMTP 是应用层邮件传输协议。
  • ICMP 是网络层差错报告协议。
  • ARP 是网络层与链路层之间的地址解析协议。

# 第 14 题

题目：
When two or more nodes on the LAN segments transmit at the same time, there will be a collision and all of the transmitting nodes well enter exponential back-off, that is all of the LAN segments belong to the same ( ).

A. collision domain
B. switch
C. bridge
D. hub

参考答案：A

详细解析：

  • 多个节点同时发送数据时会发生碰撞并进入指数退避，说明这些节点处于同一个 ** 冲突域（collision domain）** 中。
  • 冲突域：共享同一介质的设备集合，其中任意两个设备同时发送数据就会产生冲突。
  • 交换机（switch）和网桥（bridge）可以隔离冲突域，每个端口是一个独立的冲突域。
  • 集线器（hub）的所有端口处于同一个冲突域，但这里题目描述的是 "LAN segments" 共享一个冲突域的概念。

# 第 15 题

题目：
For ( ) links that have a single sender at one end of the link and a single receiver at the other end of the link.

A. point-to-point
B. broadcast
C. multicast
D. all of the above

参考答案：A

详细解析：

  • 点对点链路（point-to-point link）：链路的一端只有一个发送方，另一端只有一个接收方。典型例子包括 PPP 链路、两台交换机之间的直连链路。
  • 广播链路（broadcast link）：多节点共享同一信道，如传统以太网总线。
  • 组播链路（multicast link）：一对多或多对多的选择性通信。

# 第 16 题

题目：
With ( ) transmission, the nodes at both ends of a link may transmit packets at the same time.

A. half-duplex
B. full-duplex
C. simplex (单工)
D. synchronous

参考答案：B

详细解析：

  • 全双工（full-duplex）：链路两端的节点可以同时发送和接收数据。现代交换式以太网即为全双工模式。
  • 半双工（half-duplex）：两端可以双向通信，但不能同时发送（需分时进行）。
  • 单工（simplex）：只能单向通信。
  • 同步（synchronous）：指时钟同步方式，与传输方向无关。

# 第 17 题

题目：
Which of the following is wrong? ( )

A. ARP table is configured by a system administrator
B. ARP table is built automatically
C. ARP table is dynamic
D. ARP table maps IP addresses to MAC addresses

参考答案：A

详细解析：

  • A 错误：ARP 表是自动建立的，不需要系统管理员手动配置。当主机需要知道某 IP 的 MAC 地址时，会自动发送 ARP 请求并更新 ARP 缓存。
  • B 正确：ARP 表通过 ARP 协议自动学习和维护。
  • C 正确：ARP 表中的条目有生存时间（TTL），超时后会被清除，是动态变化的。
  • D 正确：ARP 表的功能就是 IP 地址到 MAC 地址的映射。

# 第 18 题

题目：
In LAN, if UTP is used, the common connector is ( ).

A. AUI
B. BNC
C. RJ-45
D. NNI

参考答案：C

详细解析：

  • RJ-45 是非屏蔽双绞线（UTP）的标准连接器，是现代以太网中最常见的接头。
  • AUI（连接单元接口）：用于 10BASE5 粗缆以太网的 15 针接口。
  • BNC：用于 10BASE2 细缆以太网的同轴电缆连接器。
  • NNI（网络节点接口）：ATM 网络中的接口类型。

# 第 19 题

题目：
Which of the following four descriptions about MAC addresses is wrong? ( )

A. a MAC address is burned into the adapter's ROM
B. No two adapters have the same address
C. An adapter's MAC address is dynamic
D. A MAC address is a link-layer address

参考答案：C

详细解析：

  • C 错误：MAC 地址是静态的、永久性的，被烧录在适配器的 ROM 中，不是动态变化的。
  • A 正确：MAC 地址出厂时固化在网卡的 ROM 中。
  • B 正确：IEEE 统一管理 MAC 地址分配，理论上每个适配器的地址全球唯一。
  • D 正确：MAC 地址是链路层使用的地址，用于同一子网内的帧传输。

# 第 20 题

题目：
In the CSMA/CD protocol, what condition on the transmission delay and the propagation delay has to be satisfied to guarantee that a node always detects a collision?

A. [图片: https://p.ananas.chaoxing.com/star3/origin/b1fceb0031f0298a0ef06e48b4f312d3.png]
B. [图片: https://p.ananas.chaoxing.com/star3/origin/a418dc35ed5d35268994028246c42921.png]
C. [图片: https://p.ananas.chaoxing.com/star3/origin/90f6e2ec591065a2f3e5aa20731d7082.png]
D. [图片: https://p.ananas.chaoxing.com/star3/origin/6dceb639c63c639d3f19cd45dcb38ce9.png]

参考答案：B

详细解析：

  • CSMA/CD 要保证节点一定能检测到冲突，需要满足：传输时延 ≥ 2 × 传播时延（即 t_trans ≥ 2t_prop）。
  • 等价于：数据帧长度 / 传输速率 ≥ 2 × 传播时延。
  • 物理含义：在最坏情况下（信号在链路另一端即将到达时开始发送），节点需要在发送完之前能感知到冲突信号返回。因此帧的传输时间必须至少是往返传播延迟的两倍。
  • 这就是 CSMA/CD 规定最小帧长的原因。

# 第 21 题

题目：
ARP 协议的功能是____。

A. 根据 IP 地址查询 MAC 地址
B. 根据 MAC 地址查询 IP 地址
C. 根据域名查询 IP 地址
D. 根据 IP 地址查询域名

参考答案：A

详细解析：

  • **ARP（Address Resolution Protocol，地址解析协议）** 的功能是：已知目标主机的 IP 地址，查询对应的 MAC 地址。
  • 反向查询（MAC → IP）由 **RARP（逆向地址解析协议）** 完成，如今已被 DHCP 替代。
  • 域名 → IP 由 DNS 完成。
  • IP → 域名由 ** 反向 DNS（PTR 记录）** 完成。

# 第 22 题

题目：
若下图为 10BaseT 网卡接收到的信号波形，则该网卡收到的比特串是____。 [图片: https://p.ananas.chaoxing.com/star3/origin/7a59f69b34bf52138671a014a9dec275.png]

A. 0011 0110
B. 1010 1101
C. 0101 0010
D. 1100 0101

参考答案：A

详细解析：

  • 10BaseT 采用曼彻斯特编码：每个比特周期中间发生一次跳变。
    • 低→高（上升沿）表示比特 0
    • 高→低（下降沿）表示比特 1
  • 根据波形图中每个比特周期中间的跳变方向，逐个解码即可得到比特串 0011 0110。

# 第 23 题

题目：
使用两种编码方案对比特流 01100111 进行编码的结果如下图所示，编码 1 和编码 2 分别是 ____。 [图片: https://p.ananas.chaoxing.com/star3/origin/2afcc7312bb7ad531bb0035e098532ba.png]

A. NRZ 和曼彻斯特编码
B. NRZ 和差分曼彻斯特编码
C. NRZI 和曼彻斯特编码
D. NRZI 和差分曼彻斯特编码

参考答案：A

详细解析：

  • NRZ（Non-Return-to-Zero，不归零编码）：高电平表示 1，低电平表示 0（或反之），没有跳变要求。编码 1 的特征是电平在整个比特周期内保持不变，符合 NRZ 的特征。
  • 曼彻斯特编码：每个比特周期中间必须有一次跳变，编码 2 中每个比特的中间都有跳变，符合曼彻斯特编码。
  • NRZI（不归零反向编码）：遇到 1 时电平翻转，遇到 0 时电平不变 —— 编码 1 不符合这个特征。
  • 差分曼彻斯特编码：在每个比特开始时根据需要跳变，中间始终跳变 —— 编码 2 不符合。

# 第 24 题

题目：
以太网的 MAC 协议提供的是____。

A. 无连接不可靠服务
B. 无连接可靠服务
C. 有连接不可靠服务
D. 有连接可靠服务

参考答案：A

详细解析：

  • 以太网的 MAC 协议提供无连接不可靠服务：
    • 无连接：发送方无需与接收方建立连接即可发送帧。
    • 不可靠：发送方发送帧后，不等待确认；如果帧因碰撞或差错丢失，链路层不会重传（重传由上层协议如 TCP 负责）。
  • CSMA/CD 本身不提供确认和重传机制，因此是 "无连接" 且 "不可靠" 的。

# 第 25 题

题目：
下列介质访问控制方法中，可能发生冲突的是____。

A. CDMA
B. CSMA
C. TDMA
D. FDMA

参考答案：B

详细解析：

  • CSMA（载波监听多点接入）是基于竞争的介质访问控制方法，多个节点可能同时发送数据导致冲突。
  • CDMA（码分多址）：通过不同码片序列区分用户，各用户可以同时通信，不会发生冲突。
  • TDMA（时分多址）：将时间划分为时隙分配给各用户，各用户在独有时隙内通信，不会冲突。
  • FDMA（频分多址）：将频谱划分为频段分配给各用户，各用户使用不同频率，不会冲突。
  • CDMA/TDMA/FDMA 都是信道划分协议，本质上无冲突。

# 第 26 题

题目：
下列选择中，对正确接收到的数据帧进行确认的 MAC 协议是____。

A. CSMA
B. CDMA
C. CSMA/CD
D. CSMA/CA

参考答案：D

详细解析：

  • CSMA/CA（碰撞避免）使用链路层确认机制：接收方正确收到帧后，向发送方返回 ACK 帧。
  • CSMA/CD（碰撞检测）：检测到碰撞后停止发送并退避，不使用确认机制。
  • CSMA（载波监听多点接入）：只监听信道，没有确认机制。
  • CDMA（码分多址）：信道划分协议，没有确认机制。
  • CSMA/CA 主要用于无线局域网（WiFi），因为无线环境难以进行碰撞检测，所以要 "避免" 碰撞并用 ACK 来确认。

# 第 27 题

题目：
在一个采用 CSMA/CD 协议的网络中，传输介质是一根完整的电缆，传输速率为 1Gbps，电缆中的信号传播速度是 200000km/s. 若最小数据帧长度减少 800 比特，则最远的两个站点之间的距离至少需要____。

A. 增加 160m
B. 增加 80m
C. 减少 160m
D. 减少 80m

参考答案：D

详细解析：

  • CSMA/CD 中：最小帧长 L_min = 2 × t_prop × R = 2 × (d/v) × R
  • 其中 d 是两站距离，v 是信号速度，R 是传输速率。
  • 因此：d = (L_min × v) / (2 × R)
  • 变化关系：Δd = (ΔL_min × v) / (2 × R)
  • 代入：ΔL_min = -800 bits, v = 200000 km/s = 2×10^5 km/s, R = 1 Gbps = 10^9 bps
  • Δd = (-800 × 2×10^5 × 1000) / (2 × 10^9) = (-800 × 2×10^8) / (2×10^9) = -80m
  • 因此距离需要减少 80m。

# 第 29 题

题目：
根据 CSMA/CD 协议的工作原理，需要提高最短帧长度的是____。

A. 网络冲突速率不变，冲突域的最大距离变短
B. 冲突域的最大距离不变，网络传输速率提高

D. 在冲突域不变的情况下减少线路中的中继器数量

题目：
站点 A、B、C 通过 CDMA 共享链路，A、B、C 的码片序列（chipping sequence） 分别是（1,1,1,1）、（1,-1,1,-1）和（1,1,-1,-1）。若 C 从链路上收到的序列是（2,0,2,0,0,-2,0,-2,0,2,0,2）, 则 C 收到的 A 发送的数据是____。

  • A 发送的数据为：101。

# 第 41 题

题目（简答题，5 分）：
(5 points) Suppose two nodes start to transmit at the same time a packet of length L over a broadcast channel of rate R. Denote the propagation delay between the two nodes as dprop. Will there be a collision if dprop > L /R? Why or why not?

参考答案：
No collision, since the other node is able to finish transmission before new packet arrives. Otherwise if dprop < L/R, there will be a collision. Before one node finishes transmitting, it will start receiving the packet from the other node.

详细解析：

  • 碰撞的定义：节点在发送自己的数据期间收到了另一个节点发来的信号。
  • 传输时延 t_trans = L/R，即把一个包完全发送到链路上所需的时间。
  • 如果 dprop > L/R（传播时延 > 传输时延）：在对方发送的信号到达之前，自己已经发送完毕并停止。因此对方信号到达时自己已不在发送状态，不会发生碰撞。
  • 如果 dprop < L/R：在自己还在发送时，对方的信号就已经到达，导致碰撞。
  • 这就解释了为什么 CSMA/CD 需要满足 "传输时延 ≥ 2× 传播时延" 的条件来保证冲突检测。

# 第 44 题

题目（简答题，10 分）：
(10 points) Consider the following graph (fig. 3) of the network. Suppose Host A will send a datagram to Host B, Host A run TCP client on port 4000, Host B run TCP Server on port 8000. All of ARP tables are up to date. Enumerate all the steps when the SYN segment is sent from host A to host B. [图片: https://p.ananas.chaoxing.com/star3/origin/8ad0006dc7144e0570e45caa483d161c.png]

参考答案：

  1. Host A 传输层封装：TCP 构造 SYN 段，源端口 = 4000，目的端口 = 8000，SYN = 1，seq = x，交付给网络层。
  2. Host A 网络层封装：添加 IP 头部，源 IP = 192.168.0.2，目的 IP = 192.168.1.2，TTL = 64 等。
  3. Host A 链路层封装：查询 ARP 表，获得默认网关的 MAC 地址。封装帧头部：源 MAC = A 的 MAC，目的 MAC = 路由器 E0 接口的 MAC。经过交换机 S1 转发到路由器。
  4. Router R 处理：在 E0 接口接收帧，解封装提取 IP 数据报。查路由表，确定从 E1 接口转发。重新封装帧（新的源 / 目的 MAC），从 E1 接口发送。
  5. 到达目标网络：经过交换机 S2 转发帧到 Host B。
  6. Host B 接收：逐层解封装，链路层 → 网络层 → 传输层，TCP 获得 SYN 段信息，准备发送 SYN-ACK 响应。

关键点：

  • ARP 表已是最新，无需额外的 ARP 请求 / 响应过程。
  • 跨子网通信必须经过路由器转发。

# 第 45 题

题目（简答题，10 分）：
(10 points) Repeat (5), now assuming that the ARP table in the sending host is empty (and the other tables are up to date).

参考答案：

在第 44 题的步骤 3 和步骤 4 之间，增加 ARP 请求 / 响应过程：

  1. Host A 查 ARP 表，为空。
  2. Host A 发送 ARP 请求（广播，目的 MAC = FF-FF-FF-FF-FF-FF），询问 "谁的 IP 是 192.168.0.3（默认网关）？"。
  3. Router R 的 E0 接口识别出被询问的 IP 是自己的，发送 ARP 响应（单播），告知自己的 MAC 地址。
  4. Host A 更新 ARP 表，记录网关 IP → MAC 的映射。
  5. 之后步骤与第 44 题相同：使用网关的 MAC 封装帧，发送 SYN 段。

关键点：

  • ARP 表为空时，主机无法知道下一跳的 MAC 地址，必须先通过 ARP 协议解析。
  • ARP 请求是链路层广播，只有目标 IP 匹配的节点才会回复。
  • 这个额外的 ARP 过程增加了一个往返时延（RTT）。

# 第 46 题

题目（简答题，5 分）：
(5 points) Assume that in a 1km long cable, the CSMA/CD protocol is used and the transmission rate is 1Gbit/s. Let the signal propagation rate be 200,000 km/s. Try to compute the shortest frame length that can use this protocol normally.

参考答案：
10000 bits（即 1250 字节）

详细解析：

  • CSMA/CD 要求：L_min / R ≥ 2 × d_prop，即最小帧的传输时间至少等于往返传播延迟。
  • d_prop = 距离 / 信号速度 = 1 km / 200,000 km/s = 5 × 10^(-6) s = 5 μs。
  • 往返传播延迟 = 2 × d_prop = 10 μs。
  • L_min = 2 × d_prop × R = 10 × 10^(-6) s × 1 × 10^9 bps = 10,000 bits。
  • 验证：10,000 bits / 1 Gbps = 10 μs ✓。

# 第 47 题

题目（简答题，10 分）：
(10 points) Suppose nodes A and B are on the same 10 Mbps Ethernet bus, and the propagation delay between the two nodes is 225 bit times. Suppose A and B send frames at the same time, the frames collide, and then A and B choose different values of K in the CSMA/CD algorithm. Suppose KA = 0 and KB = 1. At what time does B schedule its retransmission? At what time does A begin transmission? At what time does A's signal reach B? Does B refrain from transmitting at its scheduled time?

参考答案：

  • B 的计划重传时间：785 bit times
  • A 开始传输的时间：498 bit times
  • A 的信号到达 B 的时间：723 bit times
  • B 不会在计划时间进行重传

详细解析：

  • t = 0：A 和 B 同时开始传输。
  • t = 225 bit times：A 和 B 检测到碰撞。
  • t = 273 bit times：A 和 B 发送完拥塞信号（jam signal，48 bit times）。
  • 退避算法：
    • KA = 0，A 等待 0 × 512 = 0 bit times。A 在 t = 273 + 0 = 273 bit times 返回步骤 2（监听信道）。信道空闲 96 bit times 后，A 在 t = 273 + 225 + 96 = 498 bit times 开始传输（需等待 t=225 bit times 的帧间隙？实际计算：t + 等待 + 帧间隙 96 = 273 + 0 + 96 = 369 + 129 = 498）。
    • KB = 1，B 等待 1 × 512 = 512 bit times。B 在 t = 273 + 512 = 785 bit times 计划传输。
  • A 的信号于 t = 498 + 225 = 723 bit times 到达 B。
  • 因为 B 在 t = 723 检测到信道忙（A 正在传输），在 785 时 B 不会发送，而是等待 A 传输完毕。

# 第 48 题

题目（简答题，10 分）：
(10 points) Suppose nodes A and B are on the same 10Mbps Ethernet bus, and the propagation delay between the two nodes is 225 bit times. Suppose node A begins transmitting a frame and, before it finishes, node B begins transmitting a frame. Can A finish transmitting before it detects that B has transmitted? Why or why not?

参考答案：
A 不能在检测到 B 的发送信号前发送完毕。

详细解析：

  • 最坏情况分析：
    • t = 0：A 开始发送帧，最小帧长 = 512 + 64 = 576 bit times（数据 + 前导码和 SFD）。
    • 在最坏情况下，B 在 A 的第一个比特恰好到达 B 之前（即 t = 224 bit times 时）开始发送。
    • t = 225 bit times：B 检测到碰撞（A 的信号到达 B）。
    • B 发出的信号在 t = 225 + 225 = 450 bit times 时到达 A。
    • 此时 A 还在发送（576 > 450），A 在 t = 450 bit times 时检测到碰撞并停止。
  • 因此 A 不可能在 576 bit times 之前发送完毕而不检测到 B 的传输。
  • 如果以太网的最小帧长再缩短，可能出现 A 发送完毕后才检测到 B 信号的情况，所以最小帧长必须满足 L_min ≥ 2 × R × d_prop。

# 第 49 题

题目（简答题，10 分）：
(10 points) Suppose four active nodes—nodes A, B, C and D—are competing for access to a channel using slotted ALOHA. Assume each node has an infinite number of packets to send. Each node attempts to transmit in each slot with probability p.

  1. What is the probability that node A succeeds for the first time in slot 5?
  2. What is the probability that some node (either A, B, C or D) succeeds in slot 4?
  3. What is the probability that the first success occurs in slot 3?
  4. What is the efficiency of this four-node system?

参考答案：

  1. P (A 在时隙 5 首次成功) = p (1-p)^3 × [1 - p (1-p)3]4

  2. P (某个节点在时隙 4 成功) = 4p (1-p)^3

  3. P (首次成功发生在时隙 3) = [1 - 4p (1-p)3]2 × 4p(1-p)^3

  4. 效率 = 4p (1-p)^3

详细解析：

  1. A 在时隙 5 首次成功：

    • A 在某一时隙成功的概率 = A 发送且 B、C、D 都不发送 = p × (1-p)^3
    • 前 4 个时隙均无任何节点成功 = [1 - 4p (1-p)3]4
    • A 在第 5 时隙成功 = p (1-p)^3
    • 合起来：P = [1 - 4p (1-p)3]4 × p(1-p)^3

  2. 某个节点在时隙 4 成功：

    • 任意一个节点成功 = A 成功 + B 成功 + C 成功 + D 成功
    • 每个节点成功的概率 = p (1-p)^3
    • P = 4p(1-p)^3

  3. 首次成功发生在时隙 3：

    • 前 2 个时隙都没有节点成功 = [1 - 4p (1-p)3]2
    • 第 3 个时隙有节点成功 = 4p (1-p)^3
    • P = [1 - 4p(1-p)3]2 × 4p(1-p)^3

  4. 效率：成功时隙的比例 = N × p × (1-p)^(N-1) = 4p(1-p)^3

    • 当 p = 1/N = 1/4 时效率最大，约为 4 × 1/4 × (3/4)^3 = 27/64 ≈ 0.422

# 第 50 题

题目（简答题，5 分）：
(5 points) Consider Figure 4. Suppose that all links are 10 Mbps. What is the maximum total aggregate throughput that can be achieved among the 14 end systems in this network? Why? [图片: https://p.ananas.chaoxing.com/star3/origin/6426d1c0bf864a9d022d547d51400dd6.png]

参考答案：
最大总聚合吞吐量为 70 Mbps。

详细解析：

  • 该网络中使用的是交换机（switch），交换机的每个端口是一个独立的冲突域。
  • 每个端系统独享其链路的 10 Mbps 带宽，且可以与其他端口并发通信。
  • 14 个端系统，理论上半双工时每个链路可全速收发占 10Mbps，但考虑每个通信需要一对端口（发送方和接收方），所以 14 台设备最多可同时进行 7 对并发通信（每对 10 Mbps）。
  • 因此最大聚合吞吐量 = 14 × 10 / 2 = 70 Mbps。

# 第 51 题

题目（简答题，5 分）：
(5 points) Suppose the three departmental switches in Figure4 are replaced by hubs. All links are 10 Mbps. What is the maximum total aggregate throughput that can be achieved among the 14 end system in this network? Why?

参考答案：
All of the 14 end systems will lie in the same collision domain. In this case, the maximum total aggregate throughput of 10 Mbps is possible among the 14 end systems.

详细解析：

  • 如果将交换机换成集线器（hub）：集线器是物理层设备，所有端口处于同一个冲突域。
  • 同一冲突域中，只要两台设备同时发送就会产生碰撞，任何时候只能有一台设备发送数据。
  • 因此所有 14 个端系统共享 10 Mbps 的带宽，最大总吞吐量仅为 10 Mbps。
  • 对比第 50 题（交换机方案）的 70 Mbps，可以看出交换机通过隔离冲突域大大提高了网络总吞吐量。
