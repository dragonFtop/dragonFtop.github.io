# 计算机网络与因特网

# 什么是因特网？（What is Internet?）

因特网：network of networks
因特网包含了计算机网络：因特网是一个世界范围的计算机网络。
计算机网络一般指的是有传统 PC、linux 工作站以及服务器所构成的网络。但是由于越来越多非传统设备接入网络，所以计算机网络的说法就过时了。
因特网的术语称网络中的设备为主机或者端系统。

端系统通过通信链路和分组交换机的网络连接到一起。
链路上的传输速率以 bit/s 为单位。
常见的分组交换机：路由器、链路层交换机。
端系统需要通过因特网服务提供商（ISP）接入因特网。
端系统、分组交换机和其他因特网不见都要运行多个协议，这些协议控制网络中的信息的接受和发送。
因特网主要协议：TCP/IP 协议。其中 TCP 是传输控制协议，IP 是网际协议。
因特网标准由阴天野望工程任务组（IETF）制定，IETF 的标准文档称为 RFC，RFC 文档定义了 TCP，IP，HTTP（用于 web）和 SMTP 协议（用于电子邮件）。
协议：定义了在两个或多个通信实体之间交换报文的格式和顺序，以及保温的发送 / 接收或其他事件所采取的操作。

# 网络边缘（Internet Edge）

与因特网相连的计算机和其他设备处于因特网边缘，故而被称为端系统：桌面计算机、服务器、移动计算机、一些非传统物品。
主机被分为：客户端和服务器。
数据中心：提供服务的服务的计算机集群，数据中心的主机被称为刀片服务器。数据中心的目的：

  1. 存储数据：提供 web 页面
  2. 处理数据：提供计算数据能力
  3. 为其它公司提供云计算

# 接入网

将端系统物理连接到边缘路由器的网络。

  1. 家庭接入：DSL（数字用户线）、电缆、光纤（FTTH）、5G 固定式无线。
  2. 企业（和家庭）接入：以太网和 WiFi。（局域网技术，LAN）
  3. 广域网接入：3G、4G、5G（基站）。

# 物理媒介

  1. 双绞铜线：UTP（无屏蔽双绞线）常用在建筑物内的计算机网络。双绞线不容易被光纤代替的原因：双绞线可以在一定距离内提供高速的 LAN 联网。
  2. 同轴电缆：STP（屏蔽双绞线），常见于电缆电视系统应用。可被用作引导型共享媒介。
  3. 光纤：细而柔软的、能够引导光脉冲的媒介。一个光脉冲表示一个 bit。一根光纤支持极高的比特速率且不受电磁干扰，但是部署成本较前两种高。
  4. 陆地无线电信道：不需要安装物理线路，具有穿透墙壁、提供移动用户连接以及长距离承载信号的能力。但存在路径损耗、遮挡衰弱、抗干扰能力弱的弊端。
  5. 卫星无线电信道：使用同步卫星、近地轨道卫星进行无线电传输的频信道。

# 网络核心（Internet Core）

网络核心：由分组交换机和链路构成的网状网络。

# 分组交换

在网络应用中，端系统彼此交换报文（message）。
交换机主要分为两类：路由器、链路交换机。
通信传输时间：一条链路发送 L bit 的分组，链路传输速率 R bit/s，则传输时间 T=L/R。

  1. 存储转发传输：将分组存储在交换机中，然后转发给下一个交换机。具体是指一个交换机开始向输出链路传输该分组的第一个 bit 前，必须接收到整个分组才能进行。

    如上图，交换机必须接收到完整的 1、2、3 分组后才能向目的地传输。
    传输时间计算问题：
    由 N 条速率均为 R 的链路组成的路径，从源到目的地传输 L bit 的分组，传输时间 T=N*L/R。
    如果传输 P 个分组，则传输时间 T=(N+N-1)*L/R。
  2. 排队时延和分组丢失：每个分组交换机具有一个输出缓存（输出队列）。
    排队时延：当链路忙于传输其他分组时，新到达的分组必须在缓存中等待。
    分组丢失：如果缓存已满，新到达的分组将被丢弃。
  3. 转发表和路由选择协议：每个端系统具有一个 IP 地址，当源主机项目的端系统发送分组时，源在分组的首部包含了目的地的 IP 地址，而路由器中存在一个转发表，找出导向目的端的合适的传输链路，也就是将目的地址因设为输出链路。

# 电路交换

电路交换：为端系统通信会话预留资源，不存在排队等待的问题。也就是端系统直接连接成专线链路。

  1. 频分复用 (FDM)：为端系统之间的连接创建一个专设频段，也就是空间上的专属占有，这个频段只用于该通信。
  2. 时分复用 (TDM)：链路上的时间被划分为固定时段的帧，每个帧又被划分为固定数量的时隙，每个时隙被分配给特定通信连接单独使用，也就是时间上的专属占用，该时间段上只用于该通信。
  3. 分组交换与电路交换对比：分组交换不适用于实时服务，因为它存在时延和丢包的问题，但是它带宽共享更好，更简单、有效、成本低，适合传输并发服务。而电路交换适用于实时服务，但是它需要更多的带宽，更复杂、成本高，适合传输单并发服务，且在静默期，电路交换中的链路处于空闲，造成资源浪费。

# 分组交换网中的时延、丢包和吞吐量问题

# 时延

时延一般可分为四种：

  1. 处理时延（节点）：检查分组首部和决定将该分组导向何处所需要的时间。（微秒级以下）
  2. 排队时延：分组在链路上形成队列等待传输的时间。（毫秒到微秒级）
  3. 传输时延：将所有分组推向链路所需要的时间。（毫秒到秒级）
  4. 传播时延：比特从链路起点传输到链路终点所需要的时间。（毫秒级）

    流量强度： L 一个分组的 bit 大小、 R 链路传输速率， a 分组到队列的平均速率，定义流量强度为 La/R 。
    若 La/R > 1 则 bit 到达碎裂平均速率超过该队列传输出去的速率，则该队列会进行拥塞，并且排队时延会持续增大。
    若 La/R < 1 则 bit 到达碎裂平均速率小于该队列传输出去的速率，则该队列不会进行拥塞，并且排队时延不会持续增大。

# 丢包

当流量强度接近 1 时，由于队列空间有限，没有多余空间存储新来的分组，路由器会丢弃这个分组，造成丢包。

# 吞吐量

瞬时吞吐量：任一时刻，主机接收到文件的速率（ bps ，bit per second）。
平均吞吐量：主机接收 F bit 用时 T s ，平均吞吐量为 F/T 。
在实际传输中，吞吐量取决于瓶颈链路的传输速率，也就是传输速率最小的链路速率。

# 协议

# 协议分层 (layer)

在协议分层的架构中，每一层都可以向上一层提供服务，被称为服务模型。每层通过在该层执行某些操作或者使用下一层提供的服务来提供完整服务。 一层协议常常分布在网路组件的各个部分中。
协议分层的优缺点：

  1. 优点：概念化和结构化，对系统组件的讨论更清晰。模块化是的系统组件更新更容易。
  2. 缺点：
    （1）功能冗余（重复实现）
    高层可能重复实现低层已有的功能，造成资源浪费。
    比如在链路层和传输层都实现了差错恢复机制。链路层负责每一段链路的错误检测与重传，而传输层又实现了端到端的差错恢复。
    （2）跨层信息依赖（违反层次分离）
    分层设计的核心目标是层次分离，即每层只依赖其直接下层提供的服务。
    某层功能需要依赖其他层才有的信息才能正常工作，例如传输层需要依赖数据链路层或网络层的时间戳信息来计算 RTT（往返时间）。

# 因特网协议栈

因特网协议栈自顶向下依次为：应用层、运输层、网络层、链路层、物理层。

  1. 应用层：网络应用程序以及应用层协议存在的地方。常见协议有 HTTP、SMTP、DNS 域名系统。
    这一层的信息分组被称为 报文（message）
  2. 运输层：在应用程序断电之间传输应用层报文。常见协议有 TCP、UDP 。
    其中 TCP 是面向连接的，提供端到端的可靠性、流量控制、将长报文划分为短报文、拥塞控制机制。 UDP 是无连接的，提供简单、快速、无可靠传输，它不提供非必要服务。
    这一层信息分组被称为 报文段（segment）
  3. 网络层：将网络层的信息分组，被称为 数据报（datagram） ，从一台主机主动移动到另一台主机。简而言之就是提供导航服务：寻址和标识、路由选择等。常见协议有 IP、ICMP 。
  4. 链路层：网络层依托于链路层，完成数据报在链路上节点间的传输，覆盖范围是上一个节点的网络层到下一个节点的网络层。常见的协议有以太网、DOCSIS 协议。
    链路层的信息分组被称为 帧（frame）
  5. 物理层：这一层仍是链路相关，也就是对链路层的物理描述，协议内容与数据传输的物理介质有关。这一层的信息被称为 比特（bit）

# 封装（encapsulate

每一层都会在上一层的信息分组的基础上添加本层特有的首部字段，形成本层的信息分组，这个过程叫做封装。
自顶向下的封装过程如下：应用层报文（application-layer message）-> 运输层报文段（transport-layer segment）-> 网络层数据报（network-layer datagram）-> 链路层帧（link-layer frame）。
每一层的分组 = 首部字段 + 有效载荷字段（上一层分组）。

# OSI 七层协议栈模型与 TCP/IP 四层协议栈模型

OSI 七层协议栈模型：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。
TCP/IP 四层协议栈模型：应用层、传输层、网络层、链路层。

# 网络攻击

常见网络攻击类型：

  1. 有害程序攻击：攻击者将 恶意程序（malware） ，如木马、病毒、蠕虫、后门、恶意软件、恶意脚本通过网络发送给目标主机，并运行，而大多数恶意程序可以 自我复制（self-replicating） 进一步感染更多主机，形成 僵尸网络（botnet） 。
  2. 攻击者访问控制攻击：攻击者访问控制攻击，也叫 拒绝服务攻击（DoS attack） ，攻击者通过发送大量请求，导致目标主机无法响应，从而导致目标主机无法正常处理用户请求。
  3. 网络嗅探攻击，例如使用 wireshark 这类网络流量分析攻击，抓取到传输的数据包，分析其中内容进行窃密、篡改、重放。
