# 信息泄露 - Git 泄露 - log

# 题目介绍

这是 CTFHub 平台上的一道 Git 泄露漏洞专项教学类 Web 题目，核心考察 Git 泄露漏洞的发现与利用方法。

# 出题背景

当前大量开发人员使用 Git 进行代码版本控制，并通过自动化工具部署站点。若部署配置不当，项目的 .git 文件夹会被直接部署到线上生产环境，攻击者可通过该文件夹还原整个代码仓库，引发 Git 泄露漏洞。

# 解题步骤

# 1. 发现 Git 泄露痕迹

进入题目容器后，首先观察目标页面特征，尝试直接访问 .git 相关核心文件（如 .git/HEAD 、 .git/index 等），发现可正常下载，说明网站存在 Git 泄露漏洞：

也可使用目录扫描工具 dirsearch 验证，扫描后同样能发现网站根目录下存在 .git 文件夹：

# 2. 使用 GitHack 还原 Git 仓库

在 Kali Linux 中安装并使用 GitHack 工具，还原目标网站的完整 Git 仓库（注意：该版本 GitHack 基于 Python2 开发，需使用 Python2 环境运行）。

# 3. 分析 Git 提交日志

进入还原后的 Git 仓库目录，执行 Git 日志查看命令，分析提交记录：

日志显示核心提交记录共 3 次，关键变化流程：

# 4. 对比提交差异获取 Flag

通过对比 add flag 和 remove flag 两次提交的代码差异，可直接提取出题目中的 flag：

最终获取的 Flag：

# 补充说明

  1. GitHack 工具原理：通过遍历 .git 文件夹下的文件，还原完整的 Git 仓库结构，支持大多数常见的 Git 泄露场景；
  2. 环境注意事项：若 Kali 中未安装 Python2，可通过 sudo apt install python2 快速安装；
  3. 拓展思路：除 GitHack 外，也可使用 GitTools 等工具实现类似的 Git 仓库还原操作。
