# 信息泄露 - SVN

# 题目介绍

这是 CTFHub 平台上的一道 SVN 泄露漏洞专项教学类 Web 题目，核心考察 SVN 泄露漏洞的利用方法。

# 题目背景

当开发人员使用 SVN 进行版本控制，对站点自动部署。如果配置不当，可能会将.svn 文件夹直接部署到线上环境。这就引起了 SVN 泄露漏洞。

# 解题

根据前面题目的经验直接使用 dirsearch 对网站进行目录扫描，结果如下：

发现了一个 .svn 目录，同时在其下发现了一个名叫 wc.db 的数据库文件。
这里选择使用 sqlite3 工具来分析这个数据库文件，结果如下：

根据学习了解知道：它是 SVN 1.7+ 版本后用来管理本地工作副本元数据的 SQLite 数据库，所有表都服务于跟踪文件状态、版本和操作。常见与 flag 有关的线索可能存在的位置有：NODES、PRISTINE、ACTUAL_NODES 这三个表中。

我们直接尝试在这些表中找到找到相关信息，这里直接试试能不能在 NODES 表的本地相对路径中找到一些信息：

这里看到了当前版本的网页中是存在一个 flag.txt 的文件的，想来这个就是我们要找的 flag。
但是这是个 svn 版本控制文件，其中的文件的文件名都是通过编码形成的哈希值，所以我们需要去 checksum 查找这个 flag.txt 的哈希值，如下：

$sha1$bf45c36a4dfb73378247a6311eac4f80f48fcb92
对于这种哈希值，除去前缀 $sha1$ ，剩下的就是 flag.txt 的哈希值。前两个字符 bf 是 flag.txt 被进行哈希后存放的文件夹，剩下的字符串就是 flag.txt 文件的文件名。所以我们构造如下 url 即可访问得到 flag.txt 文件。
http://example.com/bf/45c36a4dfb73378247a6311eac4f80f48fcb92
