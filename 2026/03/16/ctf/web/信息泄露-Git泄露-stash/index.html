# 信息泄露 - Git 泄露 - satsh

# 题目介绍

这是 CTFHub 平台上的一道 Git 泄露漏洞专项教学类 Web 题目，核心考察 Git 泄露漏洞的发现与利用方法，重点聚焦 git stash 相关数据的泄露与提取。

# 出题背景

当前大量开发人员使用 Git 进行版本控制，并基于 Git 实现站点自动部署。若服务器配置不当，可能会将包含完整版本记录的 .git 文件夹直接部署到线上环境，由此引发 Git 泄露漏洞。攻击者可通过该漏洞还原仓库历史、提取未提交 / 已删除的敏感代码或数据，造成信息泄露。

# 核心知识点：Git Stash 是什么

Git Stash（中文常译作 “暂存”）是 Git 的核心功能之一，作用是临时保存工作区中未提交的修改，让工作区快速恢复到干净状态（与 HEAD 提交保持一致），常用于切换分支前暂存未完成的修改。

正常情况下，Stash 是纯本地操作，数据存储在本地 .git 目录中；但如果开发者误将包含 Stash 数据的 .git 目录部署到线上服务器，就会导致 Stash 内容泄露，带来安全风险。

# 解题步骤

# 步骤 1：检测 Git 泄露漏洞

基于 Git 渗透的基础经验，可通过两种方式验证目标是否存在 Git 泄露：

  • 直接访问目标站点的 .git 目录（如 http://example.com/.git ）或关键文件（如 .git/HEAD ），若能正常访问，则说明存在漏洞；
  • 使用目录扫描工具（如 dirsearch）扫描目标，检查是否存在 .git 目录。

  2. GitHack 是还原泄露 Git 仓库的常用工具，能完整提取仓库文件；
