# 网络层

# introduction

网络层是 OSI 模型中的第三层，负责将数据从源主机传输到目的主机。其主要功能包括：

  • 路由选择 (Routing)：确定数据包从源到目的的传输路径。
  • 转发 (Forwarding)：将数据包从路由器的输入端口转移到输出端口。
  • 逻辑寻址 (Logical Addressing)：通过 IP 地址标识网络中的主机，使得跨网络通信成为可能。

网络层的核心协议是 IP 协议 (Internet Protocol)，它提供无连接、尽力而为的数据传输服务。为了支撑 IP 协议的正常运作，网络层还包含：DHCP（动态主机配置）、ICMP（网络诊断）、路由算法（如 Dijkstra、Bellman-Ford）以及路由协议（RIP、OSPF、BGP）等。

# IP 协议 (internet protocol)

# IP datagram

  • ver ：记录 IP 协议的版本号：IPv4 or IPv6
  • header len ：记录 IP 报头长度，单位为 4 字节，即 32bit。一般来说，header len 为 5，表示 IP 报头长度为 20 字节。
  • type of service ：记录 IP 报头中 tos 字段的值，tos 字段的值为 8bit，即 1 字节。标记数据包的优先级和服务需求。
  • total len ：记录 IP 报头和数据报的总长度，单位为字节。
  • IPID ：用于标识一个分片，同一个 IP 数据包的分片的 IPID 相同，分片偏移量不同。
  • flags ：用于标识分片信息，DF 标志为 1 时，表示不分片。MF 标志为 1 时，标识该分片还有后续分片
  • fragment offset ：用于标识分片的偏移量，便于拼接分片，单位为 8 字节。
  • time to live ：表示该数据包可以经过的跳数，也就是允许其传递经过的路由器数的次数，防止数据包无限循环传递。
  • upper layer protocol ：记录上层协议，即数据报所包含的上层协议（TCP、UDP、ICMP、OSPF 等）。
  • checksum ：校验和，用于检查 IP 报头和数据报的错误。

# IP Address（IPv4）

# subnet

IP 地址一共 32 位，分为 4 个字节，每个字节 8 位，共 32 位。
前三个字节标识主机所整个网络 id，最后一个字节标识子网 id 和标识主机在子网中的 id。

  • 子网(Subnet) ：在网络中不需要经过路由器便可相互到达的主机构成的网络。
  • 掩码(Mask) ：子网的掩码，用于标识子网中的主机范围。

# IP Address Class

IP 地址分为 4 类：

  • A 类：0.0.0.0-127.255.255.255
  • B 类：128.0.0.0-191.255.255.255
  • C 类：192.0.0.0-223.255.255.255
  • D 类：224.0.0.0-239.255.255.255

    从 A -> B ->C ->D：子网数量增加，但是每个子网中的主机数量减少。

另外特殊的 保留IP地址 ：

  • 0 地址（0.0.0.0）：表示本主机或任意主机
  • 广播地址：
    1. 本网广播 (255.255.255.255)：全 1 地址，其广播范围局限于主机所在子网。
    2. 指定网络广播 (x.x.x.subnet_id+host_id=1)：主机地址全为 1，表示是指定子网的广播地址。
    3. 环回地址 (127.0.0.0)： 用于本机测试，用于自己访问自己。

# 获取 IP 地址 - DHCP

DHCP (Dynamic Host Configuration Protocol)：动态主机配置协议，用于获取 IP 地址，采用租赁 (lease) 的方式分配，它作用在局域网（子网）中。
关于 IP 中子网的部分如何获取，是由 ISP (网络服务提供商) 提供。

# DHCP 的作用

  • 能够更新主机正在使用的 IP
  • 允许 IP 地址被多个主机重用，但允许连接在网络中的主机持有该 IP，其余离线的主机不能持有该 IP。
  • 支持移动用户动态连续性接入网络

# DHCP 工作流：

  1. host 向 DHCP sercer 发送 DHCP discover 报文，请求 DHCP server 分配 IP Address。
  2. DHCP server 向 host 发送 DHCP offer 报文，包含分配的 IP 地址。
  3. host 向 DHCP server 发送 DHCP request 报文，请求 DHCP server 确认 IP 地址。
  4. DHCP server 向 host 发送 DHCP ack 报文，允许 host 使用分配的 IP 地址。
    为什么是这样的机制？
    解释 ：避免 IP 地址冲突，网络中可能多个 host 同时请求到相同的 IP 地址，所以需要 DHCP server 进行确认，只允许一个 host 使用该 IP 地址。

# 路由算法分类

分类一：根据路由的信息覆盖范围

  1. 路由器上有全网完整拓扑结构和链路代价： 链路状态法(Link State) 。

分类二：根据网络中节点变化速度，也就是路由变化缓急：

  1. 路由变化缓慢：Static
  2. 路由变化频繁，需要定期更新：Dynamic

# 分层路由 (Hierarchical routing)

分层路由的 分层 ，表现在将一个 ISP 提供的网络称呼为 AS (autonomous system)，而大型 ISP 会将 AS 划分为多个 AS，每个 AS 内部自治，彼此外交。

维度自治系统内部 (Intra-AS)自治系统之间 (Inter-AS)
类比城市内部的交通城市与城市之间的高速公路
管理权统一管理 (如某公司或 ISP)各自独立管理
协议要求必须相同可以不同
协议名称IGP (内部网关协议)EGP (外部网关协议，主要是 BGP)
典型协议OSPF, RIP, IS-ISBGP (边界网关协议)

AS 之间的 “外交” 通过 Gateway Router 实现，其特点：在其 AS 的 edge 且能够到达其他 AS。

转发表由自治系统内（Intra-AS）与自治系统间（Inter-AS）的路由算法共同配置。其中，内部协议负责设置去往内部目的地的表项，而外部协议需结合内部协议，共同设置去往外部目的地的表项。

比喻：一个省份就是一个AS，你在一个AS内，得先问Intra-AS，你该去哪个车站上车，你去那个车站后，还得问Inter-AS，你该坐到哪个站进行转车或下车？

# intra-AS（AS 内部路由协议）

AS 内部的路由协议，统称为 IGP (Interior Gateway Protocol)，内部网关协议。
主要了解以下两种：

  • RIP (Routing Information Protocol)：路由信息协议
  • OSPF (Open Shortest Path First)：开放最短路径协议

# RIP 路由信息协议

RIP (Routing Information Protocol) 是一种基于 距离向量 (Distance Vector) 算法的 IGP 协议，使用 跳数 (hop count) 作为代价度量。
核心特征：

  • 最大跳数为 15，16 跳视为不可达（限制了 RIP 适用于小型网络）。
  • 每隔 30 秒与邻居交换一次完整的路由表（Response 报文）。
  • 若 180 秒未收到邻居的路由更新，则将该邻居标记为不可达。
  • 运行在 UDP 520 端口。

RIP 的工作流程：

  1. 路由器启动后，向所有接口发送 Request 报文，请求邻居的路由表。
  2. 收到 Response 后，按 Bellman-Ford 逻辑更新自身路由表：若来自邻居的距离 + 1 小于当前记录，则更新。
  3. 周期性向邻居广播自身路由表。

RIP 的缺陷：

  • 收敛慢：好消息传得快，坏消息传得慢（无穷计数问题，count-to-infinity）。
  • 路由环路：可通过毒性逆转 (poison reverse)、水平分割 (split horizon) 缓解，但不能根除。
  • 仅以跳数为度量，不区分链路带宽，可能绕远路走低速链路。
  • RIP 报文的大小不能超过 512 字节，扣除必要的协议头部后，剩下的空间刚好只能容纳 25 条路由条目。

RIP 协议在链路故障时的处理流程：

  1. 故障检测
    超时判定：若 180 秒（6 个更新周期）未收到邻居的路由通告，则判定该邻居或链路失效。
  2. 故障响应
    路由失效：立即标记经由该邻居的所有路由为无效。
    触发更新：向其他邻居发送新的路由通告，引发连锁反应，使故障信息在全网扩散（注：RIP 收敛速度相对较慢）。
  3. 防环机制
    毒性逆转：为防止 “乒乓效应” 导致的路由环路，路由器会将失效路由的跳数设为 16（即无穷大 / 不可达），并反向通告给相关邻居，明确告知路径已断开。

# OSPF 开放最短路径协议

OSPF (Open Shortest Path First) 是一种基于 ** 链路状态 (Link State)** 算法的 IGP 协议，使用 Dijkstra 算法计算最短路径。
它的 "Open" 表示协议规范公开，任何厂商均可实现。
核心特征：

  • 每个路由器维护完整的网络拓扑数据库（LSDB, Link State Database）。
  • 使用 **LSA (Link State Advertisement)** 向区域内的所有路由器通告链路状态。
  • 以 ** 代价 (cost)** 为度量，代价与链路带宽成反比（带宽越大，cost 越小）。
  • 收敛速度快，网络变化时立即洪泛 LSA，触发全网重新计算。
  • 运行在 IP 协议上（协议号 89），不使用 UDP/TCP。
  • 安全性：支持报文认证，防止恶意路由信息注入。
  • 多路径支持 (ECMP)：允许存在多条相同开销的路径进行负载分担（RIP 仅支持单路径）。
  • 服务类型 (TOS)：针对不同服务类型（如实时、尽力而为）可设置不同的链路开销，实现基于业务的路由选择。

OSPF 的分层结构 —— 区域 (Area)：

  • 单区域 (Single-Area)：所有路由器属于同一 Area，适用于小型网络，配置简单。
  • 多区域 (Multi-Area)：将 AS 划分为多个 Area，减少 LSA 洪泛范围，降低路由开销。
    • 骨干区域 (Backbone Area)：Area 0，所有其他区域必须连接到 Area 0。
    • 区域边界路由器 (ABR) 负责在不同 Area 之间交换路由信息。
    • 自治系统边界路由器 (ASBR) 负责将外部路由（如来自 BGP 的）注入 OSPF 域。

OSPF 报文类型：

类型名称作用
1Hello发现邻居、建立邻接关系、选举 DR/BDR
2Database Description(DBD)描述 LSDB 摘要，用于数据库同步
3Link State Request(LSR)请求特定的 LSA
4Link State Update(LSU)携带完整的 LSA 信息
5Link State Ack(LSAck)确认收到 LSU

OSPF 邻接建立流程（以广播网络为例）：

  1. Down → Init：发送 Hello 报文发现邻居。
  2. Init → 2-Way：收到邻居 Hello，且其中包含自身 Router ID，进入 2-Way 状态。
  3. DR/BDR 选举：在广播多路访问网络（如以太网）中，选举指定路由器 (DR) 和备份指定路由器 (BDR)，减少 LSA 洪泛流量。
  4. ExStart → Exchange：主从协商后，交换 DBD 报文同步 LSDB 摘要。
  5. Loading：通过 LSR/LSU 获取缺失的 LSA。
  6. Full：邻接关系建立完成，LSDB 完全同步。

# inter-AS（AS 间路由协议）

AS 之间的路由协议关注的是可达性与策略 (Policy)，而非单纯的最短路径。不同 AS 可能属于不同运营商或组织，因此路由选择需要考虑商业关系、流量工程等策略因素。

# BGP 边界网关协议

BGP (Border Gateway Protocol) 是 Internet 上唯一的 EGP 协议，当前版本为 BGP-4，是 Internet 运行的基石。
它是一种 ** 路径向量 (Path Vector)** 协议，在距离向量的基础上增加了 AS 路径信息，从而避免环路。
核心特征：

  • 运行在 TCP 179 端口，保证可靠传输。
  • 通告的是前缀 (prefix) + 属性 (attributes)，而非单纯的跳数 / 代价。
  • 使用 AS Path 记录路由经过的 AS 序列：收到一条路由时，若自身 AS 号已出现在 AS Path 中，则丢弃（防环）。
  • 路由选择基于丰富的路径属性，而非单一度量。

eBGP 与 iBGP：

维度eBGP (External BGP)iBGP (Internal BGP)
邻居关系不同 AS 的路由器之间同一 AS 内的路由器之间
AS Path 处理将自己 AS 号添加到 AS Path不修改 AS Path
TTL默认 TTL=1默认 TTL=255
通告规则从 eBGP 邻居学到的路由通告给所有邻居从 iBGP 邻居学到的路由不转发给其他 iBGP 邻居（全互联要求，split-horizon 规则）

BGP 路径属性（选路依据，按优先级排列）：

  1. Local Preference：本地优先级，值越大越优先（用于控制出站流量）。
  2. AS Path：AS 路径长度，越短越优先。
  3. Origin：路由起源，IGP (network 命令) < EGP < Incomplete (重分发)。
  4. MED(Multi-Exit Discriminator)：多出口鉴别器，值越小越优先（用于影响相邻 AS 的入站流量）。
  5. Next Hop：下一跳是否可达。结合 IGP（如 OSPF）计算到达下一跳的最短距离，选择距离最近的出口，此策略称为热土豆路由 (Hot Potato Routing)—— 尽快将数据包扔出本 AS。
  6. 其他：Router ID、Cluster List 等。

BGP 策略控制：

  • eBGP：用于 AS 间交换路由，从 eBGP 邻居学到的路由通告给所有邻居（iBGP 和 eBGP）。
  • iBGP：用于 AS 内部同步从 eBGP 学到的路由，但从 iBGP 邻居学到的路由不转发给其他 iBGP 邻居（全互联要求）。
  • 商业关系策略：运营商根据商业关系制定路由策略，常见的三种关系：
    • Customer（客户）：付费购买 transit 服务，路由优先级最高。
    • Provider（提供商）：提供 transit 服务获取收入。
    • Peer（对等方）：免费互连交换各自客户的路由，通常不为对方的非客户流量提供中转服务。

BGP 状态机：

状态说明
Idle初始状态，等待开始建立连接
Connect发起 TCP 连接
ActiveTCP 连接失败，重试中
OpenSent已发送 Open 报文，等待对方 Open
OpenConfirm收到对方 Open，等待 Keepalive 确认
Established连接建立，开始交换 Update 报文

BGP 报文类型：

  • Open：建立 BGP 邻居，协商参数（AS 号、Router ID、Hold Time 等）。
  • Keepalive：维持邻居关系（默认 60 秒一发，Hold Time 180 秒）。
  • Update：通告新路由或撤销失效路由。
  • Notification：检测到错误时发送，关闭邻居连接。
