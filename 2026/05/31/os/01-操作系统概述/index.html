# 操作系统概述

# 操作系统的概念、功能和目标

操作系统是用户和计算机硬件之间的接口。

# 提供的功能

接口类型说明
命令接口联机命令接口（交互式） / 脱机命令接口（批处理）
程序接口系统调用，供应用程序调用
GUI图形用户界面（Windows /iOS/ Android）

# 目标

方便用户使用，高效管理硬件资源。

# 操作系统的特征

# 并发与并行

  • 并发（Concurrency） ：多个事件在 同一个时间段内 交替发生，宏观上看是同时进行的，微观上看是交替串行执行的。核心是 "看起来同时" 。
  • 并行（Parallelism） ：多个事件在 同一个时刻 真正同时发生。核心是 "物理上同时" 。

直观类比 ：

并发并行
吃饭 + 回信息吃一口、回一条、再吃一口、再回一条...不可能（一个人只有一双手）
厨房做菜一个灶台，两道菜轮流炒两个灶台，两道菜同时炒
柜台服务一个服务员，轮流向每个顾客说一句话多个服务员，同时服务多个顾客

关键区别 ：

维度并发并行
执行方式交替执行（微观串行）同时执行（微观也同时）
硬件要求单核即可（靠快速切换）必须多核 / 多 CPU
关注点任务的 调度 与 结构 设计任务的 执行速度 与 吞吐量
解决的问题多任务如何组织、不互相阻塞如何让计算更快完成
反义词顺序（Sequential）串行（Serial）

两者关系 ：并发是逻辑上的设计概念，并行是物理上的执行概念。一个程序可以：

  • 既不并发也不并行：纯串行程序，单线程跑在单核上
  • 并发但不并行：多线程程序跑在 单核 CPU 上，OS 快速切换线程
  • 并发且并行：多线程程序跑在 多核 CPU 上，不同线程真正同时运行
  • 并行但不并发：一段数据被拆成多份，多核同时计算，但任务间没有交替协同（如 SIMD 指令）

# 共享

共享方式说明
互斥共享方式一个时间段内只允许一个进程访问该资源
同时共享方式允许一个时间段内多个进程 "同时" 访问（如磁盘读 / 写）

# 虚拟

把一个物理上的实体变为若干个逻辑上的对应物。

  • 空分复用技术
  • 时分复用技术

# 异步

在多道程序环境下，允许多个程序并发执行，但由于资源有限，进程的执行不是一贯到底的，而是走走停停，以不可预知的速度向前推进。只有系统拥有并发性，才有可能导致异步性。

# 操作系统的发展与分类

阶段特点
手工操作阶段纸带机，用户独占全机，人机速度矛盾
单道批处理系统外围机 + 磁带，引入脱机输入 / 输出
多道批处理系统操作系统开始出现，多道程序并发执行
分时操作系统轮流处理作业，强调 交互性 ，不能处理紧急任务
实时操作系统优先处理紧急任务，强调 及时性 和 可靠性
网络操作系统实现网络通信与资源共享
分布式操作系统多台计算机协同工作，呈现为一个整体
个人计算机操作系统Windows /macOS/ Linux 桌面版

# 分时操作系统（Time-Sharing OS）

核心思想 ：将 CPU 时间切成很短的时间片（time slice），轮流分配给各个作业 / 用户使用。由于时间片很短（通常几十 ms），每个用户都感觉自己独占了一台计算机。

产生背景 ：批处理系统在作业执行期间用户无法干预，交互性极差 —— 作业提交后，剩下的就是等待，直到输出结果才发现问题。分时系统就是为了解决这个痛点而诞生的。

工作流程 ：

关键特征 ：

特征说明
多路性一台主机连接多个终端，同时服务多个用户
独立性每个用户感觉独占主机，互不影响
及时性用户请求能在可接受的时间内得到响应
交互性用户可以随时输入命令，系统立即响应（编辑、调试等）

典型代表 ：CTSS（兼容分时系统）、Multics、Unix 早期版本。

主要缺点 ：不能处理紧急的实时任务（工业控制、导弹导航等），因为即使请求很紧急，也得排队等时间片。

# 实时操作系统（Real-Time OS）

核心思想 ：系统能在严格的时间限制内对外部事件做出响应。正确性不仅取决于计算结果，还取决于 响应时间 。

分类 ：

类型时限要求超时后果举例
硬实时系统必须在绝对严格的时间内完成处理失败即系统崩溃 / 灾难性后果导弹制导、自动驾驶、核电站控制
软实时系统尽量在时限内完成，偶尔超时可容忍性能下降但不造成严重后果视频播放、在线游戏、视频会议

关键特征 ：

特征说明
及时性对外部事件的响应和处理必须在规定时间内完成
可靠性工业控制 / 医疗设备等场景要求极高，不允许中途宕机
可预测性任务的最坏情况执行时间必须是可以预先评估的
事件驱动外部事件到达时，系统必须立即响应，而非定时轮询

典型应用 ：工业机器人控制、航空航天、医疗设备、通信基站等。

# 分时系统 vs 实时系统 对比

维度分时操作系统实时操作系统
核心目标交互性和多用户公平共享及时性和可靠性
响应速度较快（人能接受即可，秒级）极快（毫秒 / 微秒级）
时限约束无严格时限有严格时限要求
调度策略公平轮转（RR 等）基于优先级抢占（EDF / RM 等）
用户交互强交互，用户可随时输入交互性弱，通常自动运行
紧急任务不能优先处理可以打断当前任务优先处理
典型场景大学机房多终端系统、Unix 服务器导弹、汽车、工业控制、基站

# 操作系统的运行机制与体系结构

# 运行机制

两种指令 ：

  • 特权指令：只能由内核程序执行
  • 非特权指令：用户程序可执行

两种处理器状态 ：

  • 核心态（内核态，root 权限）
  • 用户态

两种程序 ：

  • 内核程序：运行在核心态
  • 应用程序：运行在用户态

# 操作系统内核

内核组成功能
时钟管理实现计时功能
中断处理处理各种中断信号
原语程序运行具有原子性，不可中断
进程管理创建、调度、同步、通信
存储器管理内存分配、回收、虚拟内存
设备管理I/O 设备驱动与控制

# 体系结构

类型说明
大内核将主要功能模块都作为系统内核，运行在核心态
微内核只把最基本的功能保留在内核，其他放在用户态

# 中断和异常

# 中断机制

操作系统通过中断介入并开展管理工作。"用户态 -> 核心态" 的切换通过中断实现，且中断是唯一途径。

# 中断的分类

# 外中断的处理过程

  1. 外部设备发出中断信号
  2. CPU 执行完当前指令后检查中断
  3. 保存当前程序上下文
  4. 转入中断处理程序
  5. 恢复上下文，返回原程序继续执行

# 系统调用

应用程序通过系统调用请求操作系统的服务，保证系统的稳定性和安全性。

# 系统调用 vs 库函数

系统调用库函数
层级操作系统向上层提供的接口语言 / 库层面封装
关系底层原语部分库函数是对系统调用的进一步封装
使用直接调用开销较大当今应用程序大多通过库函数间接进行系统调用
