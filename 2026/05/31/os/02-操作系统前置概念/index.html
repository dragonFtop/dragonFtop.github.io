# 计算机系统概述

# 1.1 基本构成

计算机有四个主要的结构化部件：

  1. 处理器：控制计算机操作，执行数据处理功能。
  2. 内存：存储数据和程序，一般为易失性存储器。与之相对的是磁盘存储器。
  3. 输入输出模块：在计算机和外部环境之间移动数据。包括辅助存储器设备、通信设备和终端。
  4. 系统总线：在处理器、内存和输入、输出模块之间提供通信的设备。

# 1.2 微处理器

微处理器可以在单个芯片上容纳一个处理器。

  • 发展史
  • 多处理器 ，也就是多内核处理器。
  • 图形处理单元 ：采用单指令多数据（SIMD）并行技术。如今 GPU 不仅用于渲染高级图形，还被广泛用于通用计算（GPGPU），如深度学习、科学计算等。
  • 数字信号处理器 ：对流信号，如音频和视频进行处理。
  • 片上系统 ：将 CPU、内存、输入输出设备等集成到一个芯片中，逐渐取代上述传统微处理器

# 1.3 指令执行

处理器执行的程序是由一组存储在存储器中的指令组成的。
单个指令所需要的处理成为一个指令周期。一般包括取指阶段和执行阶段（可能出现中断）。

  • 一般指令周期 ：处理器从存储器中取一条指令，程序计数器（PC）存入下一次要取得指令的地址，取出的指令存入指令存储器（IR），IR 解释指令并执行相应操作（分配存储器单元，执行运算等）。
  • 解释指令的 4 类动作 ：
  • 处理器 <-> 存储器 ：数据从处理器到存储器或从存储器到处理器。
  • 处理器与 I/O 设备之间 ：数据从输入设备到处理器或从处理器到输出设备。
  • 数据处理 ：与数据相关的算术操作或逻辑操作。
  • 控制 ：改变指令执行顺序。



    接下来尝试理解下面的指令执行例子（为了方便都是十进制），其中 AC 是累加处理单元。
  • 步骤一 ：根据当前 PC 值，从内存中取出地址为 300 的指令 1940，并保存在 IR 中，PC 值自增 1，即 PC=301。
  • 步骤二 ：IR 指令为 1940，操作码为 1，地址码为 940，即将地址为 940 的数据 0003 存入 AC 中。
  • 步骤三 ：根据当前 PC 值，从内存中取出地址为 301 的指令 5941，并保存在 IR 中，PC 值自增 1，即 PC=302。
  • 步骤四 ：IR 指令为 5941，操作码为 5，地址码为 941，将 AC 中的数据 0003 与地址为 941 的数据 0002 相加，结果存入 AC 中。
  • 步骤五 ：根据当前 PC 值，从内存中取出地址为 302 的指令 2941，并保存在 IR 中，PC 值自增 1，即 PC=303。
  • 步骤六 ：IR 指令为 2941，操作码为 2，地址码为 941，将 AC 中的数据 0005 存入地址为 941 中。

# 1.4 中断

中断最初是用于提高处理器效率的一种手段，减少处理器空闲浪费。

# 1.5 存储器层次结构

存储器系统的层次结构从速度快 / 容量小 / 价格高到速度慢 / 容量大 / 价格低排列：

  • 关键特性 ：
  • 易失性 ：断电后数据丢失（寄存器、缓存、主存）
  • 非易失性 ：断电后数据保留（硬盘、SSD、磁带）
  • 上层作为下层的缓存，利用 局部性原理 保证整体性能接近上层、容量接近底层

# 存储器访问的局部性原理

类型描述例子
时间局部性刚被访问过的数据短期内可能再次被访问循环中的变量、频繁调用的函数
空间局部性被访问数据附近的数据短期内可能被访问数组遍历、顺序执行的指令
算法局部性程序自身访问模式带来的聚集性递归栈操作、嵌套循环

这三者共同决定了缓存的命中率，是高速缓冲存储器设计的理论基础。

# 1.6 高速缓冲存储器

# 缓存设计要素

要素说明
缓存大小越大命中率越高，但成本增加、速度降低
映射方式决定主存块如何对应到缓存行
替换算法决定缓存满时淘汰哪一行
写策略决定写入时如何同步缓存与主存

# 三种映射方式

映射方式原理特点
直接映射每个主存块映射到固定的一个缓存行硬件简单，但冲突率高
全相联映射主存块可以映射到任意缓存行命中率高，但比较电路复杂
组相联映射缓存分组，组内全相联，组间直接映射综合前两种的折中方案

# 替换算法

算法说明
LRU （最近最久未使用）淘汰最长时间未被访问的行。命中率高，实现成本较高
FIFO （先进先出）淘汰最早进入缓存的行。实现简单，可能淘汰热点数据
Random （随机）随机选择一行淘汰。实现最简单，性能接近 FIFO
LFU （最不经常使用）淘汰被访问次数最少的行。需要计数器，开销较大
CLOCK （时钟算法）环形遍历，访问位 = 0 则淘汰，否则清 0 继续。折中方案

# 写策略

策略原理优点缺点
写直达（Write-Through）每次写入同时更新缓存和主存数据一致性好写操作效率低
写回（Write-Back）写入只更新缓存，淘汰时写回主存写操作效率高数据一致性复杂
写分配（Write-Allocate）写不命中时先加载到缓存再写配合写回策略使用—

# 多级缓存思想

现代 CPU 通常使用 3 级缓存：

  • L1 ：指令和数据分离（哈佛架构），最快，每核独立
  • L2 ：指令和数据统一，比 L1 慢但更大，每核独立或共享
  • L3 ：所有核心共享，容量最大，速度最慢

多级缓存能高效工作的关键原因： 绝大多数程序都表现出良好的局部性 。

# 1.7 直接内存存取（DMA）

# 为什么需要 DMA

在程序控制 I/O 和中断驱动 I/O 方式下，CPU 需要亲自参与每次数据传送。当需要传输大量数据时（如从磁盘读取一个文件），CPU 被频繁中断，效率极低。

DMA 的目标： 让 CPU 从数据搬运工作中解放出来 。

# DMA 的工作原理

# DMA 控制器的核心寄存器

寄存器全称功能
DRData Register暂存正在传输的数据
MARMemory Address Register存放目标内存地址，每传一字节自动递增
DCData Counter剩余待传输字节数，递减到 0 时传输结束
CRCommand/Status Register存放 CPU 发出的命令和设备状态

# DMA 的三种传送方式

方式说明特点
成组传送（Burst Mode）DMA 占用总线直到数据块全部传完传输效率最高，但会阻塞 CPU 使用总线
周期挪用（Cycle Stealing）DMA 在 CPU 不使用总线时窃取一个总线周期，传一个字对 CPU 影响小，但传输较慢
交替访问（Transparent Mode）DMA 和 CPU 交替使用总线，各占不同相位CPU 无感知，需要复杂的硬件时序支持

# DMA 与中断驱动的对比

维度中断驱动方式DMA 方式
数据传送单位每次一个字连续的多个块
数据流向设备 -> CPU -> 内存设备 -> 内存（直接）
CPU 干预频率每传输一个字中断一次整个数据块传输完成后中断一次
CPU 利用率低（频繁中断处理）高（CPU 做其他事情）

# 1.8 多 CPU 和多核

# 基本概念

概念说明
多处理器（Multiprocessor）一台计算机中有多个独立的 CPU 芯片 / 插槽，共享主存和 I/O 子系统
多核（Multicore）一个 CPU 芯片内部集成多个处理器核心，共享芯片内缓存和总线接口
多线程（Multithreading / SMT）一个物理核心同时运行多个线程，硬件快速切换，掩盖内存延迟

# 多处理器的分类

类型特点例子
SMP （对称多处理）所有处理器地位对等，共享同一主存和 I/O，由同一个 OS 管理现代 x86 服务器
AMP （非对称多处理）有主处理器和从处理器之分，各处理器有专门任务嵌入式 SoC
NUMA （非一致存储访问）每个处理器有本地内存，访问本地内存快，访问远程内存慢高端服务器

# 多核架构的关键设计问题

问题说明
缓存一致性多个核心的缓存中可能存在同一内存地址的副本，需要协议保证数据一致（如 MESI 协议）
核间通信核心之间如何交换数据和信号，可通过共享缓存、专用总线或消息传递
并行粒度任务级并行 vs 数据级并行 vs 指令级并行
功耗管理多核带来的功耗密度问题，需要动态频率调节（DVFS）和核心门控（power gating）

# 从单核到多核的演进历程

# GPU：另一种维度的 "多核"

维度CPUGPU
核心数量几个～几十个上千个
核心复杂度高（超标量、乱序执行、分支预测）低（简单流水线）
优化目标减少单线程延迟（Latency）最大化总吞吐量（Throughput）
适合任务逻辑复杂、分支多的串行 / 少线程任务数据并行的大规模计算任务
编程模型通用多线程SIMT（单指令多线程），如 CUDA

GPU 本质上是一种 大规模并行处理器 ，利用海量线程隐藏内存访问延迟（warp 切换），而非依赖大缓存。这与 CPU 的设计哲学正好相反。
