# 线程

# 进程与线程

进程的两个重要特点：

  1. 具有资源所有权：进程可以占据计算机资源，比如 CPU、内存、I/O 设备等。
  2. 调度 / 执行：进程执行时具有执行轨迹，不同进程的执行交替进行。进程具有执行态和被操作系统分配的优先级，可以被调度和分派。
    为了区别这两个特点，产生了线程的概念。
    进程 ：资源分配的基本单位，是运行中的程序
    线程 ：CPU 调度的基本单位，是进程内的执行路径
    可以用一个比喻来说，进程是一个房子，线程是这个房间里的人，房子内有很多财产资源，人可以在房子内完成各种不同的活动。

线程相较于进程的优点有：

  1. 在已创建的进程中创建新的线程比创建一个全新的进程快得多。
  2. 线程终止时间比进程短。
  3. 同一进程内线程的切换快于进程间的切换。
  4. 线程提高了不同执行程序之间的通信效率。进程之间的通信需借助内核，但线程由于运行在同一个进程内，其存储空间和资源是共享的，所以就不需要内核介入，故线程之间的通信效率更高。

# 多线程

# 线程的功能

  • 线程状态 ：与进程一样，线程主要状态有就绪态、运行态、阻塞态。挂起态对于线程来说没有意义，因为进程被挂起时，整个进程中的线程都被挂起，当进程被从硬盘中换出的时候，该进程的所有线程都一同被换出。
  • 线程状态改变 ：
  1. 派生：进程被创建的同时会在该进程中派生出一个线程，后续进程中的线程也可以派生出新的线程。
  2. 阻塞：线程在等待事件时出现排队阻塞。
  3. 解除阻塞：线程被解除阻塞后被转移到就绪队列
  4. 结束：线程结束后释放其上下文和栈空间。

# 线程的重要属性

属性说明
调度单位线程是处理机调度的基本单位
多核利用多 CPU 计算机中，各个线程可占用不同的 CPU
线程标识每个线程都有一个线程 ID、线程控制块（TCB）
三种基本状态就绪、阻塞、运行
资源拥有线程几乎不拥有系统资源（资源属于进程）
资源共享同一进程的不同线程间共享进程的资源
通信效率共享内存地址空间，同一进程中的线程间通信甚至无需系统干预
同进程切换同一进程中的线程切换，不会引起进程切换，系统开销很小
跨进程切换不同进程中的线程切换，会引起进程切换，系统开销较大

# 线程的实现方式

# 用户级线程（ULT）

由应用程序管理，从用户的视角能看到的线程。操作系统内核不知道 ULT 的存在，只能看到一个进程。

优点 ：线程切换不需要内核态切换，开销极小；应用程序可以自定义调度算法
缺点 ：一个 ULT 阻塞会导致该进程所有线程阻塞（内核只看到一个进程在等待）；无法利用多核并行

# 内核级线程（KLT）

由操作系统管理，从操作系统内核视角能看到的线程。内核维护每个 KLT 的上下文和调度。

优点 ：一个线程阻塞不影响同进程其他线程；可以利用多核并行
缺点 ：线程切换需要内核态切换，开销较大

# ULT 与 KLT 的对比

维度用户级线程（ULT）内核级线程（KLT）
管理方应用程序 / 线程库操作系统内核
内核感知内核不知道 ULT 的存在内核知道每个 KLT
切换开销小（无需内核态切换）大（需要内核态切换）
阻塞影响一个 ULT 阻塞会使整个进程阻塞一个 KLT 阻塞不影响其他 KLT
多核利用无法利用多核并行可以利用多核并行
调度粒度进程粒度线程粒度

内核级线程才是处理机分配的单位。n 个 ULT 可以映射到 m 个 KLT 上（n >= m）。

# 多线程模型

三种多线程模型定义了 ULT 与 KLT 的映射关系：

# 多对一模型

n 个 ULT 映射到 1 个 KLT。

优点缺点
用户空间开销小，线程切换效率高一个 ULT 阻塞，所有线程阻塞
调度由应用程序控制，灵活无法利用多核并行，并发度不高

适合场景：需要大量线程但不关心并行度的应用，或运行在单核系统上的应用。

# 一对一模型

n 个 ULT 映射到 n 个 KLT（每个 ULT 对应一个 KLT）。

优点缺点
线程阻塞不影响其他线程每创建一个 ULT 都要创建一个 KLT，内核开销大
并发能力强，充分利用多核线程数量受限，占用系统资源多

适合场景：需要高并发能力的应用，如高负载服务器。Linux 和 Windows 的线程模型即为一对一。

# 多对多模型

n 个 ULT 映射到 m 个 KLT（n >= m），中和了上述两种方案的优缺点。

优点缺点
并发度高，可利用多核实现复杂
线程切换开销可控需要内核和用户空间的协调
不会因一个线程阻塞影响全进程—

# 三种模型总结

模型映射并发度阻塞影响开销适用场景
多对一n:1低全部阻塞低不支持多核的旧系统
一对一n:n高只阻塞自己高Linux / Windows 线程模型
多对多n:m高可控中等Solaris（早期）、现代混合实现
