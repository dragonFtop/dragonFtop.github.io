# 数字签名作用

数字签名作用方面：鉴别、数据完整性、抗抵赖性
主要作用： 将消息和拥有消息的实体可信地联系起来。
数字签名分类：

# 何为数字签名

数字签名是只有信息的发送者才能产生的，别人无法伪造的一段数字串，它是对信息发送者发送信息真实性的一个有效证明。

# 数字签名的特性

  1. 签名不可伪造：除了合法的签名者之外，任何其他人伪造其签名是困难的。
  2. 签名不可抵赖：签名者事后不能否认自己的签名。
  3. 签名是可信的：任何人都可以验证签名的有效性。
  4. 签名是不可复制的：对一个消息的签名不能通过复制变为另一个消息的签名。
  5. 签名的消息是不可更改的：经签名的消息不能被篡改。一旦签名的消息被篡改，则任何人都可以发现消息与签名之间的不一致性。

# 数字签名的设计要求

  • 签名必须依赖于被签名消息的比特模式 (签名与消息应该是一个不可分割的整体)；
  • 签名必须使用某些对发送者是唯一的信息，以防止伪造和抵赖；
  • 数字签名的产生、识别和验证应该相对容易；
  • 伪造一个数字签名在计算上是不可行的；(无论是通过对已有的数字签名来构造新消息，还是对给定的消息伪造一个数字签名)

# 基于公钥密码的数字签名

公钥密码中的使用私钥加密消息可以做到数字签名的效果，例如 RSA 用私钥签名，用公钥验证。
但是直接使用 RSA 加密消息进行数字签名存在选择明文攻击：

这种问题的产生根源是：数字签名内容和消息内容为一体，验证数字签名时暴露了明文。
解决方案：将签名信息和报文分离，形成独立的签名块；

# 数字签名的执行方式

数字签名执行方式分为两类：

  1. 直接数字签名：只有通信双方参与，并假定双方有共享的秘密密钥，或接收一方知道发送方的公开密钥。
    存在问题：
    • 发送方抵赖
    • 有效性依赖于发送方密钥的安全性
  2. 具有仲裁的数字签名：在通信双方的基础上引入了第三方仲裁者参与。
    • 应用对称加密，仲裁者可以看到消息内容
    • 应用对称加密，仲裁者不能看到消息内容
    • 应用公钥加密，仲裁者不能看到消息内容

# 特殊签名方案

# 不可否认签名

必须由签名者参加的情况下才能进行签名验证，所以签名验证成功后签名者不可以否认。
不可否认签名方案包括三个部分：签名算法、验证算法、否认协议

# 盲数字签名

签名者不知道数据内容而签名。
有两个很显著的特点：

  1. 消息内容对签名者不可见
  2. 在签名被接受者公开后，签名者不能追踪签名

也就是匿名、无记名投票，其基本结构：

# 群签名

群签名允许群体中的任意成员代表群体进行签名，验证者可以验证签名是由该群体中的成员所签，但无法确定具体是哪个成员，发生争议时可由群体管理员揭示签名者身份。

# 国密 SM2 算法

SM2 是椭圆曲线公钥密码算法
其中包括：

  • 数字签名算法
  • 密钥交换协议
  • 公钥加密算法
