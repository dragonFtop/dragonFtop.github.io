# 【动态规划】单词拆分

# 题目

给你一个字符串 s 和一个字符串列表 wordDict 作为字典。如果可以利用字典中出现的一个或多个单词拼接出 s 则返回 true。

注意：不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

提示：
1 <= s.length <= 300
1 <= wordDict.length <= 1000
1 <= wordDict[i].length <= 20
s 和 wordDict[i] 仅由小写英文字母组成
wordDict 中的所有字符串 互不相同

# 解题

思路：动态规划。将大问题拆解为小问题，判断整个字符串是否可以用单词拼接，其实可以将整个字符串分成前后两个部分，如果前一部分可以用单词表中单词拼接，而后一个部分也在单词表中，则说明整个字符串可以用单词拼接。
对于一个长度为 i 的字符串 s，记断点为 j (区分前 i-j 部分，j 部分为后一部分), 如果 s [0..j-1] 可以用单词拼接，并且 s [j..i-1] 也在单词表中，则说明 s [0..i-1] 可以用单词拼接。
用 dp [i]，(bool) 表示 s [0..i-1] 是否可以用单词拼接，i 遍历子串，对于每个 i 都去进行上述可拼写性的检测，指导最后得到 dp [n]，若 dp [n] 为 true，则说明整个字符串可以用单词拼接。

# 反思

以上这个写法中我发现 i 会遍历很多无用的值，它会遍历不可能为 true（即不可能会单词间断点）的值，而且内部的 j 的循环起点也不对，如果 s [j..i-1] 的长度大于 wordSet 中的单词的最长长度，则 s [j..i-1] 不可能在 wordSet 中，所以 j 的循环起点应该从 wordSet 中的单词的最长长度开始。
