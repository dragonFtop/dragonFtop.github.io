# 传输层作业答案及详解

# 一、单选题（共 54 题，54 分）

# 第 1 题

题目：
A transport-layer protocol provides for logical communication between ____.

A. Application processes
B. Hosts
C. Routers
D. End systems

参考答案：A

详细解析：

  • 传输层协议（TCP/UDP）为运行在不同主机上的 ** 应用进程（Application processes）** 之间提供逻辑通信，而非物理通信。
  • 主机（Hosts）和端系统（End systems）是物理设备层面的概念，进程是通信的真正端点。
  • 路由器（Routers）工作在网络层，传输层协议不运行在路由器上。

# 第 2 题

题目：
Transport-layer protocols run in ____.

A. Servers
B. Clients
C. Routers
D. End systems

参考答案：D

详细解析：

  • 传输层协议运行在 ** 端系统（End systems）** 上，即通信的发送方和接收方主机。
  • 路由器（Routers）仅处理网络层及以下的数据包转发，不涉及传输层处理。
  • 服务器（Servers）和客户端（Clients）都运行在端系统上，但传输层协议不局限于某一角色。

# 第 3 题

题目：
In transport layer, the send side breaks application messages into ____, passes to network layer.

A. Frames
B. Segments
C. Data-grams
D. bit streams

参考答案：B

详细解析：

  • 传输层的发送方将应用层消息分割成段（Segments），添加传输层头部后交给网络层。
  • 帧（Frames）是链路层的数据单元。
  • 数据报（Datagrams）是网络层的数据单元。
  • 比特流（bit streams）是物理层的概念。

# 第 4 题

题目：
The receive side of transport layer reassembles segments into messages, passes to ____layer.

A. Application
B. Networking
C. Physical
D. MAC

参考答案：A

详细解析：

  • 传输层接收方将段（Segments）重组为消息（Messages），然后交付给应用层。
  • 帧（Frames）是链路层的处理单元，数据报（Datagrams）是网络层的处理单元。

# 第 5 题

题目：
Services provided by transport layer include ____.

A. HTTP and FTP
B. TCP and IP
C. TCP and UDP
D. SMTP

参考答案：C

详细解析：

  • 传输层提供的核心服务包括：可靠数据传输、拥塞控制、流量控制、多路复用 / 分解。
  • 吞吐量保证和定时保证不是传输层必须提供的服务，这些通常由应用层或 QoS 机制处理。

# 第 6 题

题目：
Which of the following services is not provided by TCP?

A. Reliable data transfers and flow controls
B. Congestion controls
C. In-order data transfers
D. Delay guarantees and bandwidth guarantees

参考答案：D

详细解析：

  • TCP 提供的服务：可靠数据传输、拥塞控制、流量控制、面向连接。
  • TCP 不提供的服务：保证最小带宽、定时保证。加密由 TLS/SSL 等上层协议提供。

# 第 7 题

题目：
These two minimal transport-layer services ____ and ____ are the only two services that UDP provides

A. congestion control, reliable data transfer
B. flow control, congestion control
C. process-to-process data delivery, error checking
D. in-order data transfer, error checking

参考答案：C

详细解析：

  • 传输层最基础的两个服务是：多路复用 / 多路分解和错误检测。
  • 这两个是 UDP 也提供的最基本服务（UDP 仅有这两项服务和校验和）。
  • 可靠数据传输、拥塞控制等是 TCP 提供的增强服务。

# 第 8 题

题目：
UDP offers which of the following benefits relative to TCP?

A. UDP consumes fewer computer resources by not maintaining connection state
B. UDP supports a self-regulating “throttle” feature that prevents network saturation
C. UDP guarantees that Individual packets of a transmission will arrive “in order”
D. None of the above

参考答案：A

详细解析：

  • UDP 相对于 TCP 的主要优势：更小的头部开销（UDP 头部仅 8 字节，TCP 至少 20 字节）。
  • 无需连接建立（无三次握手延迟），无拥塞控制（发送速率不受限制，适合实时应用）。

# 第 9 题

题目：
The job of gathering data chunks, encapsulating each data chunk with header information to create segments and passing the segments to the network is called ____.

A. demultiplexing
B. multiplexing
C. forwarding
D. routing

参考答案：B

详细解析：

  • 多路复用（Multiplexing）：发送方从不同套接字收集数据块，为每个数据块封装传输层头部信息，然后将段传递到网络层。
  • 多路分解（De-multiplexing）是接收方的对应操作。

# 第 10 题

题目：
This job of delivering the data in a transport-layer segment to the correct socket is called ____.

A. multiplexing
B. demultiplexing
C. forwarding
D. Routing

参考答案：B

详细解析：

  • 多路分解（De-multiplexing）：接收方传输层将收到的段中的数据交付到正确的套接字。
  • 通过检查段的源端口号和目的端口号来实现。

# 第 11 题

题目：
The following four descriptions about multiplexing and de-multiplexing, which one is correct?

A. UDP socket is identified by a two-tuples consisting of a source port number and a destination port number.
B. If two UDP segment have different source port number, they may be directed to the same destination process.
C. If two TCP segments with different source port number, they may be directed to the same destination process.
D. If two TCP segments with same destination IP address and destination port number, they must be the same TCP connection.

参考答案：B

详细解析：

  • B 正确：多路分解在接收端将段交付到正确的套接字，UDP 仅用目的端口号，TCP 用四元组（源 IP、源端口、目的 IP、目的端口）。
  • A 描述的是多路复用的功能，不完全准确。

# 第 12 题

题目：
The port numbers ranging from ____to ____ are called well-known port number and are restricted.

A. 0, 127
B. 0,255
C. 0, 1023
D. 0, 65535

参考答案：C

详细解析：

  • 知名端口（well-known port numbers）的范围是 0~1023。
  • 这些端口号保留给系统级服务，如 HTTP (80)、DNS (53)、FTP (21)、SSH (22) 等。
  • 102449151 为注册端口，4915265535 为动态 / 私有端口。

# 第 13 题

题目：
The UDP header has only four fields, they are____.

A. Source port number, destination port number, length and checksum
B. Source port number, destination port number, source IP and destination IP
C. source IP, destination IP, source MAC address and destination MAC address
D. source IP, destination IP, sequence number and ACK sequence number

参考答案：A

详细解析：

  • UDP 头部仅包含 4 个字段，每个 16 位（2 字节），共 8 字节：源端口号、目的端口号、长度、校验和。
  • UDP 没有序列号、确认号等字段。

# 第 14 题

题目：
The field of Length in UDP segment specifies the length of ____.

A. the UDP segment, not including the header
B. the UDP segment, including the header
C. the UDP segment’s header
D. the Length field

参考答案：B

详细解析：

  • UDP 头部中的 ** 长度字段（Length）指定 UDP 段（包括头部和数据）** 的总长度。
  • 最小值为 8 字节（仅有头部，无数据），最大为 65535 字节。

# 第 15 题

题目：
Which of the following about TCP connection is not correct?

A. It is a broadcast connection
B. It is a point-to-point connection
C. It is a pipelined connection
D. It is a full duplex connection

参考答案：A

详细解析：

  • D 错误：TCP 连接 ** 确实是端到端（end-to-end）** 的，在发送端系统和接收端系统之间建立。
  • A 正确：TCP 全双工。B 正确：TCP 点对点。C 正确：TCP 面向连接（三次握手）。

# 第 16 题

题目：
In the following four descriptions about MSS and MTU, which one is not correct?

A. The MSS is the maximum amount of application-layer data in the segment
B. The MSS is the maximum size of the TCP segment including headers
C. The MSS is typically set by MTU
D. The MTU means the largest link-layer frame

参考答案：B

详细解析：

  • MSS（Maximum Segment Size）：TCP 段中应用层数据的最大量，不包括 TCP 头部。
  • MTU（Maximum Transmission Unit）：链路层帧能承载的最大数据量。
  • MSS = MTU - TCP 头部 - IP 头部（典型值 1460 字节）。
  • B 选项正确描述了 MSS 和 MTU 的关系。

# 第 17 题

题目：
____ is the byte stream number of first byte in the segment.

A. Checksum
B. ACK number
C. port number
D. Sequence number

参考答案：D

详细解析：

  • TCP 头部中的序号字段（Sequence Number）是段中第一个字节的字节流编号。
  • TCP 将数据视为字节流，每个字节有一个序号。
  • 确认号（Acknowledgment Number）是期望收到的下一个字节的序号。

# 第 18 题

题目：
____ is the byte sequence numbers of next byte expected from other side.

A. Checksum
B. port number
C. Sequence number
D. ACK number

参考答案：D

详细解析：

  • TCP 头部中的确认号字段（Acknowledgment Number）是期望从对方收到的下一个字节的序号。
  • 例如：收到序号 38、数据 4 字节的段后，确认号为 38+4=42。

# 第 19 题

题目：
A port number in UPD is ______ bits long.

A. 8
B. 16
C. 32
D. none of the above

参考答案：B

详细解析：

  • UDP 中的端口号是 16 位长，范围 0~65535。
  • TCP 中的端口号同样是 16 位，这也是端口号最大值 65535 的原因。

# 第 20 题

题目：
There are two 16-bit integers: 1110 0110 0110 0110, 1101 0101 0101 0101. Their checksum is____.

A. 1011 1011 1011 1100
B. 1111 1111 1111 1111
C. 0100 0100 0100 0011
D. 1000 0000 0000 0000

参考答案：C

详细解析：

  • 校验和计算：1110 0110 0110 0110 + 1101 0101 0101 0101 = 1 1011 1011 1011 1011
  • 溢出回卷：1011 1011 1011 1011 + 1 = 1011 1011 1011 1100
  • 取反码（one's complement）：0100 0100 0100 0011
  • 因此校验和为 0100 0100 0100 0011。

# 第 21 题

题目：
Suppose host A sends host B one TCP segment with sequence number 418, acknowledgement number 571, and 4 bytes of data. Then the sequence number in the acknowledgement to this segment is _________

A. 422
B. 418
C. 571
D. 575

参考答案：C

详细解析：

  • 主机 A 发送：序号 = 418，数据 = 4 字节，覆盖字节 418~421。
  • 主机 B 确认：期望下一个字节 = 418+4=422，确认号 = 422。
  • B 同时发数据：序号 = 571，数据 = 2 字节（字节 571,572）。
  • B 的段：序号 = 571，确认号 = 422（捎带确认）。

# 第 22 题

题目：
Because TCP only acknowledges bytes up to the first missing byte in the stream, TCP is said to provide____.

A. Selective acknowledgements
B. 3 duplicate ACKs
C. positive ACKs
D. Cumulative acknowledgements

参考答案：D

详细解析：

  • TCP 只确认到第一个丢失字节之前的所有字节，这种确认方式称为累积确认（Cumulative ACKs）。
  • 好处是确认丢失不会导致不必要重传，缺点是可能在一个段丢失时需要重传多个段。

# 第 23 题

题目：
If we define N to be the window size, base to be the sequence number of the oldest unacknowledged packet, and next-seq-num to be the smallest unused sequence number, then the interval [nextseqnum,base+N-1] corresponds to packet that ____.

A. can be sent immediately
B. have already been transmitted and acknowledged
C. cannot be used
D. have been sent but not yet acknowledged

参考答案：A

详细解析：

  • 在 GBN（Go-Back-N）协议中，已发送但未确认的段序号范围为：[base, nextseqnum-1]。
  • base = 最早未确认段的序号，nextseqnum = 下一个要发送的序号。
  • 窗口大小 N = nextseqnum - base。

# 第 24 题

题目：
Which of the following is not a pipelining protocol?

A. Rdt1.0
B. Go-Back-N
C. Selective repeat
D. TCP

参考答案：A

详细解析：

  • TCP 本身不是一种独立的流水线协议类型，它结合了 GBN 和 SR 的特点。
  • 停止 - 等待协议不是流水线协议（每次只发一个段）。
  • UDP 也没有确认重传机制，不是流水线协议。

# 第 25 题

题目：
What is the main difference between stop-and-wait and pipelined reliable data transfer protocol?

A. The pipelined protocol uses the NAK packets, whereas in the stop-and-wait protocol senders always wait for ACK packets.
B. With the pipelined protocol, the sender can send several packets in row, whereas in the stop-and-wait protocol the sender cannot send the packets in row.
C. With the pipelined protocol, the receiver must send one ACK for several packets (cumulative ACK), whereas in the stop-and-wait protocol the receiver can not send the cumulative ACK.
D. The pipelined protocol uses timeouts, whereas the stop-and-wait protocol does not use the timeout.

参考答案：B

详细解析：

  • B 正确：流水线协议允许多个未确认的段同时存在，这是其核心特征。
  • 停止 - 等待协议一次只发一个段，效率低；但停止 - 等待也需要 ACK 确认。

# 第 26 题

题目：
Fast retransmit means in the case that ____ duplicate ACK-s are received, the TCP sender resend segment before timer expires.

A. 3
B. 4
C. 5
D. 6

参考答案：A

详细解析：

  • 快速重传（Fast Retransmit）：当发送方收到 3 个重复 ACK 时，在超时前就重传丢失的段。
  • 这是 TCP 的优化机制，3 个重复 ACK 强烈暗示该段已丢失。

# 第 27 题

题目：
____ is a speeding-matching service---matching the rate which the sender is sending against the rate at which the receiving application is reading.

A. congestion control
B. flow control
C. sliding-window control
D. variable control

参考答案：B

详细解析：

  • 流量控制（Flow control）：匹配发送方速率与接收方读取速率的速度匹配服务。
  • TCP 通过 ** 接收窗口（rwnd）** 实现流量控制。
  • 拥塞控制是匹配发送速率与网络能力。

# 第 28 题

题目：
TCP provides flow control by having the sender maintain a variable called the ____.

A. Receive window
B. Congestion window
C. Sliding window
D. Buffer

参考答案：A

详细解析：

  • TCP 通过让发送方维护 ** 接收窗口（receive window /rwnd）** 来实现流量控制。
  • 接收方在 TCP 头部的窗口字段告知 rwnd 值。
  • 发送方确保：LastByteSent - LastByteAcked ≤ rwnd。

# 第 29 题

题目：
Provided RcvBuffer = 20, LastByteRcvd = 20, LastByteRead = 15, then RcvWindow=____.

A. 20
B. 15
C. 5
D. 10

参考答案：B

详细解析：

  • 计算：rwnd = RcvBuffer - (LastByteRcvd - LastByteRead)
    = 20 - (20 - 12) = 20 - 8 = 12。
  • 发送方最多还能发送 12 字节数据。

# 第 30 题

题目：
In the following four descriptions about TCP connection management, which one is not correct?

A. Either of the two processes participating in a TCP connection can end the connection
B. If the FIN bit is set to 1, it means that it wants to close the connection
C. In the first two step of the three-way handshake, the client and server randomly choose an initial sequence number
D. In the three segments of the three-way handshake, the SYN bit must be set to 1

参考答案：D

详细解析：

  • D 错误：连接建立和释放过程中确实涉及序列号同步。
  • 三次握手期间交换 ISN（初始序列号），序列号同步是连接建立的核心目的。
  • A、B、C 描述均正确。

# 第 31 题

题目：
How does TCP sender perceive congestion?

A. Through a timeout event
B. Through a receiving duplicate ACK-s event
C. Both A and B
D. Either A or B

参考答案：D

详细解析：

  • TCP 发送方感知拥塞的方式：超时和 3 个重复 ACK。
  • 超时表示段和 / 或 ACK 丢失，3 个重复 ACK 表示某些段丢失但网络仍可交付数据。

# 第 32 题

题目：
There are two states in TCP congestion control, which are ____.

A. slow start and congestion avoidance
B. safe start and congestion avoidance
C. slow start and congestion abandon
D. safe start and congestion abandon

参考答案：A

详细解析：

  • TCP 拥塞控制的两个主要状态：慢启动（Slow Start）和拥塞避免（Congestion Avoidance）。
  • 快速恢复（Fast Recovery）是第三个可选状态，与快速重传配合使用。

# 第 33 题

题目：
In TCP, the timeout interval is a function of:

A. estimated RTT at the sender
B. MSS and the overhead of a segment
C. the size of buffer at the receiver
D. the size of sending window

参考答案：A

详细解析：

  • TCP 超时间隔 = EstimatedRTT + 4 × DevRTT（EstimatedRTT 和 DevRTT 的函数）。
  • 这样设计考虑了 RTT 的波动，提供安全余量。

# 第 34 题

题目：
In a TCP connection, there is timeout event when the value of threshold is 32 and the size of congestion window is 16. According to the TCP congestion control policy, the new value of threshold and the new size of congestion window should be ________, respectively.

A. 16, 8
B. 24, 8
C. 8, 1
D. 16, 1

参考答案：C

详细解析：

  • 超时事件发生后：ssthresh = cwnd / 2（阈值减半），cwnd = 1 MSS（窗口重置），进入慢启动。
  • 区分：3 个重复 ACK 时是 cwnd = ssthresh + 3MSS（快速恢复），而非重置为 1。

# 第 35 题

题目：
若甲向乙发起一个 TCP 连接，最大段长 MSS=1 KB, RTT=5 ms, 乙开辟的接收缓存为 64 KB, 则甲从连接建立成功至发送窗口达到 32 KB, 需经过的时间至少是____

A. 25 ms
B. 30 ms
C. 160 ms
D. 165 ms

参考答案：A

详细解析：

  • 条件：MSS=1KB, RTT=5ms, 接收缓存 = 64KB。
  • 最大吞吐量 = 64KB / 5ms = 12.8 MB/s。
  • 慢启动：1→2→4→8→16→32→64KB，需 6 个 RTT = 30ms 达到满速。
  • 第一个 RTT 发送 1KB，第二个 2KB，以此类推。

# 第 36 题

题目：
主机甲和主机乙新建一个 TCP 连接，甲的拥塞控制初始阈值为 32KB, 甲向乙始终以 MSS=1KB 大小的段发送数据，并一直有数据发送；乙为该连接分配 16KB 接收缓存，并对每个数据段进行确认，忽略段传输延迟。若乙收到的数据全部存入缓存，不被取走，则甲从连接建立成功时刻起，未发送超时的情况下，经过 4 个 RTT 后，甲的发送窗口是____。

A. 1KB
B. 8KB
C. 16KB
D. 32KB

参考答案：A

详细解析：

  • 初始 cwnd=1MSS，ssthresh=32KB。
  • 慢启动指数增长：1→2→4→8→16→32（前 5 个 RTT 发送 31 段）。
  • 第 6 个 RTT 达到阈值，发送 32 段，进入拥塞避免。

# 第 37 题

题目：
主机甲通过 128kbps 卫星链路，采用滑动窗口协议向主机乙发送数据，链路单向传播延迟为 250ms, 帧长为 1000 字节。不考虑确认帧的开销，为使链路利用率不小于 80%, 帧序号的比特数至少是_____。

A. 3
B. 4
C. 7
D. 8

参考答案：B

详细解析：

  • GBN 协议，128kbps，单向延迟 250ms，帧长 1000 字节。
  • 发送一帧时间 = 8000/128000 = 62.5ms，RTT = 500ms。
  • 发送窗口 ≥ 500/62.5 + 1 ≈ 9 帧。
  • 序列号位数：2^n ≥ 发送窗口 + 1，n=4 时 16≥9 满足。
  • 信道利用率 100% 需要窗口≥9 帧。

# 第 38 题

题目：
一个 TCP 连接总是以 1KB 的最大段发送 TCP 段，发送方有足够多的数据要发送。当拥塞窗口为 16KB 时发生了超时，如果接下来的 4 个 RTT (往返时间) 时间内的 TCP 段的传输都是成功的，那么当第 4 个 RTT 时间内发送的所有 TCP 段都得到肯定应答时，拥塞窗口大小是____。

A. 7KB
B. 8KB
C. 9KB
D. 16KB

参考答案：C

详细解析：

# 一、传输层基础与服务模型（第 1~7 题）

# 1. 传输层功能与地位

  • 传输层为应用进程（Application processes）之间提供逻辑通信。
  • 传输层协议运行在 ** 端系统（End systems）** 上，路由器不涉及传输层处理。
  • 发送方将应用消息分割成 ** 段（Segments）交给网络层；接收方将段重组为消息（Messages）** 交给应用层。

# 2. 传输层核心服务

  • 可靠数据传输、拥塞控制、流量控制、多路复用 / 多路分解。
  • 传输层最基础的两种服务：多路复用 / 多路分解 + 错误检测（UDP 也提供这两项）。
  • TCP 不提供的服务：带宽保证、定时保证、加密。

# 3. TCP vs UDP

# 二、多路复用 / 多路分解（第 8~12 题）

# 1. 核心概念

  • 多路复用（Multiplexing）：发送方从多个套接字收集数据，封装传输层头部后交给网络层。
  • 多路分解（De-multiplexing）：接收方根据端口号将数据交付到正确的套接字。
  • UDP 仅用目的端口号进行分解；TCP 用四元组（源 IP、源端口、目的 IP、目的端口）。

# 2. 端口号

  • 知名端口（Well-known ports）：0~1023，如 HTTP (80)、DNS (53)、FTP (21)、SMTP (25)。
  • 注册端口：102449151；动态 / 私有端口：4915265535。
  • UDP 端口号 16 位（TCP 同样为 16 位）。

# 三、UDP 协议（第 1314、1920、44、46~47 题）

# 1. UDP 头部结构（共 8 字节）

  • 源端口号（16 位）
  • 目的端口号（16 位）
  • 长度（16 位）：UDP 段总长度（头部 + 数据），最小 8 字节。
  • 校验和（16 位）：可选（IPv4），IPv6 中必须计算。

# 2. UDP 校验和

  • 计算范围：伪头部 + UDP 头部 + UDP 数据。
  • 伪头部含：源 IP、目的 IP、协议号、UDP 长度。
  • 计算方式：16 位字相加，溢出回卷，取反码（one's complement）。

# 3. UDP 特性

  • 无连接、不可靠，不保证数据交付。
  • 有差错检测但无差错恢复。
  • 分用仅依赖目的端口号。

# 四、TCP 可靠传输（第 15~26、37、42 题）

# 1. TCP 连接特性

  • 全双工（full-duplex）：数据可同时双向传输。
  • 点对点（point-to-point）：单个发送方与单个接收方。
  • 面向连接（connection-oriented）：需三次握手建立。

# 2. MSS 与 MTU

  • MTU（Maximum Transmission Unit）：链路层帧最大载荷（通常 1500 字节）。
  • MSS（Maximum Segment Size）：TCP 段中应用数据的最大量，不包括 TCP/IP 头部。
  • MSS = MTU - TCP 头部 - IP 头部（典型值 1460 字节）。

# 3. TCP 序号与确认号

  • 序号（Sequence Number）：段中第一个字节的字节流编号。
  • 确认号（Acknowledgment Number）：期望收到的下一个字节的序号。
  • TCP 采用累积确认（Cumulative ACKs）：只确认到第一个丢失字节之前的所有字节。
  • 累计确认优点：ACK 丢失不导致不必要重传；缺点：可能一次重传多个段。

# 4. 流水线协议

  • 停止 - 等待（Stop-and-Wait）：一次只发一个段，等 ACK 后再发下一个，效率低。
  • Go-Back-N（GBN）：窗口大小为 N，已发送未确认范围 = [base, nextseqnum-1]。
  • Selective Repeat（SR）：只重传丢失的段，接收方需独立确认每个段。
  • TCP 结合了 GBN 和 SR 的特点（累积确认 + 选择性确认 SACK）。

# 5. 快速重传

  • 收到 3 个重复 ACK 时，不等超时就重传丢失的段。

# 五、流量控制（第 2729、52、5657 题）

# 1. 核心机制

  • 流量控制是速度匹配服务：匹配发送速率与接收方读取速率。
  • TCP 通过 ** 接收窗口（rwnd / RcvWindow）** 实现。
  • 发送方维护： LastByteSent - LastByteAcked ≤ rwnd 。

# 2. rwnd 计算

  • rwnd = RcvBuffer - (LastByteRcvd - LastByteRead)
  • rwnd 在整个连接期间动态变化，接收方实时调整。
  • 每个 TCP 段头部携带当前 rwnd 值。
  • 即使接收方无数据发送，仍需发送 ACK 确认段。

# 六、TCP 连接管理（第 30、41、48、71、74 题）

# 1. 三次握手（建立连接）

  • 第一次：Client → Server：SYN=1, ACK=0, seq=client_ISN
  • 第二次：Server → Client：SYN=1, ACK=1, seq=server_ISN, ack=client_ISN+1
  • 第三次：Client → Server：SYN=0, ACK=1, seq=client_ISN+1, ack=server_ISN+1
  • SYN 段虽无数据，但消耗一个序号。

# 2. 四次挥手（释放连接）

  • FIN=1 表示该方向不再发送数据（半关闭状态）。
  • 最短释放时间 = 2 RTT（双向各一次 FIN+ACK）。

# 七、拥塞控制（第 3134、4955、6062、7273、75 题）

# 1. 拥塞感知

  • TCP 发送方通过超时（Timeout）和 3 个重复 ACK 感知拥塞。
  • 超时 = 段和 / 或 ACK 在网络中丢失。
  • 3 个重复 ACK = 部分段丢失但网络仍可交付数据。

# 2. 超时间隔计算

  • TimeoutInterval = EstimatedRTT + 4 × DevRTT
  • 不是直接等于测量的 RTT，考虑了 RTT 波动。

# 3. 拥塞控制三大状态

状态cwnd 增长方式触发条件
慢启动（Slow Start）每 RTT 翻倍（指数增长）连接开始或超时后
拥塞避免（Congestion Avoidance）每 RTT +1 MSS（线性增长）cwnd ≥ ssthresh
快速恢复（Fast Recovery）cwnd = ssthresh + 3，然后每重复 ACK +13 个重复 ACK

# 4. 拥塞事件处理

事件ssthreshcwnd后续
超时（Timeout）cwnd / 21 MSS慢启动
3 个重复 ACK（快速重传）cwnd / 2ssthresh + 3快速恢复

# 5. AIMD 原则

  • 加性增（Additive Increase）：拥塞避免阶段每 RTT cwnd+1 MSS。
  • 乘性减（Multiplicative Decrease）：拥塞事件后 cwnd 减半。
  • 拥塞窗口变化呈锯齿状。

# 6. 常见错误区分

  • 拥塞控制降低速率是因为网络拥塞（超时 / 重复 ACK），不是接收方过载。
  • 接收方过载由流量控制（rwnd）处理，两者是不同层面的机制。
  • TCP 全双工，双向数据只需一个 TCP 连接。

# 7. TCP Tahoe vs Reno

  • Tahoe：任何丢包都 cwnd=1，重新慢启动。
  • Reno：3 个重复 ACK 用快速恢复（cwnd 不减到 1），超时仍用慢启动。

# 八、综合计算（第 3540、45、4951、5354、6569、74 题）

# 1. 吞吐量与时间计算

  • 最大吞吐量 ≈ rwnd / RTT
  • 慢启动阶段：第 n 个 RTT 发送 2^(n-1) 个 MSS（cwnd 从 1 开始）
  • 发送窗口 = min (cwnd, rwnd)

# 2. GBN 窗口与序列号

  • 发送窗口 N 需满足： N ≥ RTT / 发送一帧时间 + 1 （达到 100% 利用率）
  • 序列号位数 n 需满足： 2^n ≥ N + 1 （GBN）

# 3. 典型计算模式

  • 慢启动 + 拥塞避免累计传输量：先指数增长到 ssthresh，再线性递增。
  • 超时后窗口行为：ssthresh = cwnd/2，cwnd = 1，重新慢启动。
  • 3 个重复 ACK 后窗口行为：ssthresh = cwnd/2，cwnd = ssthresh + 3，进入快速恢复。

# 九、高速网络与 TCP 改进（第 65 题）

# 问题

  • 64KB 窗口太小，无法填满高 BDP 管道。
  • 32 位序列号在高速下快速耗尽回绕。

# 解决方案

  • 窗口缩放选项（Window Scaling）：用乘数扩展窗口。
  • BBR 拥塞控制算法、TCP 快速打开、选择性确认（SACK）、多路径 TCP。

# 十、高频易混点速记

  1. 传输层数据单元：Segment（段），网络层：Datagram，链路层：Frame。
  2. UDP 头部 4 字段：源端口、目的端口、长度、校验和（共 8 字节）。
  3. 端口号 16 位：065535，知名端口 01023。
  4. MSS ≠ MTU：MSS = MTU - IP 头 - TCP 头。
  5. 序号 = 第一个字节编号，确认号 = 期望的下一个字节。
  6. TCP 累积确认：只确认连续收到的数据。
  7. rwnd = 接收缓存 - (已收 - 已读)，会动态变化。
  8. cwnd = 1 仅在超时时发生，3 个重复 ACK 时 cwnd = ssthresh + 3。
  9. 慢启动 = 指数增，拥塞避免 = 线性增。
  10. 流量控制 ≠ 拥塞控制：前者匹配接收方，后者匹配网络。
  11. 三次握手：SYN (1,0) → SYN+ACK (1,1) → ACK (0,1)。
  12. FIN = 半关闭：只关闭本方向，对方方向仍可使用。
  13. TCP Tahoe：丢包→cwnd=1；TCP Reno：3 dup ACK→快速恢复，超时→cwnd=1。
  14. GBN 窗口：[base, base+N-1]；发送中范围：[base, nextseqnum-1]。
  15. 一个 TCP 连接全双工，双方可同时收发，不需要两个连接。
