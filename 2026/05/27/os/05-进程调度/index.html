# 处理器调度

# 为什么需要设计进程 (处理器) 调度？

批处理任务业务逻辑：
任务存在硬盘中 -> 任务进入内存形成进程 -> 进程在 CPU 中执行。
这样的任务执行为什么需要调度算法？

  1. 资源竞争：任务很多，资源（ 内存和CPU ）有限，任务无法 同一时间 全部加载到内存，进程无法 同一时间 全部放入 CPU 执行。
  2. 执行顺序：同一批任务，不同的准入和执行顺序有不同性能。

进程调度设计的考虑因素：何时进行调度？如何进行调度？
注意：处理器的调度分为长程、中程、短程三个层次，常说的 "进程调度" 一般指短程调度。

# 处理器调度的类型

  1. 长程调度：控制哪些任务可以被准入（内存）
  2. 中程调度：控制被准入的任务（被划分为了进程），将内存中被挂起或阻塞的进程换出到磁盘
  3. 短程调度：分发器，分发 CPU 资源分发给进程

# 决策模式

  1. 抢占式：进程可以插队。一个进程可以在另一个进程未执行完的时候被调度执行。
  2. 非抢占式：进程必须按先来后到的顺序完成。一个进程必须在当前进程执行完成后才能被调度执行。

# 性能指标

  1. 用户层面
  2. 系统层面

对于单个任务 / 进程：

  • 周转时间：从任务提交到任务最终执行完成的时间。
  • 响应时间：从任务提交到任务开始响应的时间。

    对于一批任务的指标：
  • 吞吐：单位时间内完成的任务数量。
  • 时限：硬时限，任务必须在规定时间内完成，否则认为失败，以时限内完成数 / 总任务数为量化指标。软时限，任务尽量在规定时间内完成，一般以错过截止时间的惩罚作为定量指标。

对于系统的指标

  • 资源利用率：单位时间内处于工作状态的时间比例。

# 公平性

调度算法不能偏向于特定用户 / 任务 / 进程。
处理器调度的公平性体现在避免引入新的进程饥饿。
解释：如果一个进程的优先级很低，那么如果把其放入内存后，其一直不被执行，会造成饥饿现象。可能引发系统效率低、死锁等错误。

# 不同优先级调度

一类基于优先级的算法：前提假设为不同任务对于用户需求和系统性能的效果也不同。
仅介绍 FIFO 的优先级调度。

  1. 系统维护多个优先级等级，每一级对应一个队列
  2. 每个处于准备（ready）状态的进程放入对应的 FIFO 队列中
  3. 系统从优先级最高的非空队列选择进程进行调度

    特点：
  • 既可以是抢占式也可以是非抢占式
  • 实现了带权的周转时间 / 响应时间优化
  • 没有对吞吐和时限进行优化
  • 低优先级进程可能饥饿（公平性问题）

解决公平性问题的方案：老化机制（Aging），将长期闲置的进程提高优先级。

# 先来先服务 (FCFS)

先来先服务：First-Come First-Serve，FCFS。谁先来谁就占用资源。

问题：

  • 严重：所有的进程必须按顺序被响应，没有优化平均周转时间和系统吞吐。
  • 其次：不会出现饥饿，但是这种方式偏袒了运行时间较长的和在 CPU 中运行受限的进程。

# 最短进程优先 (SPN)

最短进程优先：Shortest Process Next，SPN。总是选择准备队列中所需服务时间最小的进程。

# 轮转 (RR)

轮转：Round Robin，RR。主要目标是降低响应时间而非周转时间。
轮转法实现：为待执行的进程分配等长 CPU 使用时间，称为时间片（quantum）。分配时间用完或进程执行结束则中止执行。

性能取决于： 进程执行时间 和 切片大小
存在问题：时间片大小

  • 时间分片过大，调度偏向长进程，短进程的响应时间过大
  • 时间分片过小会导致频繁上下文切换，且可能导致完成任务数量降低，吞吐下降

# 最短剩余时间优先 (SRT)

最短剩余时间优先：Shortest Remaining Time，SRT。
触发时机：在每次新进程被准入或者当前进程释放 CPU 触发。
被触发后，选择当前情况下剩余执行时间最短的进程执行。

优化平均周转时间和吞吐且偏向于短进程，长进程因为剩余时间多可能会饥饿。

# 多级反馈队列 (MLFQ)

多级反馈队列：Multi-Level Feedback Queue，MLFQ。通过反馈机制决定进程的优先级。

优先级的调整策略：

  • 维护多个优先级
  • 每个优先级一个队列
  • 初始放入优先级最高的队列
  • 对于调度执行结束的进程，调整优先级
    • 分片结束前主动释放 CPU，保持当前优先级
    • 使用整个时间片且被抢占，移入下一级优先级队列

      这种优先级调整策略偏向于短进程，因为其执行不会被打断，优先级不会降低，而长进程却会连续降级，甚至导致饥饿。
      有两种解决方法：
  1. 避免快速降级：采用指数增长的时间片，第 i 级队列分到的时间片长度为 2^i。
  2. 对进程进行升级：每隔一段时间将所有进程的优先级提到 最高 。

算法抢占性决策依据平均等待时间响应时间饥饿适用场景
FCFS非抢占到达顺序较长较长无批处理

MLFQ默认抢占反馈 + 优先级较短短可能（需额外机制）通用 / 交互式系统

# 实时调度

# 需要实时调度任务的特点

  1. 时限敏感：必须在规定时间内完成，否则造成严重后果
  2. 存在周期性，也就是相似的任务重复出现

实时调度中时限要求有四点：

  1. 启动时限：要求任务什么时候开始执行
  2. 完成时限：要求任务什么时候执行完成
  3. 优先级：要求一些任务必须在另外一些任务之前完成
  4. 层次结构：有些任务必须全部被完成（不能只完成一个）

# 假设约束条件

  • 每个进程具有一个确定的最坏情况运行时间
  • 只需考虑截止时限（deadline）
  • 只考虑单个 CPU 的情况
  • 进程之间没有先后顺序和层次化约束

# 最早时限优先 (EDF)

EDF（Earliest Deadline First）是一种在线调度算法，基本思路：在已经到达的进程中选择截止时限最早的进程进行调度执行。

在上述图像中，假设所有进程的执行时间都是相同时间（一个时间片），除 Schedule 外的表示那项进程从到达开始到最晚截止时限的时间段，在该时间段内开始执行那项进程都合法。
结果：只有任务 B 没有完成，因为 EDF 是在线算法，任务 B 到达后在等待任务 A 结束的期间就已经错过了截止时限，系统选择放弃这样的任务。

# 速率单调调度 (RM)

适用于周期任务，定义了周期为 T 的任务的所有实例的优先级:

  • 周期任务的优先级是这个周期任务被调用周期时间的倒数
  • 被调用速率 / 频率越高：周期任务的优先级越高

局限性：

  • 任务具有周期性，任务执行时间固定 + 周期时间固定
  • 每个任务的完成时限是下一个任务的到达时间，类比安保换岗，上一个保安下班，在换岗时间前离开，下一个保安刚好在上一个换岗时间到达。
