# 公钥密码体制

# 概念

# 为什么会有公钥密码？

对称密码体制的弊端：

  1. 系统开放性差，需要可靠的密钥传递渠道。
  2. 密钥管理量的困难：传统对称密钥两两用户维护一对密钥，则 n 个用户共需要维护 C (n,2)=n (n-1)/2 个密钥，当 n 很大时，密钥量极大，难以管理。
  3. 数字签名问题：对称加密算法难以实现抗抵赖的安全需求，因为加密和解密是对称的。

# 什么是公钥密码体制？

  1. 每个用户生成一个密钥对，一个公钥以及一个对应的私钥。
  • 公钥在系统内被公开
  • 私钥由用户本人安全管理（不公开）
  1. 用户 A 的公钥提供给系统内其他用户向用户 A 传输加密信息时的加密密钥，用户 A 的私钥用来解密他人用他的公钥加密的消息。
  2. 公钥密码又称为非对称密码体制。

# 对称 / 非对称密码对比

对称密码:

  • 对明文 / 密文变换时，加解密密钥相同，或可相互导出；
  • 双方在通信前需要安全地协商共享密钥；
  • 加解密算法效率较公开密码算法高；
  • 系统开放性差，密钥管理复杂；
  • 不能提供抗抵赖服务。

公钥密码:

  • 对明文 / 密文变换时，加解密密钥不相同，且不能由加密密钥（公钥，公开）导出解密密钥（私钥，保密）；
  • 系统开放性好，密钥管理较容易；
  • 可提供抗抵赖服务（数字签名）
  • 加解密运算复杂，效率低，不宜作数据加密；
  • 存在特有的，“可能报文攻击” 的威胁。

# RSA 的实质

在加密和解密的过程中运用到的模 p 这种运算，类比前面古典密钥中的替换密码，所以 RSA 其实就是一个替换表极大的单表替换密码。

为什么 RSA 安全性较高呢？
因为大整数 p、q 所乘积 n 极大，明文空间和密文空间极大，密文与明文之间的对应关系极难确定。

# RSA 的安全问题

  1. RSA 的关键：密钥产生，需要确定两个大素数 p、q，并计算 n=pq，d=e-1/2 (p-1)(q-1)，但是大素数的生成非常困难。

  2. RSA 的三种数学攻击方式（均极难实现）：

    • 将 n 分解为 p、q（大整数分解问题 - 极难）
    • 不分解 n 的因子，直接计算出 n 的欧拉函数
    • 不分解 n 的因子或者求 n 的欧拉函数就确定 n

  3. RSA 的非数学攻击方式：

    • 小加密指数 e 攻击，e 太小，密钥空间小
    • 选择密文攻击

  4. RSA 的体制问题

    • 不同用户不可共享模数 n（公共模数攻击）
    • 不同用户选用的素数不能相同
    • RSA 的不动点问题
    • 不宜用于数据加密，加解密速度慢，密钥长。

  5. RSA 的密钥选择

    • p 和 q 要足够大
    • p 和 q 应为强素数
    • p 和 q 的差要大
    • （p-1）和（q-1）的最大公因子要小
    • e 的选择，随机且含 1 多就安全，但加密速度慢
    • d 的选择，d 不能太小，要足够大
    • 不要许多用户共用一个模 n；易受共模攻击

# ElGamel 的算法特点

  1. 非确定性：由于密文依赖于执行加密过程的用户 A 所选择的随机数 k ，所以，加密相同的明文可能会产生不同的密文。
  2. 密文空间大于明文空间：明文空间为 Zp* ，而密文空间为 Zp* × Zp* 对于每个明文，其密文由 2 个 Zp * 上的元素组成。

# ElGamel 算法安全性

  1. ElGamal 公开密钥密码算法的安全性建立在计算 ZP 上离散对数的困难性上。
  2. 要使用不同的随机数 k 来加密不同的信息。
  3. 随机数 k 不可预测
