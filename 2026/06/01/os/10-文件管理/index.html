# 文件管理

  • 文件管理 、设备管理。目标是安全高效。

# 文件的逻辑结构

# 无结构文件

由一系列二进制数据流组成，如 .txt 文件中的字符数据。

# 有结构文件（记录式文件）

# 顺序文件

记录一个接一个顺序排列，可定长或变长，可顺序存储或链式存储。

存储方式定长记录变长记录
顺序存储可随机存取，顺序结构可快速查找无法随机存取
链式存储无法随机存取无法随机存取

按照与关键字顺序的关系：
串结构 ：记录顺序与关键字无关
顺序结构 ：按关键字排列

# 索引文件

索引表本身是定长的顺序文件，通过索引可以快速定位变长记录。

# 索引顺序文件

多级索引表嵌套查找，兼顾顺序存取和随机存取。

# 文件目录

# 文件控制块（FCB）

文件控制块支持的操作：搜索、创建文件、删除文件、显示目录、修改目录。

# 目录结构

结构说明
单级目录结构所有文件在一个目录中，不便于管理
两级目录结构主文件目录（MFD）+ 用户文件目录（UFD）
树形目录结构（多级）当代操作系统采用，层次清晰，但不便于文件共享
无环图目录结构在树形基础上通过链接实现共享

# 索引节点（i 节点）

对 FCB 进行压缩，将文件名和文件描述信息分离，减少目录项的大小，提高检索速度。

# 文件的物理结构（文件分配方式）

# 连续分配

要求每个文件在磁盘上占有一组连续的块。

  • 优点：顺序访问速度快
  • 缺点：不方便拓展，会产生磁盘碎片

# 链接分配

隐式链接 ：每个盘块中存放下一个盘块的指针。只支持顺序访问，不支持随机访问，方便拓展
显式链接 ：使用文件分配表（FAT）显式记录下一块物理块的位置。方便拓展，支持随机访问，但 FAT 会占用内存空间

# 索引分配

文件离散分配在各个磁盘块中，系统为每个文件建立一张索引表，记录各个逻辑块对应的物理块。

  • 支持随机访问
  • 索引表实现方案：链接方案、多层索引、混合索引

# 文件存储空间管理

# 存储空间的划分与初始化

文件卷（逻辑卷） 的概念

  • 目录区与文件区

# 管理方法

方法说明
空闲表法记录首位置 + 长度，回收时注意前后合并
空闲链表法空闲盘块链 / 空闲盘区链
位示图法每个 bit 表示一个盘块是否空闲
成组链接法目录区中专门用一个磁盘块作为超级块，记录空闲块信息

# 文件的基本操作

操作步骤
创建（create）1. 在外存中找到文件所需空间 2. 创建该文件对应的目录项
删除（delete）1. 找到文件名对应的目录项 2. 回收文件占用的磁盘块 3. 删除文件对应的目录项
读（read）根据目录项找到文件在外存中的位置，读取数据
写（write）将数据写入文件对应的磁盘块
打开（open）1. 找到文件名对应的目录项 2. 将目录项复制到内存的 "打开文件表" 中
关闭（close）递减文件打开计数，当计数为 0 时从内存的打开文件表中移除该文件表项

# 文件共享

# 硬链接（基于索引节点的共享）

直接指向文件的索引节点（i 节点）。多个目录项指向同一个 i 节点。

  • 限制：不能跨文件系统

# 软链接（基于符号链的共享）

相当于 Windows 的快捷方式，存储的是目标文件的路径。

  • 优点：可以跨文件系统
  • 缺点：原文件删除后，软链接失效

# 文件保护

方式说明
口令保护访问文件时需要输入口令
加密保护对文件内容加密，保密性强，不需要在系统中存储密码，但编解码需要花费时间
访问控制在每个文件的 FCB 中增加访问控制表（ACL），记录各用户可执行的操作

# 文件系统的层次结构

# 磁盘的结构

磁盘 ：由多个盘片组成
磁道 ：盘面上的同心圆环
扇区 ：磁道上的最小存储单元
柱面 ：所有盘面相同位置的磁道集合

  • 磁盘物理地址：柱面号 + 盘面号 + 扇区号

# 磁盘调度算法

# 一次磁盘读 / 写的时间

时间分量公式 / 说明
寻道时间 Ts启动磁臂、移动磁头，Ts = s + m ** n
延迟时间 Tr目标扇区转到磁头下方，Tr = 1 / (2r)
传输时间 Tt读 / 写数据，Tt = b / (rN)

# 调度算法

算法说明特点
先来先服务（FCFS）按请求顺序服务公平，但性能一般
最短寻道时间优先（SSTF）优先处理最近的磁道可能产生饥饿
扫描算法（SCAN）磁头单向移动到底再反向LOOK 优化：无请求时立即反向
循环扫描算法（C-SCAN）单向服务，返回时不处理请求C-LOOK 优化：无请求时立即返回

# 减小磁盘延迟时间的方法

交替编号 ：磁头读取一块内容后需要一小段时间处理，通过交替编号让逻辑相邻的扇区在物理上间隔排列
柱面号在盘面号之前 ：减少磁头移动消耗的时间
错位命名 ：相邻盘面的同名扇区错位排列

# 磁盘的管理

# 磁盘初始化

低级格式化 （物理分区）：划分磁道和扇区

  • 高级格式化：建立文件系统

# 引导块

ROM 不可修改，只存放很小的 "自举装入程序"。完整引导程序存放在磁盘的引导块中。

# 坏块管理

在 FAT 表上标明坏块（坏块对操作系统透明，文件系统会自动避开）。
