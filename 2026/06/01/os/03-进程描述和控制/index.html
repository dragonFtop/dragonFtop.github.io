# 进程控制和描述

# 什么是进程？

操作系统有序管理应用程序的执行，进而达到如下目标：

  • 资源对多个应用程序是可用的。
  • 物理处理器在多个应用程序间切换，以保证所有程序都在执行中。
  • 处理器和 I/O 设备能得到充分利用。
  • 进程的定义 ：
  • 一个正在执行的程序。
  • 一个正在计算机上执行的程序实例。
  • 能分配给处理器并由处理器执行的实体。
  • 由一组执行的指令、一个当前状态和一组相关的系统资源表征的活动单元。
    进程的两个基本元素：程序代码（program code）和与代码相关联的数据集（set of data）
    进程可以用以下元素来表征，而这些信息被存储在进程控制块中（process control block, PCB）

# 如何实现进程控制

原语 实现。原语做的事情：

  1. 更新 PCB 中的信息
  2. 将 PCB 插入合适的队列
  3. 分配 / 回收资源

# 进程控制相关的原语

原语转换操作步骤触发事件
创建原语无 -> 创建态 -> 就绪态申请空白 PCB、分配资源、初始化 PCB、插入就绪队列用户登录、作业调度、提供服务、应用请求
撤销原语运行态 -> 终止态回收资源、撤销 PCB正常结束、异常结束（如数组越界）、外界干预
阻塞原语运行态 -> 阻塞态（主动）更新 PCB、插入阻塞队列等待系统分配资源、等待其他进程完成工作
唤醒原语阻塞态 -> 就绪态（被动）更新 PCB、插入就绪队列等待的事件发生
切换原语运行态 <-> 就绪态保存上下文、恢复新进程上下文时间片用完、更高优先级到达、主动阻塞、进程终止

# 进程通信

进程通信的三种方式：

# 1. 共享存储

分配共享空间，且访问需互斥（通过 P/V 操作）。

基于数据结构的共享 ：固定分配，低级通信
基于存储区的共享 ：划分存储区，高级通信

# 2. 消息传递

消息结构： 消息头 + 消息体

通信方式说明
直接通信方式进程间直接发送消息
间接通信方式通过信箱间接发送和接收消息

# 3. 管道通信（pipe）

只能 半双工通信
