# 进程 - 题目解析

共 20 题，100 分，全部为单选题。

# 1. 多道环境下，处理器 / CPU 的行为，可由以下的（）来进行描述

A. 单个进程的轨迹 /trace
B. 多个进程的轨迹 /trace
C. 多个进程轨迹 /trace 的交替执行
D. 以上说法均正确

  • 正确答案：C

解析 ：单道环境下 CPU 只执行一个进程，行为就是单个进程的轨迹。但在多道环境下，多个进程并发执行，CPU 在多个进程间快速切换，宏观上同时、微观上交替。因此 CPU 的行为不再是单一轨迹，而是多个进程轨迹的交替执行。这体现了并发（Concurrency）的核心特征。

# 2. 多道环境下，单个进程的行为，可由以下的（）来进行描述

A. 单个进程的轨迹 /trace
B. 多个进程的轨迹 /trace
C. 多个进程轨迹 /trace 的交替执行
D. 以上说法均正确

  • 正确答案：A

解析 ：虽然多道环境下 CPU 在多个进程间切换，但单个进程仍然是按照自己的指令序列执行的。单个进程的执行轨迹（trace）描述的就是这个进程自身的指令执行历史，不受其他进程干扰（逻辑上独立）。进程的 "独立性" 特征保证了每个进程有自己的 trace。

# 3. 关于进程的两状态模型，其包括的两个状态是（）

A. 运行态和执行态
B. 运行态和非运行态
C. 运行态和阻塞态
D. 运行态和就绪态

  • 正确答案：B

解析 ：两状态模型是最简单的进程模型，只区分 "正在 CPU 上执行" 与 "不在 CPU 上执行" 两种情况：
运行态运行态 ：进程正在 CPU 上执行
非运行态非运行态 ：进程不在 CPU 上（可能在等待 I/O，可能在排队等调度）

这是最早的进程模型，随后发展出五状态模型（新建、就绪、运行、阻塞、退出），对 "非运行态" 做了更细的划分。

# 4. 关于进程的五状态模型，以下哪个情况可能会导致新建 / New 态的进程的产生（）

A. 新的批处理作业提交
B. 新的用户从终端登录
C. 现有的进程创建了子进程
D. 以上说法均正确

  • 正确答案：D

解析 ：进程创建的三个典型场景：
批处理作业提交批处理作业提交 ：用户提交一个作业，操作系统为其创建进程
用户登录用户登录 ：交互式系统中，用户登录时操作系统为其创建 shell 进程
进程派生（fork）进程派生（fork） ：一个进程可以通过系统调用（如 fork() ）创建子进程

这三种都会导致一个进程进入 "新建态"（New）：PCB 已创建，但尚未被调度执行。

# 5. 关于进程的五状态模型，以下哪个情况可能会导致退出态 / Exit 态的进程的产生（）

A. 执行了无效指令
B. 父进程终止
C. 系统管理员终止了它
D. 以上说法均正确

  • 正确答案：D

解析 ：进程终止的原因分为三类：

类型举例
正常结束进程执行完成，主动退出
异常结束执行非法指令、越界访问、除零错误等（选项 A）
外界干预父进程终止导致子进程终止（选项 B）、系统管理员 kill 进程（选项 C）

这三种都会导致进程进入退出态，随后操作系统回收其资源。

# 6. 关于进程的五状态模型，不包括的状态转换情况是（）

A. 从就绪态到运行态
B. 从运行态到就绪态
C. 从阻塞态到运行态
D. 从运行态到阻塞态

  • 正确答案：C

解析 ：五状态模型中合法的状态转换：

  • 就绪 -> 运行 ：被调度选中，获得 CPU（合法）
  • 运行 -> 就绪 ：时间片用完，被抢占（合法）
  • 运行 -> 阻塞 ：主动等待某事件，如 I/O（合法）
  • 阻塞 -> 运行 ：❌ 不合法！阻塞态只能先回到就绪态，再由调度分派到运行态
  • 阻塞态不能直接跳到运行态的原因是：此时可能有多个进程都已解除阻塞，需要通过调度算法决定谁获得 CPU。

# 7. 一个新进程的创建，包括了以下的哪个步骤（）

A. 为新进程分配新的内存地址空间
B. 为新进程分配一个唯一的进程标识符 / ID
C. 初始化新进程的进程控制块 / PCB
D. 以上说法均正确

  • 正确答案：D

解析 ：创建进程的完整步骤（创建原语做的事）：

  1. 申请空白 PCB ：分配进程控制块
  2. 分配唯一 PID （选项 B）：进程标识符，系统中唯一
  3. 分配资源 （选项 A）：内存地址空间、文件描述符等
  4. 初始化 PCB （选项 C）：设置状态为 "新建"，填入 PID、优先级等
  5. 将 PCB 插入就绪队列

# 8. 关于进程切换 / Process Switch 的时机，可能发生在是（）

A. 时钟中断
B. 系统调用
C. 缺页故障
D. 以上说法均正确

  • 正确答案：D

解析 ：进程切换的触发时机包括：

  • 时钟中断 （选项 A）：时间片用完，时钟中断触发，操作系统重新调度
  • 系统调用 （选项 B）：如进程主动调用 sleep() 、 wait() 等阻塞自己，导致切换
  • 缺页故障 （选项 C）：访问的页面不在内存中，进程进入阻塞态等待 I/O，触发切换

此外还有：I/O 中断（高优先级进程变为就绪）、进程终止等。

# 9. 当应用程序执行的时候，处理器 / CPU 的态 /mode 处于（）

A. 用户态 / User mode
B. 系统态 / System mode
C. 内核态 / Kernel mode
D. 以上均有可能

  • 正确答案：A

解析 ：理解本题的关键是区分 "应用程序执行" 和 "发生系统调用时"：

  • 应用程序的代码主体（如 C 程序的 main() ）运行在用户态
  • 当应用程序发起系统调用（如 read() 、 fork() ）或发生中断 / 异常时，CPU 才切换到内核态
  • 系统调用返回后，CPU 切回用户态继续执行

题干问的是 "当应用程序执行的时候"，默认情况就是用户态。应用程序的大部分时间都运行在用户态。

# 10. 关于进程切换 / Process Switch 和处理器态的切换 / CPU mode Switch，以下正确的是（）

A. 有进程切换，则一定伴随着处理器态的切换
B. 有处理器态的切换，则一定伴随着进程切换
C. 两种切换 / Switch 是等价的
D. 两种切换 / Switch 并无关系

  • 正确答案：A

解析 ：两者的关系：

  • A 正确：进程切换意味着从当前进程切换到另一个进程，这个操作必须由操作系统内核执行，因此 CPU 需要从用户态切换到内核态（通过中断 / 系统调用），完成切换后再回到新进程的用户态。所以进程切换必伴随态切换。
  • B 错误：比如应用程序发起 getpid() 系统调用，CPU 从用户态切到内核态，执行完立即返回同一进程，并未发生进程切换。所以态切换不一定伴随进程切换。
  • 两者不等价（C 错误），也不是完全无关（D 错误）。

# 11. Linux 中运行如下程序，结果是？

A. This is the child process, a=4 This is the parent process, a=6
B. This is the child process, a=4
C. This is the parent process, a=6
D. This is the parent process, a=6 This is the child process, a=5

  • 正确答案：A

解析 ： fork() 调用一次返回两次：

  • 子进程中 fork() 返回 0 ，所以 pid == 0 成立，进入 if 分支，执行 --a ： 5 - 1 = 4
  • 父进程中 fork() 返回子进程的 PID（正数），进入 else 分支，执行 ++a ： 5 + 1 = 6

注意：父子进程各自有独立的地址空间， a 的初始值都是 5（fork 时的快照），但修改互不影响。

# 12. 教材的图 3.2 中，有一个叫分派器 / Dispatcher 的功能子模块，其属于 OS 的哪个模块（）

A. 进程
B. 内存管理
C. I/O 管理
D. 文件系统

  • 正确答案：A

解析 ：分派器（Dispatcher）是进程调度模块的组成部分，负责将 CPU 的控制权交给由调度程序（Scheduler）选中的进程。其职责包括：

  1. 切换上下文（保存旧进程状态，恢复新进程状态）
  2. 切换到用户态
  3. 跳转到用户程序的新位置开始执行

Dispatcher 属于进程管理子系统的功能。

# 13. 下列关于进程的叙述中，哪一个是正确的（）

A. 优先级是进行进程调度的重要依据，一旦确定不能改变
B. 进程获得处理器而运行是通过调度而得到的
C. 进程申请处理器得不到满足时，其状态变为阻塞态
D. 在单处理器系统中，任一时刻有 2 个进程处于运行状态

  • 正确答案：B

解析 ：逐一分析：

  • A 错误：优先级是可以动态调整的，操作系统可以根据进程的行为调整优先级（如老化机制 Aging 提高长时间未执行进程的优先级；MLFQ 中根据 CPU 使用情况动态降级）。
  • B 正确：进程从就绪态进入运行态，是由调度程序（Scheduler）选择并分派 CPU 的结果。
  • C 错误：进程申请处理器得不到满足时，状态为就绪态（等待 CPU），不是阻塞态。阻塞态是等待 I/O 等事件。
  • D 错误：单处理机在任一时刻最多只有一个进程处于运行态。

# 14. 在单处理机系统中，处于运行状态的进程（）

A. 只有一个
B. 有多个
C. 不能被挂起
D. 必须在执行完成后才能被撤下

  • 正确答案：A

解析 ：单处理机 = 一个 CPU 核心，同一时刻只能有一条指令在执行，因此只能有一个进程处于运行状态。其他就绪进程在就绪队列中等待调度。如果是多核系统（如 4 核），则最多可以有 4 个进程同时处于运行态。

# 15. 一个阻塞进程所等待的事件发生后（）

A. 该进程重新占有 CPU
B. 进程状态变为就绪
C. 它的优先级变为最高
D. 其 PCB 移至就绪队列的队首

  • 正确答案：B

解析 ：状态转换的规则：阻塞态 -> 就绪态（被动等待事件发生 -> 唤醒）。

  • A 错误：变为就绪态不代表立即获得 CPU，还需要等待调度
  • B 正确：阻塞的原因解除后，进程被唤醒，从阻塞态转换为就绪态
  • C 错误：不会自动变为最高优先级
  • D 错误：PCB 从阻塞队列移动到就绪队列，不一定在队首（取决于插入策略）

# 16. 多道系统环境下，操作系统分配资源是以（）为基本单位

A. 作业
B. 指令
C. 程序
D. 进程

  • 正确答案：D

解析 ：引入线程后需要区分：

  • 进程：资源分配的基本单位（拥有地址空间、文件、设备等）
  • 线程：CPU 调度的基本单位（共享进程资源，只拥有少量私有上下文）

进程拥有资源所有权，操作系统分配内存、文件、I/O 设备时，以进程为单位。

# 17. 下面对进程的描述中，错误的是（）

A. 进程是动态的概念
B. 进程执行需要处理机
C. 进程是有生命期的
D. 进程是指令的集合

  • 正确答案：D

解析 ：D 错误 —— 进程是指令的集合是程序的定义，不是进程。进程是动态的执行实体。

程序进程
性质静态的（磁盘上的文件）动态的（执行中的实体）
组成指令的集合PCB + 程序段 + 数据段
生命期永久存在（除非删除）有创建和终止
特征—动态性、并发性、独立性、异步性、结构性

# 18. 下列几种关于进程的叙述，最不符合操作系统的进程模块的是（）

A. 进程即是程序，程序即是进程
B. 进程可以由程序、数据和进程控制块描述
C. 线程是一种特殊的进程
D. 进程是程序在一个数据集合上运行的过程，是系统进行资源分配和调度的一个独立单位

  • 正确答案：A

解析 ：

  • A（应选）：进程 ≠ 程序。一个程序可以被多个进程执行（如多个用户同时运行 vim ），一个进程也可以先后执行不同的程序（如 shell 通过 exec 替换程序）。
  • B：进程的三个组成部分就是程序段、数据段、PCB。
  • C：从某种意义上说，线程是 "轻量级进程"，共享进程资源但有自己的执行流。
  • D：这是进程的经典定义。

# 19. 下列关于进程控制块 / PCB 的描述，哪一个是错误的？

A. 操作系统利用 PCB 描述进程的基本特征
B. 一个 PCB 对应一个进程
C. PCB 可用于描述进程的一些变化，如其状态的变化
D. PCB 通常保存在二级存储的磁盘上

  • 正确答案：D

解析 ：

  • D（应选）：PCB 必须常驻内存中，不能放在磁盘上。磁盘访问速度（ms 级）无法满足调度需求（几十 ms 一次）。
  • A：PCB 包含了进程的所有描述信息（PID、状态、优先级、寄存器上下文、内存信息等）。
  • B：每个进程有且仅有一个 PCB，PCB 是进程存在的唯一标志。
  • C：当进程状态改变时，PCB 中的状态字段会同步更新。

# 20. 下列哪一项工作不是创建进程时所做的？

A. 给新进程分配一个唯一标识
B. 给新进程分配虚拟地址空间
C. 将处理器控制权交给新进程
D. 初始化新进程的进程控制块

  • 正确答案：C

解析 ：创建进程的步骤：

  1. 分配唯一标识 / PID（选项 A）✓
  2. 分配地址空间（选项 B）✓
  3. 初始化 PCB（选项 D）✓
  4. 将 PCB 插入就绪队列

选项 C：将 CPU 控制权交给新进程，是调度 / 分派阶段做的事。创建完成后进程进入就绪态，需要等待调度程序选中后才获得 CPU。
