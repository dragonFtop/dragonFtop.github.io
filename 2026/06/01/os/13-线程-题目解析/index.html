# 线程 - 题目解析

共 20 题，100 分，全部为单选题。主题：线程。

# 1. 一种将执行应用程序的进程划分为可并发运行的线程的技术称为（）

A. 多线程
B. 同步
C. 多进程
D. 以上都不是

  • 正确答案：A

解析 。多线程允许一个进程内同时执行多个任务，提高并发度和系统效率。同步是协调线程 / 进程的手段，多进程是另一种并发模型（进程间隔离更重）。

# 2. 操作系统能够独立处理 "资源所有权" 和 "调度 / 执行" 两个特点。对于支持线程的操作系统，"资源所有权" 被给与了以下哪个实体（）

A. 程序
B. 进程
C. 线程
D. 系统调用

  • 正确答案：B

解析 ：引入线程后，操作系统的两个特点被分开：

  • 资源所有权 → 进程：进程拥有地址空间、文件、I/O 设备等资源
  • 调度 / 执行 → 线程：线程是 CPU 调度的基本单位

这也是引入线程的核心动机 —— 将资源分配和调度执行分离。

# 3. 操作系统能够独立处理 "资源所有权" 和 "调度 / 执行" 两个特点。"调度 / 执行"（分派 / Dispatching）被给与了以下哪个实体（）

A. 程序
B. 进程
C. 线程
D. 系统调用

线程线程

  • 正确答案：C ：线程是 CPU 调度的基本单位，也是处理机分配的单位。分派器（Dispatcher）将 CPU 分配给就绪线程。

解析 ：注意区分：进程拥有资源，线程被调度执行。

# 4. 关于进程和线程之间的关系，下列哪一项是正确的（）

A. 在两个不同的进程之间切换所需的时间比在同一个进程中的两个线程之间切换所需的时间要少
B. 在现有进程中创建新线程所花费的时间要比创建新进程少得多
C. 终止一个进程所需的时间比终止一个线程要少
D. 以上都是

  • 正确答案：B

解析 ：

  • A 错误：进程切换需要切换整个地址空间（切换页表、刷新 TLB），开销远大于同进程内线程切换（只需切换寄存器上下文和栈）
  • B 正确：创建线程不需要分配新的地址空间，直接共享进程的资源，开销远小于创建进程（需要分配地址空间、初始化 PCB 等）
  • C 错误：终止进程需要回收其全部资源（内存、文件、所有线程），比终止单个线程开销大得多

# 5. 下列关于线程的叙述中，正确的是（）

A. 线程包含 CPU 现场，可以独立执行程序
B. 每个线程有自己独立的地址空间
C. 进程只能包含一个线程
D. 线程之间的通信必须使用系统调用函数

  • 正确答案：A

解析 ：

  • A 正确：线程有自己的寄存器上下文（CPU 现场）、栈和程序计数器，可以独立执行
  • B 错误：同进程的线程共享地址空间，这是线程与进程的关键区别
  • C 错误：一个进程可以包含多个线程（多线程模型）
  • D 错误：同进程的线程共享地址空间，可以直接通过共享变量通信，无需系统调用

# 6. 下列哪个系统应用了多线程（）

A. Solaris
B. Linux
C. Win10
D. 以上都是

  • 正确答案：D

解析 ：现代主流操作系统都支持多线程：

  • Solaris ：早期多线程先驱，支持 LWP（轻量级进程）机制
  • Linux ：通过 pthread 库支持 POSIX 线程，内核采用 NPTL（Native POSIX Thread Library）
  • Win10 ：Windows NT 系列从诞生起就支持多线程

# 7. 在 Linux 系统中，可以在一个进程中创建和执行多个线程。则其线程：进程的关系是（）

A. 1:1
B. M:N
C. M:1
D. 以上都不是

  • 正确答案：C

解析 ：从进程的角度看，一个进程内可以有多个线程，线程与进程的数量关系是 M:1（多个线程属于一个进程）。

注意区分：多线程模型中的 M:1 指的是 M 个 ULT 映射到 1 个 KLT；而本题是从进程的角度看线程与进程的数量比例关系，不要混淆。

# 8. 当一个线程需要等待一个事件时，与线程状态变化相关的基本线程操作称为（）

A. 阻塞操作
B. 解除阻塞操作
C. 生成操作
D. 以上都不是

  • 正确答案：A

解析 ：线程状态改变的基本操作：

  • 阻塞（Block） ：线程等待某个事件（如 I/O 完成），主动放弃 CPU，从运行态进入阻塞态
  • 解除阻塞（Unblock） ：等待的事件发生，线程从阻塞态回到就绪态
  • 派生（Spawn） ：创建新线程

# 9. 在纯 ULT 策略中，当一个多线程进程中的某个线程被阻塞后（）

A. 该进程的其他线程仍可继续运行
B. 整个进程都将阻塞
C. 该阻塞进程将被撤销
D. 该阻塞线程将永远不可能再执行

  • 正确答案：B

解析 ：用户级线程（ULT）由用户空间的线程库管理，操作系统内核看不到 ULT，只看到包含它们的那个进程。当某个 ULT 发起系统调用（如 I/O 操作）而阻塞时，内核会将整个进程置为阻塞态，因为内核认为这个进程在等待 I/O。结果就是该进程中的所有其他 ULT 也都无法运行。

这是 ULT 的核心缺点之一（与 KLT 对比，KLT 中一个线程阻塞不影响同进程其他线程）。

# 10. 与内核级线程（KLT）相比，用户级线程（ULT）的缺点之一是（）

A. 线程切换不需要切换到内核模式
B. 调度是基于应用的
C. 当用户级线程执行系统调用时，进程中的所有线程都被阻塞
D. 以上都是

  • 正确答案：C

解析 ：

  • A 是 ULT 的优点，不是缺点：切换在用户空间完成，无需内核态切换，效率高
  • B 是 ULT 的优点，不是缺点：应用程序可以自定义调度策略，灵活
  • C 是 ULT 的缺点：由于内核看不到 ULT，一个 ULT 的阻塞操作会导致整个进程被阻塞

# 11. 若系统中只有用户级线程，则处理器的调度单位是（）

A. 线程
B. 进程
C. 程序
D. 作业

  • 正确答案：B

解析 ：在纯 ULT 系统中，内核看不到线程，只能看到进程，因此以进程为单位进行调度 —— 内核将 CPU 分配给进程，进程内部的线程库再决定哪个 ULT 使用该 CPU 时间。内核级线程（KLT）才是以线程为调度单位的。

# 12. 下面关于线程的叙述中，正确的是（）

A. 不论是系统级线程还是用户级线程，其切换都需要内核的支持
B. 线程是资源的分配单位，进程是调度和分配的单位
C. 不管系统中是否有线程，进程都是拥有资源的独立单位
D. 在引入线程的系统中，进程仍是资源分配和调度分派的基本单位

  • 正确答案：C

解析 ：

  • A 错误：ULT 切换在用户空间完成，不需要内核介入
  • B 错误：说反了 —— 进程是资源分配单位，线程是调度单位
  • C 正确：无论是否引入线程，进程始终是资源的拥有者（地址空间、文件、设备等）
  • D 错误：引入线程后，调度分派的基本单位变为线程，进程只保留资源所有权

# 13. 在以下描述中，（）并不是多线程系统的特长

A. 利用线程并行地执行矩阵乘法运算
B. Web 服务器利用线程响应 HTTP 请求
C. 键盘驱动程序为每个正在运行的应用配备一个线程，用以响应该应用的键盘输入
D. 基于 GUI 的调试程序用不同的线程分别处理用户输入、计算和跟踪等操作

  • 正确答案：C

解析 ：键盘输入应由操作系统统一管理，再分发给前台应用，而不是为每个应用创建专门的键盘线程。这样做既不高效也不合理。

其他选项都是多线程的典型应用：矩阵运算并行化、Web 服务器每个请求一个线程、GUI 程序用不同线程处理界面和后台逻辑。

# 14. Windows 操作系统中动态 DLL 库里的系统线程，被不同的进程所调用，它们是（）的线程

A. 不同
B. 相同
C. 可能不同，也可能相同
D. 不能被调用

  • 正确答案：B

解析 ：DLL 库中的线程代码共享同一份物理内存中的指令，不同进程调用时执行的是相同的代码。从 "执行的代码" 角度判定为 "相同" 的线程代码。

# 15. 关于线程，在下面的叙述中正确的是（）

A. 线程是比进程更小的能独立运行的基本单位
B. 引入线程可提高程序并发执行的程度，可进一步提高系统效率
C. 线程的引入增加了程序执行时时空开销
D. 一个进程一定包含多个线程

  • 正确答案：B

解析 ：

  • A 错误：线程不能 "独立" 于进程而存在，线程必须依附于某个进程
  • B 正确：线程是比进程更轻量的执行单元，创建、切换开销更小，引入线程可以提高系统的并发度和效率
  • C 错误：线程的引入减少了时空开销（相比创建新进程），不是增加
  • D 错误：一个进程可以只包含一个线程（单线程进程）

# 16. 某操作系统在进程中引入了多个执行序列 —— 线程，那么下列叙述中正确的是（）

A. 进程是处理器调度的基本单位
B. 线程是资源分配的基本单位
C. 用户级线程执行时，同一进程不同线程的切换不需要内核支持
D. 线程可以独立于进程而存在

  • 正确答案：C

解析 ：

  • A 错误：引入线程后，线程是处理器调度的基本单位
  • B 错误：进程是资源分配的基本单位，线程不是
  • C 正确：ULT 由用户空间的线程库管理，切换在线程库中完成，不需要系统调用，无需内核介入
  • D 错误：线程必须依附于进程，不能独立存在

# 17. 使用 Linux 的 pthread 库创建的线程属于（）

A. 用户级线程 / ULT
B. 内核级线程 / KLT
C. 都有可能
D. 都不是

  • 正确答案：D

解析 ：该课程教材的观点是：pthread（POSIX 线程）是用户级线程，切换在用户空间完成，不需要内核介入。但实际上，现代 Linux 的 NPTL 实现中，pthread 创建的线程一对一映射到内核调度实体（LWP），本质上是 KLT。教材和工业实现存在差异，以教材答案为准。

# 18. 下列关于线程的说法，正确的是（）

A. 两个线程可以共享各类资源
B. 一个线程可以包含多个进程
C. 单处理器的计算机上，2 个线程不能同时处于运行态
D. 一个进程可以包含多个线程

  • 正确答案：D

解析 ：

  • A 错误：只有同一进程内的线程才共享资源，不同进程的线程资源是隔离的
  • B 错误：关系反了，是一个进程可以包含多个线程
  • C 正确但非最优：在单处理器上，2 个线程确实不能同时处于运行态，这一表述本身是正确的。但本题是单选题，D 是更基础的线程定义知识点，因此选 D
  • D 正确：一个进程可以包含一个或多个线程（多线程模型）

# 19. 下列关于线程的描述中，错误的是（）

A. 内核级线程的调度由操作系统完成
B. 操作系统为每个用户级线程建立一个线程控制块
C. 用户级线程间的切换比内核级线程间的切换效率高
D. 用户级线程可以在不支持内核级线程的操作系统上实现

  • 正确答案：B

解析 ：

  • A 正确：KLT 由操作系统内核管理，调度由内核完成
  • B 错误（应选）：ULT 对内核不可见，操作系统不知道 ULT 的存在，不会为 ULT 建立 TCB。TCB（线程控制块）由用户空间的线程库维护
  • C 正确：ULT 切换在用户空间完成，无需内核态切换，效率高于 KLT 切换
  • D 正确：ULT 完全由用户库实现，不需要操作系统支持内核级线程

# 20. 在一个进程中，可能有一个或者多个线程，每个线程有（）

A. 线程执行状态
B. 在未运行时保存的线程上下文
C. 用于每个线程局部变量的静态存储空间
D. 以上都有

  • 正确答案：D

解析 ：每个线程都拥有自己独立的一套：

  • 线程执行状态（A）：就绪、运行、阻塞等，由线程库和操作系统维护
  • 线程上下文（B）：寄存器值、程序计数器、栈指针等，切换时需要保存 / 恢复
  • 线程局部存储（C）：每个线程私有的变量存储空间（如 C 中的 __thread 变量、每个线程的栈区域）

这三者共同构成线程的私有资源，区别于进程中所有线程共享的全局资源和地址空间。
