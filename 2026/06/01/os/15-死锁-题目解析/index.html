# 死锁 - 题目解析

共 20 题，100 分，全部为单选题。主题：死锁与饥饿。

# 1. 一组竞争系统资源或相互通信的进程的永久阻塞被称为（）

A. 优先级
B. 死锁
C. 饥饿
D. 以上都是

  • 正确答案：B

解析 ：死锁（Deadlock）的定义：一组进程中的每个进程都在等待一个事件，而该事件只能由这组进程中的另一个进程产生，导致所有进程永久阻塞。关键特征：

  • 至少两个或两个以上进程同时发生
    永久阻塞永久阻塞 ，进程无法自行解除

区分：饥饿（Starvation）是单个进程长期得不到资源，但其他进程还在正常运行。

# 2. 所有死锁涉及的资源需求冲突有关（）

A. 一个或更多进程
B. 两个或更多进程
C. 三个或更多进程
D. 以上都不是

  • 正确答案：B

解析 。因为死锁的本质是进程之间相互等待对方持有的资源，不可能只有一个进程自己和自己产生死锁（一个进程即使资源请求不满足，也只是阻塞，不是死锁）。

# 3. 可以创建和销毁的资源被称为（）

A. 可重用资源
B. 可生成资源
C. 可消耗资源
D. 以上都是

  • 正确答案：C

解析 ：资源按使用特性分为：

类型说明例子
可重用资源使用后不消耗，可重复分配CPU、内存、打印机、锁
可消耗资源可以被创建（生产）和销毁（消费），使用后不复存在中断、信号、消息、数据缓冲区

可消耗资源由生产者创建，消费者消耗后即消失。

# 4. 下列是可消耗资源的有（）

A. 主存
B. 打印机
C. 消息
D. 以上都是

解析可消耗资源

可重用资源正确答案：C ：
可消耗资源解析 是可消耗资源：进程 A 发送消息（生产），进程 B 接收消息（消费），消息被消费后就不存在了

  • 主存、打印机都是可重用资源 —— 用完后释放，其他进程还可以继续使用

# 5. 造成死锁的条件可能是（）

A. 不可抢占
B. 互斥
C. 占有且等待
D. 以上都是

  • 正确答案：D

解析 ：死锁产生的四个必要条件（缺一不可）：

  • 互斥（Mutual Exclusion）：资源一次只能被一个进程使用
  • 占有且等待（Hold and Wait）：进程持有资源的同时还在等待其他资源
  • 不可抢占（No Preemption）：已分配的资源不能被强制剥夺
  • 循环等待（Circular Wait）：存在进程资源的环形等待链

四个条件同时满足才可能死锁，破坏任一条件即可预防死锁。

# 6. 死锁预防的一种直接方法是防止以下哪些情况的发生（）

A. 占有且等待
B. 互斥
C. 循环等待
D. 以上都是

占有且等待循环等待

不可抢占正确答案：D ：死锁预防 = 破坏四个必要条件之一。三种预防策略：

  • 破坏互斥 → 将互斥资源改造为可共享（但很多资源本质上是互斥的，如打印机）
  • 破坏占有且等待 → 要求进程一次性申请所有资源
  • 破坏循环等待 → 按顺序申请资源（资源有序分配法）
  • 破坏不可抢占 → 允许系统强制剥夺资源

选项 D"以上都是"—— 三种条件都可以被作为预防的突破口。

# 7. 是否允许当前的资源分配请求是通过判断该请求是否可能导致死锁来决定的，这种策略被称作（）

A. 死锁检测
B. 死锁预防
C. 死锁避免
D. 以上都不是

  • 正确答案：C

解析 ：三种死锁处理策略的区分：

策略时机做法
死锁预防系统设计阶段（静态）破坏四个必要条件之一
死锁避免运行中（动态）每次分配前判断是否会导致死锁，如银行家算法
死锁检测运行中（定期）不阻止分配，定期检测是否已死锁，再解除

题干描述的是 "每次分配前判断是否会导致死锁"→ 死锁避免。

# 8. 在死锁避免的资源分配拒绝方法中，安全定义为（）

A. 至少有一个潜在的进程序列不会导致死锁
B. 所有潜在的进程序列不会导致死锁
C. 数个潜在的进程序列不会导致死锁
D. 以上都不是

  • 正确答案：A

解析 ：安全状态 = 至少存在一个安全序列，使得系统可以按照该序列为每个进程分配资源，所有进程都能顺利完成，不会发生死锁。

不是 "所有序列" 都安全（B 太强），也不是 "数个"（C 太模糊），而是至少存在一个。

# 9. 下列关于银行家算法的叙述中，正确的是（）

A. 银行家算法属于死锁的预防策略
B. 当系统处于安全状态时，系统中至少有一个资源分配序列可以避免死锁
C. 当系统处于不安全状态时，系统中一定有死锁
D. 银行家算法破坏了死锁必要条件中的 "占有且等待" 条件

  • 正确答案：B

解析 ：

  • A 错误：银行家算法是死锁避免策略（动态判断），不是预防（破坏必要条件）
  • B 正确：安全状态 = 存在至少一个安全序列
  • C 错误：不安全状态不一定有死锁，只是可能发生死锁。死锁一定处于不安全状态，但不安全状态不等价于死锁
  • D 错误：银行家算法不破坏任何必要条件，而是通过动态检查避免进入不安全状态

# 10. 在死锁进程恢复中，选择一个特定进程中止或回滚的选择标准包括（）

A. 到目前为止分配的总资源最少
B. 最低优先级
C. 预计剩余时间最长
D. 以上都是

避免正确答案：D

B 正确解析 ：死锁恢复时选择 "牺牲者" 的考虑因素：

标准说明
分配资源最少撤销代价最小的进程
最低优先级优先牺牲相对不重要的进程
预计剩余时间最长避免撤销即将完成的进程
已产生的输出量最少避免大量已输出工作被丢弃
批处理（非交互式）交互式进程对用户体验更重要

注意：部分教材采用 "已执行时间最少" 或 "剩余时间最短" 作为标准（最小化已投入的计算量浪费），但本题按课程教材的 "剩余时间最长" 作答，意为优先保护即将完成的进程，避免撤销后重新长时运行。

# 11. 下面关于安全状态和非安全状态说法正确的是（）

A. 安全状态是没有死锁的状态，非安全状态是有死锁的状态
B. 安全状态是可能有死锁的状态，非安全状态也可能有死锁状态
C. 安全状态是可能没有死锁的状态，非安全状态有死锁的状态
D. 安全状态没有死锁的状态，非安全状态可能有死锁的状态

C 错误正确答案：D

不一定解析 ：安全状态与死锁的关系：

关系链：安全 ⊂ 无死锁，死锁 ⊂ 不安全。

所以 "安全 = 没有死锁，非安全 = 可能有死锁（不一定有）" 是正确描述。

# 12. 进程 P1 使用资源情况：申请资源 S1，申请资源 S2，释放资源 S1；

进程 P2 使用资源情况：申请资源 S2，申请资源 S1，释放资源 S2；
系统并发执行进程 P1，P2，系统将（）

A. 必定产生死锁
B. 可能产生死锁
C. 不会产生死锁
D. 无法确定是否会产生死锁

  • 正确答案：B

解析 ：分析 P1 和 P2 的资源申请顺序：

  • P1：申请 S1 → 申请 S2 → 释放 S1
  • P2：申请 S2 → 申请 S1 → 释放 S2

这存在典型的循环等待风险：

  • 若 P1 先拿到 S1，P2 拿到 S2，然后 P1 等 S2，P2 等 S1 → 死锁
  • 若 P1 先拿到 S1 和 S2 再释放，P2 再拿 → 不死锁

所以是可能死锁，取决于执行时序（竞态条件导致的不确定性）。

# 13. 某计算机系统中有 14 个设备，有 n 个进程竞争使用，每个进程最多需要 5 个设备。该系统可能会发生死锁的 n 的最小值是（）

A. 2
B. 3
C. 4
D. 5

正确答案：B 正确答案：C

解析 ：使用 "最坏情况" 分析法。假设每个进程已经获得了 (5-1)=4 个设备（还差 1 个才能完成）。如果此时系统已经没有多余的设备可以分配，则死锁。

最坏情况：n 个进程各持有 4 个设备 = 4n 个设备被占用。还需要至少 1 个空闲设备来打破死锁。

  • 4n + 1 ≤ 14 → n ≤ 13/4 = 3.25 → n ≤ 3 死锁不会 死锁
  • n = 4 时，4 × 4 = 16 > 14，最坏情况下所有进程都等最后一个设备 不死锁可能会死锁

所以 n 可能 4 。

# 14. 某系统有 n 台互斥使用的同类设备，3 个并发进程需要 3, 6, 8 台设备，可确保系统不发生死锁的设备数 n 最小为（）

A. 11
B. 14
C. 15
D. 17

  • 正确答案：C

解析 ：死锁避免公式（每个进程需要 r_i 台设备）：

公式含义：最坏情况下，每个进程都已获得比最大需求少 1 台的设备（即分别持 2、5、7 台），此时只需要再加 1 台设备就能让至少一个进程完成并释放设备，打破僵局。所以 n ≥ 15 时可以确保不死锁。

# 15. 下列描述的各种现象中，属于活锁现象的是（）

A. 相关进程进入阻塞状态，且无法唤醒
B. 相关进程进入阻塞状态，且可以唤醒
C. 相关进程没有阻塞，但是调度时刻被延迟推后
D. 相关进程没有被阻塞，可被调度，但是不做有用的工作

不会正确答案：D

可能会死锁解析 ：活锁（Livelock）vs 死锁（Deadlock）：

死锁活锁
进程状态阻塞（等待资源 / 事件）没有阻塞（可被调度，在运行）
是否能执行完全不能推进一直在执行，但不做有用工作
典型场景互相等待对方释放资源互相谦让资源，谁都没拿到

活锁就像是两个人面对面走来，互相让路，左让右让一直在动但谁都没走过去。

# 16. 假设 5 个进程 P0、P1、P2、P3、P4 共享 3 类资源 R1、R2、R3，这些资源总数分别为 18、6、22。T0 时刻的资源分配情况如下表所示，此时存在的一个安全序列是？

A. P0, P2, P4, P1, P3
B. P1, P0, P3, P4, P2
C. P2, P1, P0, P3, P4
D. P3, P4, P2, P1, P0

解析正确答案：D

正确答案：D 解析 ：这是一道银行家算法安全性检查题。解法：

  1. 计算当前 Available（可用资源）= 总量 - 已分配总量
  2. 检查每个进程的 Need（需求）= Max - Allocation
  3. 按候选序列逐一验证：每步检查 Need ≤ Available，满足则 "完成" 该进程，将其已分配资源回收到 Available
  4. 四个选项中只有 D 序列（P3, P4, P2, P1, P0）能通过安全性检查

由于原文中的资源分配表格未能完整提取，此处着重说明方法：

  • 安全性算法的核心：不断找到 Need ≤ Available 的进程，模拟其完成并回收资源，继续下一个
  • 若最终所有进程都能 "完成"，则存在安全序列；若中途找不到可满足的进程，则系统不安全
  • 据原始答题记录，D（P3, P4, P2, P1, P0）为通过安全性检查的正确序列

# 17. 解除死锁通常不采用的方法是（）

A. 终止一个死锁进程
B. 终止所有死锁进程
C. 从死锁进程处抢夺资源
D. 从非死锁进程处抢夺资源

解析正确答案：D

阻塞解析 ：死锁解除的三种方法：

方法说明
资源剥夺法从死锁进程处抢夺资源分给其他死锁进程
撤销进程法终止一个（或全部）死锁进程，释放其资源
进程回退法让死锁进程回退到检查点，重新执行

选项 D"从非死锁进程处抢夺资源" 不合理 —— 非死锁进程是无辜的，没有理由被剥夺资源，且剥夺它们的资源也不一定能解决死锁链。

# 18. 在系统运行过程中，通过检查系统是否处于安全状态而不让死锁发生的策略是（）

A. 死锁预防
B. 死锁避免
C. 死锁检测
D. 死锁解除

  • 正确答案：B

解析 ：再次强调三种策略的区别：

  • 死锁预防：事先破坏必要条件（静态）
  • 死锁避免：运行时动态判断，通过安全性检查决定是否分配（动态，如银行家算法）
  • 死锁检测 + 解除：先允许分配，定期检测是否死锁，死锁后再解除

# 19. 对资源采用按序分配策略能达到下列哪一个目的？

A. 死锁预防
B. 死锁避免
C. 死锁检测
D. 死锁解除

  • 正确答案：A

解析 ：按序分配是死锁预防策略。它通过破坏 "循环等待" 条件来预防死锁：

  • 对所有资源编号（如 R1, R2, ..., Rn）
  • 进程必须先申请编号小的资源，再申请编号大的
  • 这样就无法形成循环等待链

因为是在系统设计阶段就定下的规则，属于预防而非避免。

# 20. 在进程资源图中，（）是发生死锁的必要条件

A. 互斥
B. 不可剥夺条件
C. 环路
D. 以上都是

  • 正确答案：D

解析 ：死锁四个必要条件全选。在进程资源分配图中：

  • 环路（循环等待）→ 可能死锁
  • 如果每种资源只有一个实例 + 有环路 → 一定死锁
  • 如果每种资源有多个实例 + 有环路 → 不一定死锁

但互斥、不可剥夺、占有且等待、循环等待这四个条件全部满足时才会死锁，缺一不可。
