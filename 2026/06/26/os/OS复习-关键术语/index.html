地址寄存器：存放内存访问地址的 CPU 寄存器，用于指定数据的存储位置。

指令周期：CPU 处理一条指令的完整过程，包括取指、译码、执行和写回等阶段。

可编程 I/O：CPU 通过执行 I/O 指令直接控制外部设备的数据传输方式。

辅助寄存器：辅助 CPU 运算的附加寄存器，用于暂存中间结果或状态信息。

指令寄存器：存放当前正在执行的指令的寄存器。

寄存器：CPU 内部的高速小容量存储单元，用于暂存指令、数据和地址。

块：存储设备上数据读写的最小逻辑单位。

中断：硬件或软件发出的信号，使 CPU 暂停当前程序转而执行中断处理程序。

置换算法：当缓存或内存空间满时，决定替换哪个已有数据块的策略。

高速缓存：位于 CPU 与主存之间的高速小容量存储器，利用局部性原理减少平均访存时间。

中断驱动 I/O：I/O 设备准备好后通过中断通知 CPU，CPU 再介入处理数据传输的方式。

辅存：主存之外的外部存储设备（磁盘、SSD 等），容量大但速度较慢，数据可持久保存。

高速缓冲槽：Cache 中存储一个数据块的最小位置单元。

I/O 模块：连接 CPU 与 I/O 设备的硬件组件，负责数据缓冲、控制和时序协调。

槽：缓存中存放一个缓存行 / 块的位置单元。

中央处理单元（CPU）：计算机的核心部件，负责取指令、译码并执行，控制整个计算机的运行。

局部性引用：程序在执行过程中倾向于重复访问同一片存储区域的现象。

空间局部性：若某个存储位置被访问，则其邻近位置在短期内也很可能被访问的特性。

芯片多处理器：在单个芯片上集成多个 CPU 核心的处理器架构。

内存（主存）：CPU 可直接访问的主存储器，通常为 RAM，用于存放正在运行的程序和数据。

栈：后进先出（LIFO）的数据结构，用于管理函数调用、参数传递和局部变量存储。

数据寄存器：CPU 中用于暂存操作数或运算结果的寄存器。

分级存储体系：按速度、容量和成本分层组织的存储结构，从寄存器、Cache、主存到辅存逐级向下。

栈帧：每次函数调用时在栈上分配的一段区域，包含返回地址、参数和局部变量。

直接内存存取（DMA）：允许 I/O 设备绕过 CPU 直接与内存交换数据的机制，大幅减少 CPU 开销。

未命中：CPU 请求的数据不在 Cache 中找到，需要访问下级存储。

栈指针：始终指向栈顶的寄存器，用于管理栈的压入和弹出操作。

命中：CPU 请求的数据在 Cache 中被找到。

多核：单个处理器芯片上集成多个计算核心，支持真正的并行执行。

系统总线：连接 CPU、内存和 I/O 模块的公共数据传输通路。

命中率：缓存命中的访问次数占总访问次数的百分比。

多处理器：包含两个或以上独立 CPU 的计算机系统。

时间局部性：若某个存储位置被访问，则在不久的将来它很可能再次被访问的特性。

输入 / 输出（I/O）：计算机与外部设备或环境之间的数据交换过程。

处理器：执行指令、进行算术逻辑运算和控制操作的硬件核心单元。

指令：CPU 能够识别并执行的基本操作命令。

程序计数器（PC）：存放即将执行的下一条指令地址的专用寄存器。

批处理：将一批作业预先提交给系统，由系统自动逐个执行而无需用户交互的处理方式。

管程：一种高级同步机制，将共享数据和对共享数据的操作封装在一起，保证同一时刻只有一个进程访问共享资源。

时间片轮转：为每个进程分配一个固定长度的时间片，时间片用完后切换到下一个就绪进程。

批处理系统：以批处理方式运行的操作系统，作业成批提交、成批执行、成批输出结果。

单体内核：将操作系统的所有核心功能全部运行在内核态的整体架构，性能高但模块间耦合紧密。

调度：操作系统按照一定策略从就绪进程中选择一个分配 CPU 的过程。

执行上下文：进程运行所需的全部 CPU 状态信息，包括寄存器值、程序计数器、栈等。

多道批处理系统：内存中同时驻留多道作业，当一道作业等待 I/O 时 CPU 可切换到另一道作业执行。

串行处理：一次只加载和执行一个程序，程序执行完才能开始下一个。

分布式操作系统：管理多台通过网络连接的计算机，使其协同工作如单一系统的操作系统。

多道程序设计：内存中同时存放多道程序，当运行的程序等待 I/O 时，CPU 切换执行另一道程序以提高利用率。

状态：进程在生命周期中所处的阶段，如就绪态、运行态、阻塞态等。

宕机时间：系统不可用或停止服务的时间段。

多任务：操作系统同时管理多个任务 / 进程的能力，通过快速切换实现并发执行。

对称多处理（SMP）：多个相同处理器共享同一内存和 I/O 总线，任意处理器可执行任意任务。

错误：系统中发生的异常或故障情况。

多线程：一个进程内包含多个可并发执行的线程，共享进程资源。

任务：操作系统调度的基本单位，可以是一个进程或线程。

中断：暂停 CPU 当前执行的任务，转而处理更高优先级事件的机制。

核心：执行指令的物理 CPU 单元，多核处理器中每个核心可以独立运行一个线程。

线程：进程内的一个执行流，是 CPU 调度的基本单位，同一进程的线程间共享地址空间。

作业：用户提交给系统执行的一个完整计算任务单位。

面向对象程序设计：将数据和行为封装为对象的编程范式，用于操作系统设计中模块化和可扩展性的实现。

分时：操作系统将 CPU 时间划分为短的时间片，轮流分配给多个用户 / 进程使用。

作业控制语言（JCL）：用户用于向批处理系统描述作业需求和控制作业执行的语言。

操作系统：管理计算机硬件和软件资源，为用户和应用程序提供服务的系统软件。

分时系统：允许多个用户通过终端同时交互式使用同一台计算机的操作系统。

内核：操作系统的核心部分，运行在特权模式下，管理 CPU、内存、I/O 等系统资源。

物理地址：内存中实际硬件存储单元的地址，用于在地址总线上寻址。

时间片：操作系统分配给每个进程的 CPU 使用时长。

内核模式：CPU 的最高特权执行模式，操作系统内核代码在此模式下运行，可执行所有指令和访问所有资源。

特权指令：只能在特权模式下执行的指令，如 I/O 指令、修改页表基址寄存器等。

单道程序设计：内存中一次只存放一道程序，CPU 在该程序等待 I/O 时空转。

可加载模块：可以在系统运行时动态加载和卸载的内核扩展组件，增加系统灵活性。

进程：正在执行的程序的实例，包含程序代码、数据和执行上下文。

正常运行时间：系统从启动以来持续运行的时间。

平均失效时间（MTTF）：系统或组件从开始工作到发生故障的平均时间间隔。

进程状态：进程在生命周期中某个时刻所处的状态，如新建、就绪、运行等。

用户模式：CPU 的受限执行模式，运行用户程序，不能执行特权指令，保证系统的安全性。

平均修复时间（MTTR）：系统或组件从发生故障到恢复正常工作的平均修复时间。

实地址：对应物理内存中实际存储单元位置的地址，即物理地址。

虚拟地址：程序使用的逻辑地址，由 MMU 映射到物理地址，使每个进程拥有独立的地址空间。

内存管理：操作系统负责分配、回收和保护内存资源的子系统，通常包含分页或分段机制。

可靠性：系统在规定条件下和规定时间内完成规定功能的概率指标。

虚拟机：通过软件模拟的具有完整硬件环境的虚拟计算机，可在物理机上同时运行多个隔离的操作系统实例。

微内核：只将最核心的功能（如进程间通信、基本调度）放在内核中，其他服务移到用户态运行的 OS 架构。

常驻监控程序：早期批处理系统中常驻内存的管理程序，负责作业的自动切换和顺序执行。

虚拟内存：利用磁盘空间模拟出比实际物理内存更大的地址空间的技术。

循环制：循环调度策略，轮流为每个进程分配 CPU 时间。

# 第 3 章 进程描述与控制 关键术语

术语

模式切换进程切换跟踪

阻塞态：进程因等待某事件（如 I/O 完成、资源释放）而暂停执行的状态。

特权态：CPU 的最高执行模式，又称内核态 / 系统态，允许执行所有指令。

运行态：进程当前正在 CPU 上执行的状态。

子进程：由另一个进程（父进程）创建的进程，可以继承父进程的部分资源。

进程：正在运行的程序的实例，是资源分配的基本单位。

挂起态：进程被换出到辅存（磁盘），暂时不参与 CPU 调度的一种状态。

分派器：调度程序的一部分，负责将 CPU 控制权移交给选中的进程。

交换：将整个进程从内存移到辅存（换出）或从辅存移回内存（换入）的操作。

退出态：进程执行完毕或被终止后所处的状态，等待父进程回收。

进程控制信息：PCB 中用于管理进程的控制数据，包括进程状态、优先级、资源占用等信息。

系统态：同内核态 / 特权态，CPU 执行操作系统代码时所处的模式。

中断：异步事件信号，使 CPU 暂停当前进程转而执行中断处理程序。

进程映像：进程在内存中的完整表示，包括代码、数据、栈和 PCB。

任务：从操作系统角度看待的一个独立执行单位。

内核模式：CPU 的特权执行模式，可访问所有资源，执行所有指令。

进程克隆：创建新进程的一种方式，复制父进程的资源生成子进程。

时间片：操作系统分配给每个进程的 CPU 使用时间长度。

模式切换：CPU 从用户模式切换到内核模式（或反之）的过程，通常由系统调用或中断触发。

进程切换：CPU 从执行一个进程转为执行另一个进程的过程，需要保存和恢复上下文。

跟踪：记录和监控程序执行过程的技术，用于调试和性能分析。

新建态：进程刚被创建但尚未就绪的状态，操作系统正在为其分配资源。

程序状态字（PSW）：包含 CPU 当前状态信息的寄存器，如条件码、中断屏蔽位、CPU 模式位等。

陷阱：一种软件产生的中断，通常由程序错误（如除零）或系统调用触发。

父进程：创建了其他进程（子进程）的进程。

就绪态：进程已准备好运行，等待被调度程序选中分配到 CPU 的状态。

用户模式：CPU 的受限执行模式，用户程序运行在此模式，不能执行特权指令。

抢占：调度程序强制暂停当前运行的进程，将 CPU 分配给其他进程使用。

轮转：依次轮流为每个进程分配时间片的调度方式，进程用完时间片后回到就绪队列尾部。

# 第 4 章 线程 关键术语

术语
应用消息任务
纤程多线程线程
套管命名空间线程池
作业对象端口用户级线程
内核级线程进程用户模式调度（UMS）
轻量级进程

应用：面向最终用户的程序，运行在操作系统之上，利用系统提供的服务完成特定功能。

纤程：完全由用户级库调度的轻量级执行流，不需要内核参与，调度开销极小。

多线程：一个进程内包含多个并发执行的线程，共享进程的地址空间和资源。

任务：操作系统管理的独立执行单位，可以是进程或线程。

套管：一种将线程与完成端口等机制封装在一起的并发编程模型，简化线程管理。

消息：线程间或进程间传递的信息，用于通信和同步。

线程：进程内的一个执行序列，是 CPU 调度的基本单位，同一进程的线程共享地址空间。

作业对象：Windows 中用于管理一组相关进程的容器，可统一定义资源限制和安全属性。

命名空间：用于隔离和组织命名资源（如文件、目录）的逻辑边界，防止名称冲突。

线程池：预先创建并维护的一组线程，可以重复用于执行异步任务，避免频繁创建 / 销毁线程的开销。

内核级线程：由操作系统内核直接管理的线程，内核负责调度和切换，一个线程阻塞不影响同进程的其他线程。

端口：进程间通信的一种端点抽象，发送方将消息发送到端口，接收方从端口读取。

用户级线程：完全在用户空间通过线程库管理的线程，内核感知不到其存在，切换开销小但一个线程阻塞会阻塞整个进程。

轻量级进程（LWP）：用户级线程与内核级线程之间的中间层映射，每个 LWP 对应一个内核线程，支持用户线程到内核的绑定。

进程：正在运行的程序的实例，拥有独立的地址空间和资源，是资源分配的基本单位。

用户模式调度（UMS）：允许应用程序在用户模式下控制和调度线程，减少内核模式切换开销。

# 第 5 章 并发：互斥与同步 关键术语

术语
原子性临界资源互斥
二元信号量临界区互斥锁（互斥）
阻塞死锁无阻塞
忙等待直接寻址竞争条件
并发一般信号量信号量
并发进程间接寻址自旋等待
条件变量活锁饥饿
协同程序消息传递强信号量
计数信号量管程弱信号量

原子性：操作要么完整执行，要么完全不执行，中间状态不可被其他进程观测到的特性。

临界资源：一次只允许一个进程访问的共享资源，如打印机或共享变量。

互斥：保证同一时刻最多只有一个进程可以访问临界资源的约束条件。

二元信号量：取值仅为 0 或 1 的信号量，功能等同于互斥锁，用于保护互斥访问。

临界区：进程中访问临界资源的代码段，必须保证互斥执行。

互斥锁（互斥 /mutex）：用于实现互斥的锁机制，进入临界区前加锁，退出后解锁。

阻塞：进程因等待某个事件（如信号量、I/O 完成）而主动放弃 CPU，进入等待状态。

死锁：两个或多个进程相互等待对方持有的资源，导致都无法继续执行的状态。

无阻塞：进程不会因为等待资源而被挂起，通常通过非阻塞原语来尝试获取资源。

忙等待：进程反复检查某个条件是否满足，条件不满足时持续消耗 CPU 而非放弃 CPU 去睡眠。

直接寻址：直接使用源或接收者的标识符进行消息传递的通信方式。

竞争条件：多个进程并发访问共享数据，最终结果取决于进程执行的相对时序的不确定性情况。

并发：多个进程或线程在时间上重叠执行，从宏观看同时推进。

一般信号量：也称计数信号量，取值不限于 0/1，可用于管理多份同类资源，支持成组的进程同步。

信号量：用于进程间同步的整型变量机制，通过 wait（P）和 signal（V）原语操作。

并发进程：在同一时间段内同时（宏观上）执行的多个进程。

间接寻址：消息通过共享的信箱或端口进行传递，发送方和接收方不直接指定对方。

自旋等待：同忙等待，进程在获取锁之前反复循环检查锁状态而非放弃 CPU。

条件变量：管程内部用于进程等待和唤醒的同步机制，与 wait 和 signal 操作配合使用。

活锁：进程不断改变自身状态以响应其他进程，但无法取得进展的持续循环状态。

饥饿：某个进程长时间得不到所需资源或 CPU 服务，虽然系统整体仍在运行。

协同程序：能够在执行过程中挂起和恢复的特殊子程序，支持协作式多任务，控制权显式传递。

消息传递：进程间通过发送和接收消息来实现通信和同步的机制。

强信号量：等待进程按照 FIFO 顺序被唤醒的信号量，保证不会出现饥饿。

计数信号量：取值范围为 0 到 n 的信号量，用于管理有限数量的同类型资源。

管程：封装了共享数据和对共享数据的操作的编程语言级同步结构，同一时刻只允许一个进程进入管程执行。

弱信号量：不保证等待进程的唤醒顺序的信号量，可能导致饥饿。

银行家算法：经典的死锁避免算法，在分配资源前判断系统是否会进入不安全状态，仅安全的请求才被批准。

敏感区域：可能被多个进程并发访问并引发竞争条件的代码区域，需要同步保护。

抢占：强制从持有资源的进程中收回资源。打破死锁条件时可能使用资源抢占策略。

循环等待：死锁的必要条件之一，进程间形成封闭的等待环：P1 等待 P2 持有的资源，P2 等待 P3 的……PN 等待 P1 的资源。

占有且等待：死锁必要条件之一，进程在持有一部分资源的同时请求新的资源，且请求被阻塞时不释放已持有的资源。

资源分配图：用有向图表示资源分配情况的模型，进程→资源边表示请求，资源→进程边表示分配。出现环则可能死锁。

可消耗资源：使用后即消失的资源（如信号、消息、中断），可被创建和销毁。

联合进程图：描述进程在并发执行中各步骤间先后关系的图形表示，用于分析死锁可能性。

可重用资源：使用后不会消失、可以被多个进程轮流使用的资源（如 CPU、内存、文件）。

死锁：两个或以上进程因互相等待对方持有的资源而永远无法推进的状态。

内存屏障：一种硬件 / 编译器指令，用于限制内存操作的执行顺序，保证并发程序的正确可见性。

安全状态：存在某种执行顺序使每个进程都能顺利完成且不会发生死锁的系统状态。

死锁避免：在运行过程中持续监控资源分配请求，仅当分配后系统仍处于安全状态时才分配。

消息：进程间传递的信息单位，可用于同步和数据交换。

自旋锁：忙等待形式的互斥锁，线程在获取锁之前持续循环检查锁状态，适用于临界区极短的场景。

死锁检测：允许死锁发生，但通过定期检查资源分配图来发现死锁，并采取措施恢复。

互斥：死锁必要条件之一，资源一次只能被一个进程使用，申请时得不到就等待。

管道：一种进程间通信机制，数据以 FIFO 方式在读写端之间单向传输。

死锁预防：通过破坏死锁的四个必要条件之一（互斥、占有等待、不可抢占、循环等待）来从根本上阻止死锁的发生。

不安全状态：不存在确保所有进程都能顺利完成的执行序列，但在该状态下未必已经发生死锁。

动态分区：根据进程的实际大小动态创建分区，分区大小和数量可变，减少内部碎片。

逻辑组织：从程序设计角度看内存的划分方式，通常按模块或段组织。

可重定位加载：加载程序时将逻辑地址转换为物理地址的方式，程序可以在内存中任意位置运行。

动态运行时加载：程序运行到某个模块时才将该模块加载到内存。实现延迟加载，节省内存。

内存管理：操作系统负责分配、回收、保护和共享内存资源的子系统。

到达速率：新进程到达就绪队列的速率，是调度算法分析和系统性能建模的重要参数。

中程调度程序：负责挂起和换入进程的调度层，通过交换操作控制系统并发度。

短程调度程序：最频繁执行的调度器，从就绪进程中选择下一个进程分配 CPU。

分派器：将 CPU 控制权交给选中的进程，包括上下文切换、切换到用户模式和跳转到程序正确位置。

多级反馈队列：允许进程在不同优先级队列之间迁移的调度算法，根据进程行为（如 CPU 使用率）动态调整优先级。兼顾短进程优先和长进程公平。

吞吐量：单位时间内系统完成的作业数量，是调度性能的核心指标。

指数平均：一种估算下一个 CPU 区间长度的预测方法，加权历史值：τ_{n+1}=α・t_n+(1-α)・τ_n。

时间片：分时系统中分配给每个进程的 CPU 时间长度，用完即被抢占。

公平共享调度：在用户或组之间按预定比例而非按进程平均分配 CPU 资源。

驻留时间：作业从进入系统到被处理完毕的总时长，等于等待时间加服务时间。

周转时间：从作业提交到完成的总时间，含等待时间和服务时间。

公平性：各进程或用户获得 CPU 资源的合理程度，防止某个进程长期被忽略。

响应时间：在交互系统中，从用户提交请求到系统产生首次响应的时间。

利用率：CPU 或其他资源处于使用状态的时间百分比。

先来先服务（FCFS）：最简单的调度算法，按进程到达的先后顺序分配 CPU，非抢占。

轮转（RR）：为每个进程分配固定时间片，轮流执行，用完后排到队尾，确保公平响应。

等待时间：进程在就绪队列中等待 CPU 的时间总和。

先进先出（FIFO）：同先来先服务，按到达顺序服务，先到的先执行。

调度优先级：为每个进程分配一个优先级值，调度时优先选择高优先级进程，是调度算法的重要维度。

非周期性任务：没有固定到达周期、执行时间不可预测的任务，软实时系统中常见。

负载分配：在多处理器系统中将任务分配到各个处理器以平衡负载的策略。

速率单调调度（RMS）：固定优先级抢占式调度算法，按照任务的周期分配优先级：周期越短，优先级越高。

限期调度：以任务的完成截止时间为调度约束条件的调度方式，确保任务在截止时间前完成。

周期性任务：以固定时间间隔反复到达、每次执行时间可预测的实时任务。

实时操作系统（RTOS）：能够在确定的时间内对事件做出响应，保证任务在截止时间前完成的操作系统。

确定性操作系统：操作系统操作的时间延迟有确定上界，行为可预测，满足实时系统对时间约束的要求。

优先级天花板：使用信号量的最高优先级任务的优先级值，防止优先级反转 —— 任务进入临界区后其优先级被临时提升到此天花板。

实时调度：满足实时系统时间约束条件的调度方法，硬实时确保不超截止时间，软实时尽量满足。

故障弱化运行：系统在部分组件故障时仍能以降低的性能继续运行的能力，实时系统的重要特性。

优先级继承：当高优先级任务因低优先级任务持有的锁而阻塞时，低优先级任务临时继承高优先级，减缓优先级反转。

响应性：系统对事件做出反应的速度，是实时系统的重要指标。

组调度：将一组相关线程作为一个整体进行调度，它们同时被分配 CPU 运行，减少组内通信等待。

优先级反转：高优先级任务因等待低优先级任务持有的资源而被阻塞，且可能被中优先级任务进一步延迟的现象。

软实时任务：有时间期限要求，但偶尔超期不会造成灾难性后果的任务，超期后结果仍然有效。

粒度：任务或调度单位的大小，细粒度（调度频繁，响应快但开销大）或粗粒度。

处理器亲和性：进程 / 线程倾向于继续在同一个处理器上执行，以利用该处理器缓存中保留的数据。

线程调度：为线程分配 CPU 的决策过程，可能由内核或用户级线程库完成。

硬亲和性：进程 / 线程固定绑定到指定处理器核心，不允许调度到其他核心。

拉取机制：空闲处理器主动从其他处理器的就绪队列中拉取任务来执行的负载平衡机制。

无界优先级反转：优先级反转的时间没有上界，高优先级任务可能被无限延迟，实时系统必须防止。

硬实时任务：必须在截止时间前完成的任务，超期会导致系统失败或灾难性后果。

推送机制：忙碌处理器将任务推送到空闲处理器执行，是负载平衡策略之一。

循环缓冲区：固定大小的缓冲区，当填满后从头部重新开始写入，形成逻辑环路。

I/O 缓冲区：用于临时存放 I/O 操作数据的存储区域，平滑设备和 CPU 之间的速度差异。

扇区：磁盘盘片上数据存储的最小物理单位，通常为 512 字节或 4KB。

设备 I/O：直接控制物理设备进行数据读写的底层操作。

I/O 处理器：具有独立指令集的专用处理器，处理 I/O 操作，比 I/O 通道更智能。

面向流的设备：以字节流形式访问的设备（如键盘、网卡），没有固定块结构。

直接内存访问（DMA）：允许 I/O 设备直接与内存传输数据，无需 CPU 逐一处理每个数据字。

I/O 通道：专用的 I/O 处理器，可执行通道程序来控制多台设备的大量数据传输。

分条：RAID 中将数据按条带分布在多块磁盘上的技术，实现并行读写提高性能。

磁盘访问时间：一次磁盘读写操作的总时间 = 寻道时间 + 旋转延迟 + 传输时间。

逻辑 I/O：文件系统层面的逻辑数据处理，不涉及物理设备细节。

磁道：磁盘盘片表面的同心圆环数据轨道，每个磁道包含多个扇区。

磁盘缓存：磁盘或操作系统维护的高速缓冲区，存放最近访问的或预取的磁盘数据，减少实际磁盘访问。

不可移动磁盘：盘片永久固定在驱动器内的磁盘，即固定磁盘（如内置硬盘）。

传输时间：数据从磁盘缓冲区传输到主存的时间，取决于旋转速度和数据量。

双缓冲：使用两个缓冲区交替工作，处理器处理一个缓冲区时设备填充另一个，实现计算与 I/O 的重叠。

程序控制 I/O：CPU 通过反复轮询 I/O 模块的状态寄存器来控制 I/O 操作，每传输一个字 CPU 都要忙等。

间隙：磁盘扇区之间的间隔，用于分隔扇区和包含纠错信息。

# 第 12 章 文件管理 关键术语

术语
访问方法文件分配主文件表（MFT）

访问方法：应用程序访问文件中数据的逻辑方式，如顺序访问、直接访问（随机访问）、索引访问等。

文件分配：决定文件数据在磁盘上如何存放的策略，包括连续分配、链式分配和索引分配三种基本方式。

主文件表（MFT）：NTFS 文件系统中的核心元数据文件，为每个文件存储一组属性，包括文件名、时间戳、数据位置等。

基本文件系统：文件系统层次架构的最底层组件，负责向设备驱动程序发出实际的磁盘读写命令。

文件分配表（FAT）：FAT 文件系统中记录每个簇的分配状态和文件簇链的表格，通过链表方式跟踪文件占用的磁盘空间。

分区引导扇区：磁盘分区中的第一个扇区，包含该分区的文件系统信息和引导代码。

位表：用位图表示磁盘空闲空间的方法，每个位对应一个磁盘块，0 表示空闲，1 表示已分配。

文件目录：将文件名映射到文件控制块 / 索引节点的目录结构，组织文件的命名空间。

路径名：从根目录或当前目录出发到达目标文件的目录遍历序列，用于定位文件。

块：磁盘存储的基本分配单位，由多个连续扇区组成。

文件管理系统：操作系统负责文件存储、检索和管理的那一部分子系统。

物理 I/O：直接与磁盘硬件交互的底层输入输出操作，发生在设备驱动层。

文件名：用户可见的文件标识符，目录中用于查找文件的主要键。

堆：文件的一种数据结构形式，数据按任意顺序存放，无特定结构，适用于自由格式访问。

链式文件分配：每个文件块中包含指向下一块地址的指针，通过指针链将属于同一文件的块串联起来，无外部碎片。

散列文件：通过散列函数将关键字映射到桶 / 块位置的文件组织方式，支持快速按关键字查找。

分区：磁盘上被格式化为独立文件系统的一段连续区域，每个分区可包含不同文件系统。

连续文件分配：将文件的全部数据块存储在磁盘上物理连续的区域，访问速度快但会产生外部碎片。

索引文件：文件组织为记录数组，通过一个或多个索引表实现按关键字快速查找。数据库系统中常见。

记录：文件中逻辑上相关的一组字段组合，是文件的基本逻辑访问单位。

索引文件分配：为每个文件维护一个索引块，索引块中存放该文件所有数据块的磁盘地址，避免链式分配的随机访问慢的问题。

顺序文件：记录按某个关键字有序排列的文件组织形式，常用在需要按顺序处理的场景。

数据库：结构化存储的数据集合，由文件系统或多文件组成，支持高效查询和事务操作。

索引顺序文件：综合索引文件（随机访问）和顺序文件（批量处理）优点的文件组织方式，记录有索引又有物理顺序。

系统目录：操作系统根目录，文件系统命名空间的顶层起点。

磁盘分配表：记录磁盘上各分配单位（块 / 簇）使用状态的数据结构，如 FAT 文件系统中的 FAT 或 Unix 中的空闲块位图。

索引节点（inode）：Unix/Linux 文件系统中存储文件元数据和数据块指针的核心数据结构，一个文件对应一个 inode。

系统文件：操作系统自身使用的文件，通常对用户不可见，如配置文件、日志文件、可执行文件等。

域：一组相关字段组成的逻辑数据单位，是记录的组成部分。

关键域：记录中用于唯一标识该记录的域（主键），或用于建立关联的域（外键）。

虚拟文件系统（VFS）：操作系统提供的文件系统抽象层，使不同文件系统可以通过统一接口被访问。

文件：命名的一组相关信息的集合，存储在辅存上，是用户视角下数据持久化的基本单位。

逻辑 I/O：文件系统层次架构中的较高层，处理逻辑记录读写，不关心底层物理存储细节。

工作目录：当前进程所在的目录，相对路径名的解析起点。

# 考试趋势提示

  • PV 操作与同步问题：408 卷几乎每年一道大题，信号量的定义和 P/V 语义必须准确。
  • 分页地址转换：必考大题，逻辑地址→物理地址的三步骤（拆页号 + 偏移→查页表→拼物理地址），注意多级页表和 TLB 计算。
  • 银行家算法：轮流出现，Given 资源总量和分配情况，判断安全状态并给出安全序列。
  • 磁盘调度：平均寻道长度计算是固定题型。
  • 综合对比题：近年倾向于综合考察，如 "分页 + 虚拟内存 + 页面置换" 联合出题，或 "进程状态 + 信号量 + 死锁" 组合。
