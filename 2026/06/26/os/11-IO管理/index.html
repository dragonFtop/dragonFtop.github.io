# I/O 管理

# I/O 设备的概念和分类

I/O 即输入（Input）/ 输出（Output）。

# 按使用特性分类

类型举例
人机交互外部设备键盘、鼠标、显示器、打印机
存储设备磁盘、磁带、SSD
网络通信设备网卡

# 按传输速率分类

  • 低速设备（键盘、鼠标）
  • 中速设备（打印机）
  • 高速设备（磁盘、网卡）

# 按信息交换单位分类

类型特点
块设备以数据块为单位传输，可寻址，如磁盘
字符设备以字符流为单位传输，不可寻址，如键盘

# I/O 控制器

I/O 设备由机械部件和电子部件（I/O 控制器）组成。

# 功能

功能对应组件
接受和识别 CPU 发出的命令控制寄存器
向 CPU 报告设备的状态状态寄存器
数据交换数据寄存器
地址识别地址译码器

# 编址方式

  • 内存映射 I/O ：设备寄存器与内存统一编址
  • 寄存器独立编址：设备寄存器有独立的地址空间

# I/O 控制方式

# 1. 程序直接控制方式

轮询 ：CPU 不断检查设备状态

  • 每次读 / 写一个字
  • 实现简单，但会使 CPU 忙等（busy waiting）
  • CPU 干预频繁

# 2. 中断驱动方式

  • CPU 发出 I/O 指令后继续做其他事情
  • 设备完成 I/O 后向 CPU 发中断信号
  • 每次读 / 写一个字
  • CPU 和 I/O 可并行工作
  • 大量中断会降低 CPU 效率

# 3. DMA 方式（直接存储器存取）

数据直接从设备传输到内存，减少 CPU 干预。

DMA 控制器的寄存器：

寄存器功能
DR（数据寄存器）存放传输的数据
MAR（内存地址寄存器）存放目标内存地址
DC（数据计数器）剩余读写字节数
CR（命令 / 状态寄存器）存放命令和状态

特点：

  • 数据传输单位：连续的多个块
  • 仅在传输开始和结束时需要 CPU 干预

# 4. 通道控制方式

  • 通道是一种专门处理 I/O 操作的专用处理器，指令集精简但专注于 I/O 控制
    通道程序 ：任务清单，由通道执行
  • CPU 发送 I/O 命令给通道，然后通道自主处理 I/O
  • 处理完成后，通道向 CPU 发送中断信号

# I/O 软件层次结构

层次功能
用户层软件实现与用户交互的接口，向上提供方便易用的库函数
设备独立性软件提供统一的调用接口（read/write）；设备保护、差错处理、设备分配与回收、数据缓冲区管理；建立逻辑设备名到物理设备名的映射
设备驱动程序设置设备寄存器、检查设备状态，与硬件直接交互
中断处理程序进行中断处理
硬件执行 I/O 操作（机械部件 + 电子部件）

# I/O 核心子系统

层次核心功能
用户层软件假脱机系统（SPOOLing）
设备独立性软件I/O 调度、设备保护、设备分配与回收、缓冲区管理
设备驱动程序设备具体操作
中断处理程序中断响应与处理
硬件物理执行

# 假脱机技术（SPOOLing）

# 脱机技术

脱离主机的控制进行输入 / 输出控制。SPOOLing 系统必须要有多道程序并发执行。

# 实现原理

组件说明
输入井 / 输出井磁盘上用于暂存输入 / 输出数据的区域
输入进程 / 输出进程负责数据从输入井到内存 / 从内存到输出井
输入缓冲区 / 输出缓冲区内存中的缓冲区域

# 共享打印机

通过 SPOOLing 实现打印机共享：多个用户进程的输出先暂存在输出井中，由输出进程统一调度打印。

# 设备的分配与回收

# 设备分配时应考虑的因素

设备的固有属性 ：独占设备、共享设备、虚拟设备
设备分配算法 ：先来先服务、优先级高者优先
分配中的安全性 ：为进程分配设备后将进程阻塞，本次 I/O 完成后才唤醒

# 静态分配 vs 动态分配

方式说明
静态分配进程运行前分配全部所需资源，运行结束后归还
动态分配运行中动态申请和释放设备

# 设备分配的数据结构（树形）

数据结构包含信息
系统设备表 SDT设备类型、设备标识符、DCT、驱动程序入口
设备控制表 DCT设备类型、设备标识符、设备状态、指向控制器表的指针、设备队列指针
控制器控制表 COCT控制器标识符、控制器状态、指向通道表的指针、控制器队列指针
通道控制表 CHCT通道标识符、通道状态、连接控制器表首址、通道队列指针

# 设备分配的步骤

根据进程请求的物理设备名 -> SDT -> DCT -> COCT -> CHCT -> 通道

# 改进方法

建立逻辑设备名到物理设备名的映射，实现设备独立性。

# 缓冲区管理

# 缓冲区的作用

作用说明
缓和速度差异CPU 与 I/O 设备之间速度不匹配
减少中断频率批量传输，减少 CPU 中断次数
解决数据粒度不匹配组装小块数据成大块，或拆散大块数据
提高并行性CPU 和 I/O 设备可并行工作

双缓冲内存中分配两块缓冲区，交替使用max(T, C + M)

C = CPU 处理时间，T = I/O 传输时间，M = 数据搬移时间

# 缓冲池

由系统中共享的缓冲区组成，分为三个队列：

  • 空缓冲队列
  • 装满输入数据的缓冲队列
  • 装满输出数据的缓冲队列

# 如何提高 I/O 效率

# 为什么需要提高 I/O 效率

实际情况中 I/O 的运行速度非常慢，在一些场景中 I/O 会拖累 CPU，而降低系统性能：

  1. 将内存中的数据存入磁盘中，CPU 能极快地输入数据给 I/O，但是 I/O 一次只能读取少量的数据，所以 CPU 只能慢下来等 I/O，如果没有合适的调度的话，CPU 资源将在等待 I/O 中浪费掉。
  2. 另一个例子 CPU 需要读取 I/O 设备的输入，但是 I/O 运行速度很慢，CPU 必须等待 I/O 设备的输入。

# I/O 缓冲

I/O 缓冲可以让 CPU 快速将数据存储在缓冲区中，I/O 设备在缓冲区中读取数据。也就是将数据的读写变为 异步执行 在此种情况下下 CPU 效率更高。

  • CPU 将 100MB 数据甩入缓冲区后直接走人，然后 I/O，比如磁盘自己慢慢地从缓冲区中读取。

根据缓冲单元的数量可以分为单缓冲、双缓冲、循环缓冲。

  • 单缓冲
  • 双缓冲
  • 循环缓冲

Tr 和 Td 时间都是由物理机械所决定的，无法通过调度来实现优化，所以这里只能通过调度来优化寻道时间。
具体如何优化寻道时间？
通过使用合适的寻道顺序的调度可以优化。
磁盘调度的性能指标：

  • 平均寻道长度
  • 最大寻道延迟

# 磁盘调度的几种策略

  1. 先入先出（FIFO）：按照请求访问的顺序进行寻道。
  2. 后入先出（LIFO）：按照请求访问的顺序的逆序进行寻道。
  3. 最短服务时间优先（SSTF）：选择对于当前磁头位置最近的目标进行寻道（类似于贪心算法）。
  4. 磁道扫描（Scan）：选择当前磁臂移动的方向上距离当前磁道最近的请求。到达磁盘一端后调转方向扫描。
  5. 循环扫描：与扫描的区别为到达磁盘的一端后不会调转方向，会继续当前方向（转圈的方式）寻找距离当前磁道距离最近的请求。

前两种调度没有优化平均寻道时间，第三种不能保证最优寻道时间，但是通常性能效果为：SSTF>FIFO>=LIFO
其中 SSTF 存在公平性问题（偏安全），比如某个进程发出大量位置较近的请求，这个进程会较长时间抢占 I/O。
