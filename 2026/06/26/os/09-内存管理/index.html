# 内存管理

# 内存的基础知识

  • 存储单元 ：每个地址对应一个存储单元
    内存地址 ：用于标识存储单元位置

# 指令的工作原理

程序执行时，CPU 使用的是逻辑地址，需要通过地址转换机制映射到物理地址。

逻辑地址 （相对地址）：程序视角的地址，从程序起始位置开始的偏移
物理地址 ：内存中的实际地址，硬件访问使用的最终地址

# 进程运行的基本原理

物理地址 ：内存中的实际地址，硬件访问使用的最终地址

# 三种链接方式

方式说明
静态链接程序运行前，将各目标模块及库函数连接成一个完整的可执行文件
装入时动态链接将各目标模块装入内存时，边装入边链接
运行时动态链接程序执行中需要该模块时才对它进行链接，便于修改和更新

# 三种装入方式

方式说明
绝对装入编译时就知道程序放在内存的哪个位置
静态重定位装入时将逻辑地址转换为物理地址
动态重定位地址转换推迟到程序真正要执行时才进行

# 逻辑地址 vs 物理地址

逻辑地址 （相对地址）：程序视角的地址
物理地址 ：内存中的实际地址

# 内存管理的概念

内存管理的四大功能：

内存空间的分配与回收
内存空间的扩充 （虚拟内存技术）
地址转换 （逻辑地址 <-> 物理地址）
存储保护

  • 设置上下限寄存器
  • 采用重定位寄存器（基址寄存器）+ 界地址寄存器（限长寄存器）

# 覆盖与交换

内存空间扩充的两种技术：

# 覆盖技术

将程序分为多个段：
固定区 ：常驻内存，调入后不调出
覆盖区 ：不常用的段，用到了调入，不用时调出

# 交换技术

内存空间紧张时，系统将某些进程暂时换出外存，把外存中已具备运行条件的进程换入内存。（PCB 常驻内存，不会被换出）

# 连续分配管理方式

方式说明特点
单一连续分配内存分为系统区（低地址）和用户区，用户区独享简单，利用率低
固定分区分配用户区划分为若干固定分区分区大小可相等或不相等，有分区说明表
动态分区分配根据进程大小动态建立分区灵活，但会产生外部碎片

# 碎片

内部碎片 ：分配给某进程的内存区域中，有些部分没有被用上
外部碎片 ：空闲分区因太小而难以利用（可采用紧凑技术解决）

# 动态分区分配算法

# 1. 首次适应算法（First Fit）

每次从低地址开始查找，找到第一个能满足大小的空闲分区。

# 2. 最佳适应算法（Best Fit）

空闲分区按容量递增次序链接，优先使用更小的空闲区，尽可能留下大片空闲区。

缺点：会产生很多难以利用的小碎片。

# 3. 最坏适应算法（Worst Fit）

按容量递减次序排列，每次尽可能用大的分区。

缺点：大空间容易被快速用完。

# 4. 邻近适应算法（Next Fit）

从上次查找结束的位置开始检索。

缺点：大空间容易被用完，整体性能不如首次适应。

# 基本分页存储管理

将内存分为大小相等的 "页框"（帧），将进程空间也分为大小相等的 "页面"，以页框为单位分配。

  • 物理地址 = 页面起始位置 + 偏移量
  • 计算机用 2 的整数幂表示页面大小
    页表：存放页号和块号的对应关系

# 基本地址变换机构

页表寄存器（PTR）存放页表在内存中的起始地址 F 和页表长度 M。进程未执行时，这些信息存放在 PCB 中；当进程被调度时，操作系统内核将它们放入页表寄存器。

地址变换过程：

物理地址 = 页面起始位置 + 偏移量，计算机中用 2 的整数幂表示页面大小，这样可以用位运算快速拆分逻辑地址。

# 具有快表的地址变换机构

局部性原理 ：

  • 时间局部性 ：访问某个变量后，不久的将来还会被访问
  • 空间局部性 ：访问了某个存储单元，不久之后其附近的存储单元也很有可能被访问

快表（TLB） ：又称联想寄存器，是一种访问速度比内存快很多的高速缓冲存储器，用来存放当前访问的若干页表项，以加速地址变换过程。内存中的页表常称为 "慢表"。

# 两级页表

单级页表存在两个问题：

  1. 所有页表项必须连续存放，页表过大时需要很大的连续空间
  2. 一段时间内并非所有页面都用得到，没必要让整个页表常驻内存

解决方案 ：将长长的页表再分页，形成两级页表。

  • 逻辑地址结构：（一级页号、二级页号、页内偏移量）
  • 页目录表 / 外层页表 / 顶级页表
  • N 级页表访问一个逻辑地址需要 N+1 次访存（无快表时）
  • 各级页表大小不能超过一个页面

# 分段 vs 分页

维度分页分段
划分依据物理单位，大小固定逻辑单位，按程序自身逻辑关系划分
地址维度一维二维（段号 + 段内地址）
访存次数2 次2 次
信息共享与保护不方便更容易实现
  • 纯代码 / 可重入代码：不能被修改的代码，不属于临界资源，可以被多个进程共享。

# 段表

每个程序被分段后，用段表记录该程序在内存中存放的位置。

段表结构：段号 | 段长 | 基址

# 段页式管理方式

# 设计动机

  • 分页：利用率高、碎片少，但不方便信息共享和保护
  • 分段：方便信息共享和保护，但段长大时容易产生外部碎片

# 实现

段号 + 页号 + 页内偏移量：段页式的地址结构，是二维的。

# 段表、页表

段表 ：每个段对应一个段表项，记录该段的页表长度和页表始址
页表 ：每个段内有一张页表，记录该段内各页面对应的内存块号

# 地址变换过程（4 步）

若引入快表（TLB），命中时可减少访存次数至 1 次。

# 虚拟内存

# 传统存储管理的缺点

一次性 ：作业必须全部装入内存后才能开始运行，导致并发性下降
驻留性 ：一旦作业被装入内存，就会一直驻留在内存

# 虚拟内存的特征

特征说明
多次性无需一次性全部装入，可被分成多次调入内存
对换性运行时允许换入换出
虚拟性从逻辑上扩充了内存容量
  • 虚拟内存最大容量由计算机地址结构确定
  • 虚拟内存实际容量 = min (内存 + 外存容量，CPU 寻址范围)

示例：某计算机地址结构为 32 位，按字节编址，内存 512MB，外存 2GB

  • 虚拟内存最大容量 = 2^32 B = 4GB
  • 虚拟内存实际容量 = min (4GB, 512MB + 2GB) = 2GB + 512MB

# 请求分页管理方式

# 页表机制

请求分页存储的页表包含：

字段说明
内存块号对应物理块
状态位页面是否在内存中
访问字段记录最近访问情况
修改位页面内容是否被修改
外存地址页面在外存中的位置

# 缺页中断

属于内中断（异常），可被修复。当访问的页面不在内存中时，触发缺页中断，由操作系统将所需页面调入内存。

# 页面置换算法

# 1. 最佳置换算法（OPT）

每次淘汰以后永不使用或在最长时间内不再被访问的页面。保证最低的缺页率，但实际中无法预知未来的访问序列，仅作为理论标杆。

# 2. 先进先出置换算法（FIFO）

每次淘汰最早进入内存的页面。

Belady 异常：当分配的内存块增大时，缺页次数反而增加。

# 3. 最近最久未使用置换算法（LRU）

每次淘汰最近最久未使用的页面。性能接近 OPT，但实现开销较大。

# 4. 时钟置换算法（CLOCK / 最近未用 NRU）

简单的 CLOCK：最多经历两轮扫描。访问位初始为 1，扫描遇到 1 则置 0，扫描遇到 0 则淘汰。

# 5. 改进型时钟置换算法

优先淘汰没有被修改过的页面（修改位 = 0），因为不需要进行 I/O 回写操作。

淘汰优先级： (0,0) -> (0,1) -> (1,0) -> (1,1)
（访问位，修改位）

# 页面分配策略

# 驻留集

请求分页存储管理中给进程分配的物理块的集合。

# 分配与置换策略

策略说明
固定分配局部替换驻留集大小不变，只能置换自己的物理块
可变分配全局替换可将系统保留的空闲物理块分配给缺页进程
可变分配局部替换动态调整驻留集大小，但只能置换自己的物理块

# 调页时机

预调页策略 ：运行前一次调入若干个相邻页面
请求调页策略 ：运行时缺页再调入

# 从何处调页

  • 文件区 ：速度慢，采用离散分配方式
  • 对换区 ：速度快，采用连续分配方式

# 抖动（颠簸）现象

刚刚换出的又要换入，刚刚换入的又要换出。根因是分配的物理块不够用。

# 工作集

在某段时间间隔内，进程实际访问页面的集合。通过工作集模型可以有效防止抖动。
