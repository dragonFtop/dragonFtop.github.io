# 并发 - 题目解析

共 20 题，100 分，全部为单选题。主题：并发、互斥与同步。

# 1. 以下哪个术语不属于 "并发 / Concurrency" 的 "核心的四个要解决的问题" 的范畴（）

A. 同步 / Synchronization
B. 互斥 / Mutual Exclusion
C. 死锁 / Deadlock
D. 进程状态的跟踪

  • 正确答案：D

解析 ：并发编程需要解决的四大核心问题：

  • 互斥（Mutual Exclusion） ：同一时刻只有一个进程进入临界区
  • 同步（Synchronization） ：协调进程之间的执行顺序
  • 死锁（Deadlock） ：进程相互等待对方释放资源
  • 饥饿（Starvation） ：进程长期得不到执行

"进程状态的跟踪" 是进程管理的基础功能，但不属于并发核心要解决的四大问题。

# 2. 多个进程或线程在读写一个共享数据时，如果最终的运行结果依赖于它们执行的相对时间，这个现象被称为（）

A. 同步
B. 互斥
C. 死锁
D. 条件竞争 / Race Condition

解析

  • 正确答案：D

解析 ：竞态条件（Race Condition）的定义就是：多个进程 / 线程并发访问共享数据，最终结果取决于它们执行的相对时序（谁先谁后）。这是并发编程中最核心的 bug 来源，必须通过同步机制（锁、信号量等）来消除。

# 3. 当一个进程在临界区访问某个共享资源时，其他进程不能进入该临界区访问此共享资源的情形，它被称为（）

A. 同步
B. 互斥
C. 死锁
D. 条件竞争 / Race Condition

解析

  • 正确答案：B ：互斥（Mutual Exclusion）的定义：当一个进程正在访问临界资源（在临界区内）时，其他任何进程都不能进入访问同一临界资源的临界区。互斥是保护共享数据一致性的基本手段。

区分：

  • 互斥 → "不能同时进"（规则）
  • 竞态 → 没有互斥保护导致的错误结果（后果）
  • 同步 → 按特定顺序执行（协作）

# 4. 为了在进程所竞争的关键资源上实现互斥，同一时间只允许一个程序（）

A. 展示合作
B. 进入程序的临界区
C. 进行消息传递
D. 以上都不是

解析

进入临界区进入临界区 （访问临界资源的那段代码）。选项中 "进入程序的临界区" 就是互斥约束的精确描述。

临界区（Critical Section）= 进程中访问临界资源的那段代码。

# 5. 临界区指的是：并发进程中访问共享变量的（）

A. 堆栈 / Stack 段
B. 堆 / Heap 段
C. 数据 / Data 段
D. 代码 / Code 段

  • 正确答案：D

解析 ：临界区（Critical Section）是访问临界资源的那段代码，属于代码段。它不是数据段、堆或栈中的数据本身，而是操作这些共享数据的代码逻辑。

# 6. 实现互斥需要满足下列哪些要求（）

A. 一次只允许一个进程进入临界区
B. 一个进程驻留在临界区中的时间必须是有限的
C. 对相关进程的执行速度没有任何要求和限制
D. 以上都是

  • 正确答案：D

解析 ：互斥的四个条件（也是临界区设计原则）：
空闲让进空闲让进 （A）：临界区空闲时，允许一个进程进入
忙则等待忙则等待 ：已有进程在临界区时，其他进程必须等待
有限等待有限等待 （B）：进程在临界区外等待的时间是有限的 → 避免饥饿
让权等待让权等待 ：等待进入临界区时应释放 CPU，不能忙等

选项 C 意味着不能对进程的执行速度做任何假设（不同机器、不同调度下速度不同），这是正确的设计原则。

# 7. 为了实现互斥，可以采用哪些方法 / 技术（）

A. 关中断
B. 通过特殊的硬件指令
C. 信号量
D. 以上均可

  • 正确答案：D

解析 ：实现互斥的三种层次的方法：

方法说明局限
关中断进入临界区前禁止中断，出来再开启仅适用于单核；影响整个系统
特殊硬件指令TSL（Test and Set Lock）、Swap/XCHG、CAS 等原子指令可能忙等（不让权等待）
信号量操作系统层面的同步原语，支持阻塞 / 唤醒需要内核支持

三种方法都可以实现互斥，各有适用场景。

# 8. 没有指定进程从队列中移除的顺序的信号量称为（）

A. 弱信号量
B. 强信号量
C. 二元信号量
D. 以上都不是

  • 正确答案：A

解析 ：信号量按队列出队策略分为：

  • 弱信号量（Weak Semaphore） ：不规定等待进程从队列中移除的顺序，谁先被唤醒不作保证
  • 强信号量（Strong Semaphore） ：按 FIFO（先进先出）顺序移除，保证公平性
  • 二元信号量（Binary Semaphore） ：值只能为 0 或 1，类似互斥锁

# 9. 生产者 / 消费者问题不能视为一个特殊的读者 / 写者问题的原因是（）

A. 生产者 / 消费者各自一个，而读者 / 写者各自可以有多个
B. 生产者和消费者其实同时是读者和写者
C. 当写者执行写入的时候，读者也可以执行读取；而生产者 / 消费者不同
D. 以上都不是

这是本题最容易出错的地方。生产者 / 消费者问题不是读者 / 写者问题的特例，关键原因是：

消费者（需要检查缓冲区是否已满）→ 既是写者也是读者
读者（需要更新缓冲区指针和计数器）→ 既是读者也是写者

而读者 / 写者问题中，读者只读不写，写者只写（通常不读），角色是分离的。生产者 / 消费者中双方同时具有读者和写者的特性，因此不能简单归类。

# 10. 读者 / 写者问题需要要求满足的条件有（）

A. 多个写者可同时写某个共享文件
B. 多个读者可同时读某个共享文件
C. 当写者在写某个共享文件时，读者也可以读取这个共享文件
D. 以上都不是

写写

  • 正确答案：B ：读者 / 写者问题的核心约束：
    解析 （B 正确）：读操作不改变数据，允许并发
    多个读者可同时读多个读者可同时读 ：写操作改变数据，必须互斥
    一次只能一个写者写一次只能一个写者写 ：写时排他，读者也不能读
    写者写时其他人不能访问写者写时其他人不能访问

选项 A 错误（多个写者不能同时写），选项 C 错误（写时不能读）。

# 11. 若有 4 个进程共享同一程序段，而且每次最多允许 3 个进程进入该程序段，则信号量的变化范围是（）

A. 3, 2, 1, 0
B. 3, 2, 1, 0, -1
C. 4, 3, 2, 1, 0
D. 2, 1, 0, -1, -2

  • 正确答案：B

解析 ：

  • 信号量初始值 = 最大允许进入的进程数 = 3
  • 当 0 个进程进入：S = 3
  • 当 3 个进程进入：S = 0
  • 当第 4 个进程试图进入但被阻塞：S = -1（负数绝对值 = 等待进程数）

所以信号量的变化范围是 3, 2, 1, 0, -1。

# 12. 下列对临界区的论述中，正确的是（）

A. 临界区是指进程中用于实现进程互斥的那段代码
B. 临界区是指进程中用于实现进程同步的那段代码
C. 临界区是指进程中用于实现进程通信的那段代码
D. 临界区是指进程中用于访问临界资源的那段代码

  • 正确答案：D

解析 的那段代码。注意区分：

  • 临界区 ≠ 实现互斥的代码（那是 "进入区" 和 "退出区"，即上锁 / 解锁代码）
  • 临界区 ≠ 同步代码
  • 临界区 = 真正操作共享数据的那段业务代码

# 13. 一个正在访问临界资源的进程由于申请等待 I/O 操作而被中断时，它（）

A. 允许其他进程进入与该进程相关的临界区
B. 不允许其他进程进入任何临界区
C. 允许其他进程抢占处理器，但不得进入该进程的临界区
D. 不允许任何进程抢占处理器

访问临界资源访问临界资源

  • 正确答案：C 的临界区。但：
  • 其他进程可以抢占处理器（C 正确，D 错误）：被中断后 CPU 空闲，可以分配给其他进程
  • 其他进程不能进入该临界区（A 错误）：临界资源仍被占用
  • 其他进程可以进入别的临界区（B 错误）：不同临界资源互不影响

# 14. 以下关于管程的叙述中，错误的是（）

A. 管程是进程同步工具，解决信号量机制大量同步操作分散的问题
B. 管程每次只允许一个进程进入管程
C. 管程中 signal 操作的作用和信号量机制中的 semSignal 操作相同
D. 管程是被进程调用的，管程是语法范围，无法创建和撤销

  • 正确答案：C

解析 ：

  • C 错误（应选）：管程中的 signal 和信号量的 semSignal / V 操作不同。管程的 signal 如果没有等待进程则信号丢失（无记忆效应），而信号量的 V 操作会累加计数器值（有记忆效应）。
  • A 正确：管程将同步操作封装在一起，解决了 PV 操作分散在代码各处的问题
  • B 正确：管程保证每次只有一个进程在管程内执行
  • D 正确：管程是语言级构造，编译时定义，不像信号量可以动态创建 / 销毁

# 15. 当一进程因在记录型信号量 S 上执行 semSignal (S) 操作而导致唤醒另一进程后，S 的值为（）

A. > 0
B. < 0
C. >= 0
D. <= 0

  • 正确答案：D

解析 ：记录型信号量中 semSignal （V 操作）的逻辑：

  1. S.value++（先恢复资源计数）
  2. 若 S.value <= 0，说明有进程在等待，唤醒一个等待进程

唤醒另一进程的条件是 S.value <= 0（原有等待者），唤醒后 S.value 仍然是 <= 0。所以执行 semSignal 唤醒另一进程后，S 的值一定 <= 0。

举例：S = -2（2 个进程在等），执行 V 后 S = -1（1 个进程仍在等），唤醒一个。

# 16. 若信号 S 的初值为 2，当前值为 -1，则表示有（）个等待进程

A. 0
B. 1
C. 2
D. 3

  • 正确答案：B

解析 ：

  • 信号量初值为 2 → 有 2 个可用资源
  • 当前值为 -1 → 2 个资源已被分配完，还有 1 个进程在等待
  • 负数绝对值 = 等待进程数：|-1| = 1
  • 资源变化：2 → 1 → 0 → -1（3 次 P 操作）

# 17. 在下面关于并发性的叙述正确的是（）

A. 并发性是指若干事件在同一时刻发生
B. 并发性是指若干事件在不同时刻发生
C. 并发性是指若干事件在同一时间间隔发生
D. 并发性是指若干事件在不同时间间隔发生

并发并发

同一时间间隔内同一时间间隔内 ：并发（Concurrency）vs 并行（Parallelism）的经典区分：

并行并行 发生（宏观同时，微观交替）。单核 CPU 即可实现
同一时刻同一时刻 同时发生（微观也同时）。必须多核硬件支持

选项 A 描述的是并行，C 描述的是并发。

# 18. （）操作不是 semWait 信号量原语可完成的

A. 为进程分配处理机
B. 使信号量的值变小
C. 可用于进程的同步
D. 使进程进入阻塞状态

  • 正确答案：A

解析 ： semWait （P 操作）的功能：

  1. 将信号量值减 1（B 正确）
  2. 若值 < 0，将进程放入等待队列，进入阻塞态（D 正确）
  3. 可用于同步（C 正确，如 P (full) 等待产品）

semWait 调度器调度器 的职责，信号量只负责资源管理和阻塞 / 唤醒。

# 19. 下列关于临界资源的描述中正确的是（）

A. 临界资源是非共享资源
B. 临界资源是任意共享资源
C. 临界资源是互斥共享资源
D. 临界资源是同时共享资源

  • 正确答案：C

解析 ：临界资源（Critical Resource）的关键理解：

  • 它是共享的（多个进程都想用）
  • 它是互斥共享的（同一时间只能一个进程用）

临界资源 = 互斥共享资源。如打印机、共享变量等。

# 20. 设两个进程共用一个临界资源的互斥信号量 mutex，当 mutex = -1 时表示（）

A. 一个进程进入了临界区，另一个进程等待
B. 没有一个进程进入临界区
C. 两个进程都进入临界区
D. 两个进程都在等待

  • 正确答案：A

解析 ：互斥信号量 mutex 的初值为 1。当 mutex = -1 时：

  • 值从 1 → 0 → -1，经历了 2 次 P 操作
  • 第一个 P：mutex 从 1 变 0，第一个进程进入临界区
  • 第二个 P：mutex 从 0 变 -1，第二个进程试图进入，被阻塞加入等待队列

mutex = -1 的绝对值 1 = 等待进程数。即一个进程在临界区内，另一个进程在等待。
