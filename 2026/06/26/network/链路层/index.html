# 链路层

# 链路层提供的服务

链路层提供的服务是负责相邻节点之间的数据传输（transferring datagram from one node to adjacent node over a link）
不同的链路有不同的链路层协议，不同链路层协议提供不同服务。
主要提供两类服务：

  • 流量控制 (flow control)
  • 差错检测 (error detection)
  • 半双工和全双工 (half-duplex and full-duplex)

# 奇偶校验（Party Checking）

在固定长度 d bits 的 data 段后掺入一个 parity bit 位用于记录该段 data 内的 1 的个数：

  • even 校验：这 d+1bit 中 1 的个数为偶数
  • odd 校验：这 d+1bit 中的 1 的个数为奇数

# 循环冗余校验（CRC）

首先选择一个 n 次多项式，确定其模 2 的形式（二进制），然后在 data 的尾部上 n 个 0，然后进行除法运算得到余数 R，注意除法运算中，均采用类似异或的方式进行（非最高位都通过异或计算）。

# 信道划分

信道划分协议将信道资源划分为更小的 "片段"（时隙、频段、码片），分配给各节点使用。主要包括：TDMA（时分多址）、FDMA（频分多址）、CDMA（码分多址）。

# 时隙 ALOHA (Slotted ALOHA)：

前提：

  • 所有帧等长
  • 所有时隙等长且恰好能发送一帧
  • 只能在时隙开始时发送
  • 节点时钟是同步的
  • 多节点同时发送，所有节点都能检测到冲突
    实际运作：如果节点之间没有冲突（collision），那么节点就发送帧，如果节点之间存在冲突，那么节点都不发送，在后续的时隙中各个节点以概率 p 重传帧，重新检测是否冲突，直到不再冲突，成功发送。

    优点：
  • 单节点可以全速率传输
  • 去中心化，只需要时隙保持同步（sync）
    缺点：
  • collision 造成的时隙浪费
  • 存在空闲时隙（idle slots）
  • 时钟不一定能完全同步
  • 节点可能在传输数据包的同时才发现冲突（晚矣）

# 载波侦听多路访问（CSMA）

CSMA 总之就一句话：信道空闲，传送帧；信道忙，延迟传输。不打扰当前链路上传输的任务。
但是仍然存在 collision，因为存在传播时延，当帧在链路上传输的时候，其他节点可能还没有监听到，就像声音传播也是需要时间的。
所以也需要引入 Collision Detection 机制：当检测到链路忙时，等待足够时间（确保当前链路传输任务完成）后，再发送帧。

# 轮流发送（Taking turns）

所有节点采用轮流获得发送权限的方式发送帧，类似于排队，这样不会存在 collision。
一个 master 节点邀请多个 slave 节点来轮流发送帧。
在轮询过程中，使用令牌传递的方式，只有获得令牌的节点才能发送帧。这样看来有一个极大的缺点，如果一个节点拿到了令牌而不传递，导致整个 turn 出现问题：单点故障。

# MAC 地址

MAC 地址由 IEEE 分发，其前三个字节表示设备厂商信息。

  • Unicast 单播地址：用于一对一通信的地址。
  • Broadcast 广播地址：用于一对所有通信的地址，只作为目的地址。
  • 组播 / 多播地址：用于一对多通信的地址，只作为目的地址。

IP 地址和 MAC 地址的区别：
MAC 地址就像你的名字，而 IP 地址就像你的住址，一个 LAN 网络就像一个地区。你在所有的地区（LAN）中的名字（MAC）不变，但是当你从一个地区（LAN）搬到另一个地区（LAN）中后，你的住址（IP）变了。IP 取决于网络分布，而 MAC 取决于设备的网卡。

# 以太网的拓扑结构

  1. 总线拓扑结构：所有节点都连接在一条通信总线（在同一个冲突域 collision domain）
  2. 星型拓扑结构：使用一个交换机作为中心连接各端系统，这样将各个端系统分开，不会彼此冲突。

两个 Q：

  1. 为什么以太网帧中没有数据长度字段？
    答：因为以太网采用的帧的边界可以根据前导字的 SFD（帧开始位置）开始计算，跳过直到最后的长度减去 CRC 固定长度 4 个字节，即可得到数据段长度。
  2. 为什么以太网帧中的最短的数据长度是 46 字节？
    答：因为需要配合 CSMA/CD 冲突检测协议，需要保证帧的发送时间大于往返的传播时延（使得冲突能被检测出），除去前导码和 SFD 后，数据部分至少需要 46 字节。
