# 密钥管理

# 为什么需要密码管理

因为现代密码体制的安全性依赖于密钥，“一切秘密予于密钥之中！”。
密码算法可以公开，密码设备可以丢失，但密钥不能丢失或者泄露

概念：
密钥管理设计密钥的产生、检验、分发、传递、保管、使用、销毁的全部过程。

目的：
维持系统中各实体之间的密钥关系，以抗击可能的威胁：

  • 密钥的泄露
  • 秘密密钥或公开密钥的真实性丧失

密钥管理系统的要求：

  • 密钥难以被非法获取
  • 在一定条件下获取了以前的密钥用处也很小
  • 密钥的分配和更换过程对用户是透明的

密码管理的原则：

  • 全程安全原则：密钥从产生到销毁的全过程都必须采取妥善的安全管理
  • 最小权力原则：应当只分发给用户进行某一事物处理所需的最小密钥集合
  • 责任分离原则：一个密钥应当专职一种功能，不要让一个密钥兼任几种功能
  • 密钥分级原则：可以减少受保护的密钥的数量，又可简化密钥的管理工作（典型为三层分级密钥体系）
  • 密钥更新原则：密钥必须按时更新，密钥使用的时间越长，敌手截获的密文越多，破译密码的可能性就越大。
  • 密钥长度足够
  • 密码体制不同，密钥管理也应不同

# 典型的三层密钥体系

  1. 初级密钥（工作 / 会话密钥）：直接对数据进行加密和解密的密钥，处于最底层。会话密钥：一般是动态变化的，开启一次会话，就会更新一个新的会话密钥。文件加密密钥：对磁盘中的文件进行加密的密钥。
  2. 密钥加密密钥（二级密钥）：用来保护会初级密钥的密钥。
  3. 主密钥：对密钥加密密钥加密保护的密钥，处于最高层次。

# 分层次密钥管理的优点

  • 安全性能提高：下层密钥被破译不影响上层密钥安全，而且下层密钥可以动态更新，即使攻击者破译了一次会话的会话密钥也对整个体系没什么影响。
  • 密钥管理自动化：不需要人工介入会话密钥层的密钥分配，只需要人工配置主密钥层的密钥，其他两层密钥由密钥系统自动生成和更新。

# 密钥的生成

对于密钥的生成来说，应秉持好的密钥具有良好的随机性和密码特性的原则。

  1. 主密钥：严格保证随机性
  2. 二级密钥：可以采用随机数生成器
  3. 会话密钥：通过安全算法动态产生

# 密钥分配与协商

密钥分配与密钥协商过程都是用以在保密通信双方之间安全建立通信所使用的密钥的协议 (或机制)。

  1. 密钥分配：保密通信中的一方或者是可信的第三方（TA）生成并选择秘密密钥，然后将其安全地传送给通信的相关各方。包括 预分配和在线分配 这两种方式。
  2. 密钥协商：通信双方可以再一个公开信道上传送一些消息来共同建立一个安全的共享秘密密钥。典型算法为 Diffle-Hellman 密钥交换协议

二者区别：分配中，主权在于保密通信中的一方；协商中，主权在于保密通信中的多方。

# 密钥分配（分发）

密钥分配通过两种方式进行：

  1. 离线分配：通过非通信网络的可靠物理渠道分发密钥
  2. 在线分配：通过通信网络的密钥在线进行自动分发，主要通过建立一个密钥分发中心（KDC）来实现。


# 在线分发

  • 无 KDC 的对称密钥分发
  • 有 KDC 的对称密钥分发
  • 基于公钥密码体制的对称密钥分配

# 公开密钥的分配

  1. 公开发布：直接通过类似广播的方式，向每一个用户发送自己的公钥
  2. 公钥目录表：建立一个公用的公钥动态目录表，公用目录表的建立、维护以及公钥的分配必须由一个受信任的实体或组织承担，称这个实体或组织为公用目录表的管理员。
  3. 带认证的公钥分发（在线服务器形式）：在建立公钥目录的基础上增加认证功能

  4. 使用数字证书的公钥分发（离线服务器形式）：采用公钥证书的方案，允许用户通过公钥证书相互之间交换公钥而无需与公钥管理机构联系。公钥数字证书公钥证书是一种包含持证主体标识、持证主体公钥等信息，并由可信任的 CA 签署的信息集合
